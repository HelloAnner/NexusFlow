# SA 后台、账号与邀请注册功能设计

## 1. 功能目标

SA 后台用于系统初始化、组织人员基础数据维护、邀请注册、角色入口配置和运行状态管理。邀请注册用于降低使用门槛，不同角色使用不同邀请链接，注册后自动进入对应组织并获得对应角色的默认权限。

目标：

- 支持 SA 超级管理员维护系统全量基础数据。
- 支持按组织、角色、项目生成邀请链接。
- 支持用户通过邀请链接自助注册。
- 注册后自动绑定组织、角色、人员等级、项目归属等默认信息。
- 支持 SA 后台配置人员全部信息。
- 支持不同角色进入系统后看到不同首页、导航、快捷入口和待办内容。
- 所有账号、邀请、角色、组织变更必须留痕。

## 2. 角色定义

### 2.1 SA 超级管理员

SA 是系统最高管理角色，不参与普通业务派发审批，但拥有系统初始化和兜底维护权限。

SA 能力：

- 维护全量组织。
- 维护全量人员。
- 维护全量角色和权限。
- 生成和失效邀请链接。
- 配置注册策略。
- 配置角色默认首页和导航。
- 配置基础字典、任务模板、审批规则、告警规则。
- 查看系统运行状态。
- 查看审计日志。
- 处理账号锁定、人员合并、误注册修正。

SA 限制：

- SA 默认不作为任务发起人、审批人或验收人。
- SA 执行业务数据修正必须填写原因。
- SA 查看隐藏项目和个人事项需写入敏感访问审计。

### 2.2 系统管理员

系统管理员是可由 SA 授权的运维配置角色，权限范围可被限制在某个中心或组织。

系统管理员能力：

- 维护授权范围内组织和人员。
- 维护任务类型、标签、资料类型等配置。
- 查看配置异常。
- 不默认拥有隐藏项目查看权。
- 不默认拥有 SA 的账号兜底能力。

## 3. 邀请注册模型

### 3.1 邀请链接类型

邀请链接按用途分为：

- 组织角色邀请：加入指定组织并获得指定系统角色。
- 项目成员邀请：加入指定项目并获得项目身份。
- 管理角色邀请：加入指定组织并获得部门主任、技术监督、项目负责人等角色。
- 普通员工邀请：加入指定部门，默认角色为员工。
- 临时协作邀请：加入指定项目或任务，权限受有效期和范围限制。

### 3.2 邀请模板字段

| 字段 | 说明 | 必填 |
| --- | --- | --- |
| 邀请名称 | 便于后台识别 | 是 |
| 邀请类型 | 组织角色、项目成员、管理角色等 | 是 |
| 默认组织 | 注册后自动加入的主组织 | 是 |
| 默认工作室 | 可自动加入多个工作室 | 否 |
| 默认系统角色 | 注册后自动赋予的角色 | 是 |
| 默认人员等级 | 管理等级或专业等级 | 否 |
| 默认项目 | 注册后自动加入的项目 | 否 |
| 默认项目身份 | 负责人、核心成员、参与成员、支持人员 | 否 |
| 默认负责工作 | 项目中的负责工作 | 否 |
| 有效期 | 邀请链接失效时间 | 是 |
| 最大使用次数 | 单人或批量邀请 | 是 |
| 是否需要审批 | 注册后是否进入待审核 | 是 |
| 审核人来源 | SA、组织负责人、默认审批人 | 否 |
| 必填注册字段 | 注册表单字段集合 | 是 |
| 邀请状态 | 草稿、启用、停用、过期 | 是 |

### 3.3 邀请链接字段

- 邀请 token。
- 邀请模板 ID。
- 生成者。
- 生成时间。
- 失效时间。
- 已使用次数。
- 最大使用次数。
- 最近使用时间。
- 状态。

邀请链接必须不可枚举。链接 token 只保存哈希，页面只在生成时展示一次完整链接。

## 4. 注册流程

标准流程：

```text
SA 或授权管理员生成邀请链接
-> 用户打开邀请链接
-> 系统展示邀请对应组织和角色说明
-> 用户填写注册信息
-> 系统校验 token、有效期、使用次数和字段
-> 创建账号和人员档案
-> 自动绑定组织、角色、项目归属
-> 根据策略进入待审核或直接启用
-> 用户进入对应角色首页
```

### 4.1 注册字段

基础字段：

- 姓名。
- 登录账号。
- 手机号或邮箱。
- 初始密码或验证码。
- 工号，若组织要求可必填。
- 主组织，来自邀请模板，不允许用户自由修改。
- 默认角色，来自邀请模板，不允许用户自由修改。

可选字段：

- 专业等级。
- 技能标签。
- 项目归属。
- 负责工作。
- 个人简介。

### 4.2 自动入组规则

注册成功后系统自动写入：

- 主组织。
- 工作室归属。
- 系统角色。
- 人员等级。
- 项目归属。
- 项目身份。
- 负责工作。
- 每日标准工时。
- 默认首页视图。

如果邀请链接配置了“需要审批”，则用户状态为“待审核”。待审核用户只能看到资料完善页和审核状态页，不能访问业务数据。

### 4.3 重复注册处理

匹配规则：

- 登录账号重复。
- 工号重复。
- 手机号或邮箱重复。

处理方式：

- 已存在账号时，不创建新人员。
- 可将邀请绑定到已有账号，补充组织、角色或项目归属。
- 若存在冲突，进入 SA 待处理列表。

### 4.4 邀请失效规则

邀请链接在以下场景失效：

- 超过有效期。
- 达到最大使用次数。
- 被 SA 或生成者停用。
- 关联组织、角色或项目被停用。
- 邀请模板被停用。

失效后访问链接时展示失效原因和联系管理员入口。

## 5. 人员全量配置

SA 后台支持配置人员全部信息：

- 基础信息。
- 登录账号。
- 主组织。
- 工作室。
- 系统角色。
- 管理等级。
- 专业等级。
- 项目归属。
- 项目身份。
- 负责工作。
- 技能标签。
- 工作状态。
- 每日标准工时。
- 是否可派发。
- 是否启用。
- 默认首页。
- 数据范围扩展授权。
- 隐藏项目授权。
- 联系方式。
- 审计备注。

规则：

- SA 可编辑全量字段。
- 普通管理员只能编辑授权范围字段。
- 管理者不能给自己提升角色或扩大数据范围。
- 角色、组织、项目、隐藏授权变更必须写审计日志。
- 账号禁用后，人员历史任务和资料署名保留。

## 6. 角色默认入口

角色默认入口由 SA 或配置管理员维护。

配置项：

- 默认首页组件。
- 默认导航菜单。
- 默认数据范围。
- 默认时间范围。
- 快捷入口。
- 待办类型。
- 是否展示报表。
- 是否展示配置入口。
- 是否展示 SA 后台入口。

用户拥有多个角色时：

- 系统选择优先级最高角色作为默认首页。
- 用户可在权限允许时切换角色视角。
- 切换角色视角只改变页面默认范围，不改变实际权限。

## 8. 安全与审计

必须审计：

- 邀请模板创建、修改、停用。
- 邀请链接生成、复制、停用。
- 邀请链接使用。
- 注册成功、注册失败。
- 注册审批通过、拒绝。
- 人员组织、角色、项目归属变更。
- SA 查看隐藏项目或个人事项。
- SA 修改账号状态。

安全要求：

- 邀请 token 使用高强度随机值。
- 数据库存储 token 哈希。
- 邀请链接支持有效期和使用次数。
- 注册接口限制频率。
- 密码或验证码策略可配置。
- SA 敏感操作二次确认。

## 9. 与其他模块联动

- 组织人员：注册后自动创建或绑定人员档案。
- 权限：邀请模板决定初始角色和数据范围。
- 项目：项目邀请自动写入项目归属和项目身份。
- 首页：角色默认入口决定注册后首屏内容。
- 配置中心：维护角色入口、邀请策略和注册字段。
- 审计：记录账号、邀请和权限变更。
