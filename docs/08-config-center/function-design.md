# 配置中心功能设计

## 1. 功能目标

配置中心用于把系统规则沉淀为可维护配置，减少硬编码。它决定任务类型、字段、模板、流程、权限、告警、页面视图和工具入口。

目标：

- 支持不同任务类型加载不同字段和流程。
- 支持角色权限和数据范围统一维护。
- 支持审批、告警、资料要求配置化。
- 支持首页、甘特图、列表默认视图按角色配置。
- 支持配置变更留痕。

## 2. 配置模块

- 组织与人员配置。
- 角色与权限配置。
- 项目配置。
- 任务类型配置。
- 任务模板配置。
- 分工模板配置。
- 审批规则配置。
- 告警规则配置。
- 资料类型配置。
- 技能标签配置。
- 首页与视图配置。
- 工具台配置。
- 邀请注册配置。
- 角色入口配置。
- 系统运行配置说明。

## 2.1 系统运行配置说明

系统运行配置不在页面中直接维护，部署时通过 `.env` 提供。配置中心只展示只读状态和健康检查结果。

`.env` 负责：

- `APP_PORT`，默认 `8089`。
- `DATABASE_URL`。
- `REDIS_URL`。
- `S3_ENDPOINT`、`S3_BUCKET`、对象存储访问密钥。
- `SESSION_SECRET`、`JWT_SECRET`。
- 上传大小、导出过期时间、搜索后端等运行参数。

规则：

- 不使用 Docker 作为部署前提。
- 基础组件只要求可连接，不关心部署位置。
- 页面不得展示密钥明文。
- 健康检查只展示连接状态、延迟和错误摘要。

## 2.2 项目配置

配置项：

- 项目类型。
- 项目级别。
- 项目状态。
- 项目成员身份。
- 隐藏项目授权范围。
- 项目资料要求。
- 项目归档校验规则。

项目配置影响：

- 新建项目表单。
- 项目成员管理。
- 项目任务默认模板。
- 项目甘特图和项目报表。

## 3. 任务类型配置

字段：

- 类型名称。
- 父级类型。
- 是否启用。
- 是否参与负载。
- 是否改变人员状态。
- 默认可见性。
- 默认审批流程。
- 默认资料要求。
- 默认字段模板。

示例：

- 休假：参与负载，改变人员状态，不要求分工，不要求资料。
- 科研：参与负载，要求负责人、分工、里程碑、资料。
- 出差：参与负载，改变人员状态，可要求出差报告。
- 安全：可参与负载，可要求检查记录、整改报告或安全学习材料。

## 4. 任务模板配置

模板内容：

- 基础字段。
- 自定义字段。
- 里程碑模板。
- 分工模板。
- 必需资料。
- 验收标准。

字段属性：

- 字段名称。
- 字段类型。
- 是否必填。
- 默认值。
- 是否在列表展示。
- 是否允许编辑。
- 可见角色。

## 5. 审批规则配置

审批触发条件：

- 跨部门派发。
- 负载超限。
- 全天任务重叠。
- 隐藏项目授权。
- 任务重大变更。
- 强制安排。

审批人来源：

- 目标成员部门主任。
- 目标成员部门副主任。
- 技术监督。
- 项目负责人。
- 分管领导。
- 指定角色。

规则要求：

- 支持按任务类型配置。
- 支持按风险等级配置。
- 支持审批超时提醒。
- 支持升级审批。

## 6. 告警规则配置

告警类型：

- 任务延期。
- 分工延期。
- 审批超时。
- 资料缺失。
- 成果未提交。
- 负载超限。
- 验收退回。

配置项：

- 触发条件。
- 提前提醒天数。
- 提醒对象。
- 提醒渠道。
- 风险等级。
- 是否生成待办。

## 7. 视图配置

支持按角色配置：

- 首页组件。
- 任务列表默认筛选。
- 甘特图默认维度。
- 全局搜索默认范围。
- 项目列表默认筛选。
- 冲突中心默认范围。
- 人员列表默认列。

## 7.1 邀请注册配置

配置项：

- 邀请链接默认有效期。
- 邀请链接最大使用次数。
- 是否允许批量邀请。
- 注册字段必填规则。
- 是否允许已有账号绑定邀请。
- 待审核策略。
- 审核人来源。
- 注册成功后的默认首页。

规则：

- SA 可维护全局邀请策略。
- 授权管理员只能维护授权组织范围内邀请模板。
- 邀请模板引用的组织、角色、项目停用后，模板自动失效。

## 7.2 角色入口配置

配置项：

- 角色。
- 默认首页。
- 导航菜单。
- 首页组件。
- 默认数据范围。
- 默认时间范围。
- 快捷入口。
- 待办类型。
- 是否展示报表。
- 是否展示配置入口。
- 是否展示 SA 后台入口。

规则：

- SA 超级管理员必须进入 SA 后台首页。
- 待审核用户只能进入审核状态页。
- 多角色用户可配置默认角色视角。

## 8. 配置发布

配置状态：

- 草稿。
- 已发布。
- 已停用。

发布规则：

- 草稿配置不影响线上流程。
- 发布后只影响新建任务，历史任务使用创建时模板快照。
- 权限和告警规则可即时生效。
- 配置变更必须保留版本。
