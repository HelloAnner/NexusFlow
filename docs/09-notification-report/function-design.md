# 通知、待办与报表功能设计

## 1. 功能目标

通知和待办负责驱动用户处理工作，报表负责让管理者看到整体运行情况。

目标：

- 所有需要用户处理的事项都形成待办。
- 所有关键变化都能通知相关人。
- 待办可直达具体处理页面。
- 报表指标口径清晰，和任务、负载、资料数据一致。

## 2. 待办类型

待办包括：

- 任务确认。
- 跨部门审批。
- 分工进度填报。
- 阶段成果提交。
- 阶段成果确认。
- 任务验收。
- 验收退回处理。
- 冲突处理。
- 资料补充。
- 配置异常处理。

## 3. 待办字段

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| 待办编号 | 自动生成 | 全局唯一 |
| 待办类型 | 枚举 | 审批、填报、验收等 |
| 标题 | 文本 | 页面展示 |
| 关联对象 | 任务/分工/资料/审批 | 跳转目标 |
| 处理人 | 人员 | 当前待办负责人 |
| 发起人 | 人员 | 来源人 |
| 截止时间 | 日期时间 | 可为空 |
| 风险等级 | 枚举 | 低、中、高 |
| 状态 | 枚举 | 待处理、已处理、已关闭 |

## 4. 通知场景

通知触发：

- 被设置为任务负责人。
- 被加入任务。
- 分工创建或变更。
- 协调审批待处理。
- 审批通过或拒绝。
- 任务即将截止。
- 分工延期。
- 成果被退回。
- 任务验收通过。
- 任务归档。
- 强制安排影响已有任务。

通知对象：

- 发起人。
- 负责人。
- 成员。
- 审批人。
- 验收人。
- 受影响任务负责人。

## 5. 报表类型

基础报表：

- 中心任务总览。
- 部门任务总览。
- 人员负载排行。
- 延期任务清单。
- 重点任务进度。
- 跨部门协调统计。
- 资料归档完整率。
- 成果提交及时率。

## 6. 指标口径

任务完成率：

```text
已完成任务数 / 应完成任务数
```

人员负载率：

```text
某周期内预计投入工时 / 某周期内标准工时
```

延期任务数：

```text
当前日期 > 截止日期 且 状态不是已完成或已归档
```

资料完整率：

```text
已确认必需资料数 / 模板要求必需资料数
```

协调通过率：

```text
通过协调单数 / 已完成协调单数
```

## 7. 数据权限

报表必须遵守数据范围：

- 员工只能看个人数据。
- 负责人看所负责任务。
- 部门主任看本部门。
- 中心领导看中心。
- 隐藏项目未经授权不得进入统计明细。
