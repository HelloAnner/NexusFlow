# 新建任务（New Task）

> 参考 Linear 的 `Cmd + N` 快速创建和 Wrike Request Forms 的表单化派发，让新建任务既快又规范。

## 1. 页面目标

提供统一的任务创建入口，支持快速创建和完整派发两种模式。复杂流程通过抽屉/对话框完成，不跳转独立页面。

## 2. 触发方式

- 顶部栏 **+ New** → New Task。
- 快捷键 `Cmd/Ctrl + N`。
- 命令面板 `/new task`。
- 项目空间内点击 **New Task**（自动关联项目）。
- 人员详情面板点击 **派任务**（自动选择该人员）。
- 甘特图/日历点击空白日期（自动预填时间）。

## 3. 快速创建模式

轻量对话框，字段最少：

- 任务名称（必填）
- 任务类型（必填，默认上次使用）
- 负责人（必填，默认自己）
- 截止时间（必填）
- 优先级（选填）
- 关联项目（选填）

创建后自动打开任务详情面板继续完善。

### 步骤 1：基本信息

- 名称、类型、级别、优先级
- 所属组织、关联项目、可见性
- 开始/截止时间、概述、成果要求、里程碑

### 步骤 2：负责人与成员

- 负责人选择器（带负载提示）
- 成员选择器：
  - 按姓名/组织/技能筛选
  - 从项目成员选择
  - 系统推荐候选人
- 人员卡片展示：姓名、部门、技能、当前负载、冲突提示、是否跨部门

### 步骤 3：分工草案

- 为每个成员创建分工
- 默认单成员自动生成同名分工
- 字段：工作内容、类型、起止时间、每日投入/全天、成果要求

### 步骤 4：冲突检查

- 实时展示：工时超载、全天重叠、状态冲突、跨部门审批、隐藏项目授权
- 每个冲突提供建议动作

### 步骤 5：确认发布

- 汇总信息
- 确认后进入待确认/待协调/进行中

## 5. 草稿机制

- 每步自动保存草稿。
- 草稿不占用人员工时。
- 关闭抽屉后提示已保存。

## 6. 模板选择

- 创建前可选择模板：科研、市场、出差、休假、自定义。
- 选择后自动填充字段和分工。

## 7. 快速创建细节

- 快速创建只服务于“先占坑再完善”，字段必须少；创建成功后立即打开 Task Detail，让用户继续补概述、成果和分工。
- 任务名称为空时不允许提交；截止时间默认可用“今天、明天、本周五、下周一”快捷项。
- 负责人默认自己，但从人员详情触发时默认该人员；从项目空间触发时默认项目负责人。
- 关联项目不是必填，但如果用户在项目空间、甘特项目行、项目详情中触发，必须自动带入且可修改。
- 快速创建失败时保留已填内容，错误聚焦到第一个失败字段。

## 8. 完整派发细节

- 步骤 1 保存后即可生成草稿任务 ID，后续成员、分工、附件都关联草稿。
- 步骤 2 系统推荐候选人时，需要展示推荐理由：技能匹配、负载可用、同项目经验。
- 步骤 3 分工时间默认落在任务起止时间内，不允许超出任务时间范围，除非用户先调整任务时间。
- 步骤 4 冲突检查分为阻断型和提示型：权限缺失、审批缺失属于阻断；轻度超载可提示后继续提交。
- 步骤 5 确认发布需要明确任务发布后进入哪个状态：待确认、待协调或进行中。
- 用户从任一步返回修改时，后续步骤中受影响的数据需要标记“需要重新检查”。

## 9. 草稿、模板与权限

- 草稿每 10 秒或字段失焦自动保存，关闭抽屉后可从个人草稿入口继续。
- 草稿不占用工时、不触发通知、不进入报表统计。
- 模板变更只影响之后创建的任务，已套用模板的草稿保留当时字段快照。
- 用户无权创建跨部门任务时，成员选择器仍可搜索到人员基础信息，但发布时引导发起协调或申请权限。
- 隐藏项目授权不足时，不展示隐藏项目敏感名称，只提示需要授权审批。
- 发布成功后需要同步 Home、My Work、Project Tasks、Schedule，并向相关人员发送 Inbox 待确认或通知。

## 建议组件

新建任务页面是高频创建入口，需要在“轻量快速”与“完整规范”之间切换。建议基础控件统一使用 **shadcn/ui**，并通过 CSS 变量映射到 NexusFlow 色板：快速创建弹窗背景 `--bg-secondary`（#FFFFFF），完整派发抽屉 `--bg-secondary`，表单字段背景 `--bg-tertiary`（#F5F4F2）或透明，边框 `--border-subtle`（rgba(0,0,0,0.05)），文字 `--text-primary`（#1A1A1A）/`--text-muted`（#7A7A7A）。冲突提示统一使用语义色 `--error`/`--warning`，成功/正常提示使用 `--success`。

**快速创建模式**使用 shadcn/ui `Dialog`（居中，宽度 480px），字段包括任务名称、任务类型、负责人、截止时间、优先级、关联项目。表单校验使用 **react-hook-form + zod**，必填项错误提示 `--error`。任务类型、负责人、关联项目使用 shadcn/ui `Select` 或自研可搜索选择器；截止时间使用 shadcn/ui `DatePicker`（基于自定义日期弹窗，背景 `--bg-secondary`，选中 `--text-primary`，今天 `--bg-tertiary`）；优先级使用 shadcn/ui `Badge` 或 `RadioGroup`。创建后自动打开任务详情面板。

**完整派发模式**使用 **vaul** 或 shadcn/ui `Sheet`（右侧滑出，宽度 720px），采用 5 步向导。步骤条建议自研 **Stepper 组件**：已完成步骤使用 `--text-primary` 实心圆，当前步骤使用 `--text-primary` 边框圆，未进行步骤使用 `--text-muted`。步骤标题 `--text-primary`，描述 `--text-muted`。步骤间支持返回修改，已填写数据保留。

步骤表单统一使用 **react-hook-form + zod** 做分步校验与整体验证：
- 步骤 1（基本信息）：名称 `Input`、类型/级别/优先级/所属组织/关联项目/可见性 `Select`、起止时间 `DatePicker` 范围、概述 `Textarea`、成果要求与里程碑使用自研清单输入组件。
- 步骤 2（负责人与成员）：自研 **Person Selector 组件**，支持按姓名/组织/技能搜索，从项目成员选择，系统推荐候选人。人员卡片使用 shadcn/ui `Card`，展示头像（`Avatar`）、姓名、部门、技能标签、当前负载（语义色）、冲突提示（`--error`）、是否跨部门 badge。负载提示使用进度条或文字，超载使用 `--error`。
- 步骤 3（分工草案）：自研分工列表，每行包含工作内容 `Input`、类型 `Select`、起止时间 `DatePicker`、每日投入/全天 `Switch`、成果要求 `Textarea`。默认单成员自动生成同名分工。支持新增、删除、复制分工。
- 步骤 4（冲突检查）：自研 **Conflict Visualization 组件**，实时展示工时超载、全天重叠、状态冲突、跨部门审批、隐藏项目授权。每项冲突使用语义色左侧竖线（`--error` 高、`--warning` 中），推荐动作使用 shadcn/ui `Button` 的 `ghost`/`outline` 变体。
- 步骤 5（确认发布）：汇总信息使用只读表单展示，确认按钮填充 `--text-primary`。

草稿机制每步自动保存，关闭抽屉后提示已保存，使用 **Sonner**，背景 `--bg-secondary`，语义色反馈。模板选择弹窗使用 shadcn/ui `Dialog`，模板卡片使用 `Card`，hover `--bg-tertiary`。

所有日期校验与计算由 **date-fns** 处理，确保起止时间、截止时间与冲突检查逻辑一致。新建任务触发入口包括顶部栏 + New、快捷键 `Cmd/Ctrl + N`、命令面板 `/new task`、项目空间 New Task、人员详情面板派任务、甘特图/日历点击空白日期。模板引擎建议后端维护模板与默认值，前端根据模板类型自动填充表单；冲突检查可在前端预检后由后端最终校验，保证数据一致性。
