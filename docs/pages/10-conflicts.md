# 冲突与风险中心（Conflicts）

> 参考 Linear Triage 和 Wrike Risk Management，将所有风险和冲突集中到一个处理队列，支持快速决策和批量处理。

## 1. 页面目标

集中展示系统中所有待处理的风险和冲突，让管理者能够快速发现、分析、处理并留痕。

## 2. 整体布局

```
┌────────────────────────────────────────────────────────────────┐
│  Conflicts            [筛选] [排序] [批量处理] [导出]           │
├───────────────────────┬────────────────────────────────────────┤
│                       │                                        │
│  冲突列表             │  冲突详情 / 处理面板                   │
│                       │                                        │
├───────────────────────┴────────────────────────────────────────┤
│  汇总统计面板                                                  │
└────────────────────────────────────────────────────────────────┘
```

## 3. 顶部筛选

- 类型：人员超载 / 时间冲突 / 跨组织 / 权限 / 资源。
- 风险等级：高 / 中 / 低。
- 状态：待处理 / 处理中 / 已解决 / 已忽略。
- 时间范围。
- 涉及人员 / 项目。

## 4. 冲突列表

每行展示：

- 类型图标。
- 风险等级 badge。
- 标题与描述。
- 涉及人员/任务/项目。
- 发生时间。
- 状态。
- 推荐动作。

支持分组：按风险等级 / 类型 / 责任人。

## 5. 冲突详情 / 处理面板

- 冲突详细信息。
- 涉及对象可点击跳转。
- 处理动作：
  - 调整人员
  - 调整时间
  - 调整投入
  - 升级审批
  - 强制安排（需原因）
  - 标记已解决
  - 忽略

## 6. 汇总统计

- 待处理总数 / 高风险数 / 已解决数。
- 按类型分布图表。
- 部门负载预警。
- 趋势曲线。

## 7. 批量处理

- 选中多个冲突后批量转交、标记已解决、忽略。

## 8. 处理流程细节

- 冲突列表默认按风险等级、发生时间、影响范围排序，高风险且未处理的冲突始终排在前面。
- 用户点击冲突后，右侧详情面板必须展示“为什么发生、影响什么、建议怎么处理”三段信息。
- 推荐动作不应只是按钮文案，需要展示执行后预计影响，例如“调整到 6 月 24 日，新增 0 个冲突”。
- 标记已解决前，系统应重新校验冲突是否仍存在；如果仍存在，需要要求填写说明或改为忽略。
- 忽略冲突必须填写原因和有效期，避免永久隐藏真实问题。
- 强制安排需要更高权限，并写入审计日志，同时通知受影响人员。

## 9. 类型差异

- 人员超载：展示人员当日任务构成、已分配投入和可用工时。
- 时间冲突：展示冲突任务在时间轴上的重叠位置，提供调整日期入口。
- 跨组织：展示涉及组织、审批链、当前审批状态。
- 权限冲突：展示当前可见性规则和缺失授权，不直接暴露隐藏项目敏感信息。
- 资源冲突：展示资源占用时间、申请人、替代资源建议。

## 10. 状态与联动

- 冲突处理后需要同步更新 Schedule、Task Detail、Home 风险摘要和相关用户 Inbox。
- 已解决冲突保留在历史中，默认 30 天内可见；忽略中的冲突到期后重新进入待处理。
- 批量处理只能处理同类型或同动作兼容的冲突，混合选择时禁用不适用动作。
- 冲突来源对象被删除时，冲突自动进入已解决或失效状态，并保留审计记录。
- 统计面板点击指标后反向筛选列表，而不是跳转离开当前页面。

## 11. 当前落地切片

本阶段优先补齐冲突中心的处理队列体验，不新增后端批量接口，先复用现有 `resolve` / `force` 单条接口顺序执行，确保可在服务器真实验证。验收点如下：

- 冲突列表必须支持多选，选中后顶部出现批量操作栏，展示已选数量、高风险数量和可执行动作。
- 批量标记已解决必须要求填写统一处理说明，并对每条选中冲突调用 `POST /api/conflicts/{id}/resolve`；成功后刷新列表和详情。
- 批量强制排期必须要求填写业务原因，并对每条选中冲突调用 `POST /api/conflicts/{id}/force`；弹窗文案必须明确“会写入审计、仍保留风险痕迹”。
- 统计卡片支持反向筛选：点击“高风险”切换 `risk_level=high`，点击“待处理”切换 `status=open`，点击“人员超载”切换 `conflict_type=overload`。
- 推荐动作必须展示预计影响，不只展示按钮文案；至少包含“预计新增冲突”“需要审批/负责人确认”“会影响的入口”三类提示。
- 批量处理失败时保留当前选择和弹窗说明，展示失败数量或后端错误，不能静默吞掉部分失败。

## 建议组件

冲突与风险中心是一个处理队列型页面，强调“快速决策 + 批量处理 + 数据洞察”。基础控件使用 **shadcn/ui**，并通过 CSS 变量映射到 NexusFlow 色板：列表区背景 `--bg-secondary`（#FFFFFF），筛选/统计区 `--bg-tertiary`（#F5F4F2），边框 `--border-subtle`（rgba(0,0,0,0.05)），文字 `--text-primary`（#1A1A1A）/`--text-muted`（#7A7A7A）。风险等级颜色严格使用语义色：`--error`（高）、`--warning`（中）、`--success`（低），状态颜色同样取自语义色，不新增分类色。

顶部筛选区使用 shadcn/ui `DropdownMenu`、`Select`、`Popover`（时间范围）、`Button`。筛选维度包括类型（人员超载/时间冲突/跨组织/权限/资源）、风险等级（高/中/低）、状态（待处理/处理中/已解决/已忽略）、时间范围、涉及人员/项目。批量处理按钮在未选中时禁用，选中后启用；危险批量操作（批量忽略）使用 `outline` 变体并设置 `--error` 文字色。

冲突列表建议自研 **Conflict List Item 组件**，行高 56px，左侧类型图标使用 Lucide 线图标（`--text-muted`），标题 `--text-primary`，描述 `--text-muted`，涉及人员/任务/项目可点击跳转（hover 下划线），发生时间 `--text-muted`，状态 badge 使用语义色。hover 行背景 `--bg-tertiary`，右侧渐显快捷处理按钮（调整人员/调整时间/调整投入/升级审批/强制安排/标记已解决/忽略），使用 shadcn/ui `Button` 的 `ghost` 变体，尺寸 `sm`。高风险行可额外左侧 3px `--error` 竖线提示。列表支持分组：按风险等级/类型/责任人。长列表使用 **TanStack Virtual**。

冲突详情 / 处理面板桌面端使用 shadcn/ui `Sheet`（右侧 560px），移动端使用 **vaul**。面板背景 `--bg-secondary`，内部分割线 `--border-subtle`。面板内展示冲突详细信息、涉及对象链接、推荐动作。处理动作根据冲突类型动态渲染，使用 shadcn/ui `Button` 分组。强制安排、忽略等敏感操作使用 shadcn/ui `AlertDialog` 要求填写原因并记录审计。操作结果使用 **Sonner** 反馈，成功 `--success`，失败 `--error`。

汇总统计面板位于页面底部或右侧，使用 **recharts** 或 **tremor** 绘制：待处理总数/高风险数/已解决数使用指标卡片，类型分布使用柱状图（颜色固定 `--text-primary` 或按类型使用语义色：人员超载 `--error`、时间冲突 `--warning`、跨组织 `--info`、权限 `--text-primary`、资源 `--text-muted`），趋势曲线使用折线图（线条 `--text-primary`，网格线 `--border-subtle`），部门负载预警使用自研列表或表格。所有图表关闭默认彩色主题，工具提示背景 `--bg-secondary`、边框 `--border-subtle`。

批量处理通过 shadcn/ui `Checkbox` 实现多选，Shift + 点击连续多选。选中后顶部出现悬浮批量操作栏，背景 `--bg-secondary`、边框 `--border-subtle`、阴影 `shadow-sm`，操作包括批量转交、批量标记已解决、批量忽略。冲突检测建议由后端定时任务触发，前端通过 WebSocket 或轮询获取最新冲突列表；高风险冲突可推送通知到 Inbox。
