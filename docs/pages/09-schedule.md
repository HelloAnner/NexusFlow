# 排期中心（Schedule）

> 参考 Instagantt、Wrike Gantt 和 Linear Timeline，将甘特图、日历、负载整合为一个统一的排期中心。
> 页面入口在主导航中显示为“排期”。

## 1. 页面目标

提供全局视角的排期与资源协调视图，让管理者能够发现冲突、调整计划、平衡资源。

## 2. 整体布局

```
┌────────────────────────────────────────────────────────────────┐
│  排期中心             [甘特] [日历] [负载] [筛选] [导出]        │
├────────────────────────────────────────────────────────────────┤
│  统计卡片：进行中 / 待验收 / 开放风险 / 已归档                  │
├────────────────────────────────────────────────────────────────┤
│  视图工具栏：时间粒度 / 日期范围 / 只看风险 / 隐藏已完成        │
├────────────────────────────────────────────────────────────────┤
│                                                                │
│  Main View（Gantt / Calendar / Workload）                       │
│                                                                │
└────────────────────────────────────────────────────────────────┘
```

- 顶部统计卡片区（进行中、待验收、开放风险、已归档）始终保留，用于快速感知当前范围下的任务分布。

## 3. 甘特视图

### 3.1 布局

- 左侧：可收起的树形分组列（任务/人员/部门/项目），默认收起，主区域直接展示时间轴。
- 右侧：SVG 时间轴，支持平移、缩放、粒度切换。
- 顶部：时间粒度切换（日/周/月/季度）、日期范围、今天标记、回到今天、左侧树展开/收起。

### 3.2 甘特条

- 任务条：24px 高度，圆角 4px。
- 进度填充：半透明覆盖层。
- 里程碑：菱形标记。
- 风险：红色边框或条，条尾显示红色圆点标记。
- 冲突：条上方红色闪电图标（本次先展示风险等级）。
- 今天：蓝色虚线竖线。

### 3.3 分组维度

- 按任务
- 按人员
- 按部门
- 按项目

### 3.4 交互

- 点击任务条或左侧行，右侧滑出详情面板，展示：
  - 名称
  - 含义（summary）
  - 负责人
  - 归属项目
  - 状态
  - 开始 / 截止时间
  - 进度
  - 风险等级
- 详情面板提供“打开详情”按钮，使用新标签页打开任务/项目详情。
- 时间轴支持：
  - 通过顶部日期输入调整范围。
  - 拖拽时间轴平移查看。
  - 切换日/周/月/季度粒度。
  - “回到今天”快速定位当前日期。

## 4. 日历视图

- 月/周视图。
- 展示任务截止、开始、里程碑、人员休假。
- 点击日期创建任务（预填日期）。

## 5. 负载视图

- 人员为行，日期为列。
- 单元格颜色表示负载。
- 支持按部门/角色分组。
- 点击单元格查看当天任务。

## 6. 交互

- 点击任务条打开任务详情面板。
- 点击人员行打开人员详情面板负载页签。
- 点击冲突标记打开冲突详情。
- 支持拖拽调整日期（需权限，后端校验）。

## 7. 排期操作规则

- 默认进入甘特视图，时间范围为本周前 7 天到未来 30 天，用户可切换到月/季度。
- 左侧树形列默认收起，用户可通过工具栏按钮展开。
- 拖拽任务条只改变计划日期，不直接改变任务状态；状态仍由任务详情中的生命周期动作控制。
- 调整日期后必须重新计算人员负载、任务冲突和里程碑影响，并在任务条上即时反馈风险。
- 如果调整跨越项目起止时间，前端提示“超出项目周期”，是否允许由后端权限和规则决定。
- 批量调整任务时间时，必须展示影响清单：涉及任务数、人员数、新增冲突数、需要审批数。
- 今天线始终可见；当用户滚动到远期时，工具栏提供“回到今天”。

## 8. 数据表达细节

- 甘特左侧树形列应保留任务名称、负责人、状态三类最关键信息，更多字段放到 hover tooltip 或详情面板。
- 时间轴缩放不改变选中对象；用户从日切到月后，当前选中任务仍需保持高亮。
- 里程碑没有持续时间，用菱形标记；延期里程碑使用错误语义色边框。
- 休假、出差、全天占用在负载视图中必须与普通任务区分，避免被误判为可调整任务。
- 只看风险模式隐藏无风险任务，但保留其所在项目/人员分组，避免上下文丢失。

## 9. 性能与边界

- 大数据量下先渲染当前视口内任务和日期，非视口内容延迟渲染。
- 横向和纵向滚动时表头、左侧树列、今天线必须同步，不允许错位。
- 导出时使用当前筛选、分组和时间范围，不默认导出全量。
- 用户无权限拖拽时，任务条仍可点击查看，但鼠标不显示拖拽态。
- 冲突标记点击失败时，保留任务详情入口，并提示冲突详情暂不可用。

## 10. 当前落地切片：筛选甘特视图

- `/gantt` 先以甘特视图为主，补齐文档中最关键的筛选与口径：时间范围、项目、负责人、组织、状态、只看风险和维度切换。
- 前端筛选栏中的项目、负责人、组织来自 `/api/projects`、`/api/users`、`/api/orgs/tree`，筛选值同步到 URL query，刷新后可恢复。
- 前端请求 `/api/gantt` 时传递 `start`、`end`、`project_id`、`owner_id`、`org_id`、`status`、`risk_only`；后端必须在权限范围内重新过滤，不能只依赖前端过滤。
- `GET /api/gantt/summary` 需要同样应用当前账号的数据范围，至少返回 `in_progress`、`acceptance_pending`、`archived` 和 `open_risk`，避免管理者看到全局未授权统计。
- “只看风险”模式由后端和前端双重保障：后端仅返回有 open conflict 的任务，前端在维度聚合后保留有风险分组。
- 导出按钮当前先生成 JSON 数据包，包含 `{ generated_at, filters, dimension, granularity, items }`，用于后续 PDF/PNG 导出前的数据口径验证。
- 权限提示在工具栏常驻展示：当前甘特图是 `data_scope_applied=true` 的服务器过滤结果；隐藏项目/任务以脱敏标题展示，不暴露未授权名称。
- 后端 `/api/gantt` 返回的任务对象必须包含 `summary`，用于右侧详情面板的“含义”展示。

## 建议组件

排期中心是技术难度最高的页面之一，尤其是甘特图，不建议直接套用第三方重型库，而是自研 SVG/Canvas 核心 + 虚拟滚动，以获得性能与视觉风格的完全控制。基础控件使用 **shadcn/ui**，并通过 CSS 变量锁定色调：页面背景 `--bg-primary`（#FAF9F7），视图容器 `--bg-secondary`（#FFFFFF），工具栏/侧边树 `--bg-tertiary`（#F5F4F2），所有边框 `--border-subtle`（rgba(0,0,0,0.05)），文字 `--text-primary`（#1A1A1A）/`--text-muted`（#7A7A7A）。甘特图内部所有颜色必须从系统语义色与中性色中选取，禁止引入新的项目彩虹色。

**甘特图（Gantt）核心**建议自研：
- 左侧树形分组列使用 div 表格，表头背景 `--bg-tertiary`，行高 40px，分割线 `--border-subtle`，展开/折叠图标使用 Lucide 线图标（`--text-muted`）。树节点按任务/人员/部门/项目分组，支持折叠展开。
- 右侧时间轴使用 SVG 绘制，日期刻度、网格线、今天线、任务条、里程碑、风险标记均通过 **date-fns** 生成坐标。时间粒度支持日/周/月/季度，刻度文字 `--text-muted`，周末/节假日背景 `--bg-tertiary` 30% 透明度。
- 任务条高度 24px，圆角 4px，填充使用 `--text-primary` 80% 透明度；进度填充使用 `--text-primary` 实心覆盖；有风险任务条边框 2px `--error`；里程碑菱形 `--text-primary`；冲突标记使用 `--error` 闪电图标；今天线为 2px `--info` 虚线。
- 双虚拟滚动：左侧任务树与右侧时间轴使用 **TanStack Virtual** 分别垂直虚拟化，水平方向仅时间轴虚拟化（按日期列切片）。对于 1 万条任务的目标，首屏渲染控制在 300ms 内。滚动时两侧同步滚动，使用统一的 scrollTop。
- 任务条拖拽调整日期使用 **@dnd-kit**，拖拽时任务条透明度 80%，拖拽到位后调用后端校验并重新计算冲突。拖拽光标使用 `--text-primary`。

顶部视图切换（甘特/日历/负载）使用 shadcn/ui `Tabs`。视图工具栏使用时间粒度 `Select`、日期范围 `Popover`、只看风险/隐藏已完成 `Switch`、导出 `Button`。Tooltip 用于 hover 任务条显示详情，背景 `--bg-secondary`，边框 `--border-subtle`。

**日历视图**使用 **react-big-calendar** 并完全自定义样式：月/周网格边框 `--border-subtle`，今天背景 `--bg-tertiary`，任务事件条背景 `--text-primary`，截止日事件使用 `--info` 20% 透明度，休假/全天事件使用 `--bg-tertiary` 边框。点击日期创建任务（预填日期）。

**负载视图**自研 **Workload Matrix 组件**：人员为行、日期为列，单元格颜色按负载强度映射语义色透明度（正常 `--success` 20%、警告 `--warning` 20%、超载 `--error` 20%），全天任务/休假置灰 `--text-muted` 15% 透明度。点击单元格弹出 shadcn/ui `Sheet` 展示当天任务。支持按部门/角色分组。

所有日期计算（刻度、工作日、冲突检测）由 **date-fns** 完成。交互反馈动画 150–300ms，保持全局规范。甘特图导出建议使用后端生成 PDF/PNG，或前端使用 html2canvas 捕获时间轴区域；日历与负载视图的数据可与甘特图共享同一 API，仅按视图维度聚合。
