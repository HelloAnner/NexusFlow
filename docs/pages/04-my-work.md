# 我的工作（My Work）

> 参考 Linear My Issues 和 Asana My Tasks，聚合当前用户所有负责或参与的工作，是个人工作的统一入口。

## 1. 页面目标

让用户在一个页面内查看和管理所有与自己相关的工作：负责的任务、参与的分工、待提交的成果、待处理的事项。

## 2. 整体布局

```
┌────────────────────────────────────────────────────────────────┐
│  My Work              [视图切换] [筛选] [分组] [排序] [ + New ] │
├────────────────────────────────────────────────────────────────┤
│  [ 进行中 ] [ 待确认 ] [ 待验收 ] [ 已归档 ] [ 全部 ]           │
├────────────────────────────────────────────────────────────────┤
│                                                                │
│  Main View（List / Table / Board / Timeline / Calendar）        │
│                                                                │
├────────────────────────────────────────────────────────────────┤
│  Context Panel（点击任务后滑出）                                │
└────────────────────────────────────────────────────────────────┘
```

### 2.1 任务上下文面板嵌入结构

My Work 不单独定义任务详情内容，点击任务后复用 `Task Detail` 面板，但页面级布局需要明确主视图与右侧面板的关系：

```text
+--------------------------------------------------------------------------------+
| My Work                                                                         |
+--------------------------------------------------------------------------------+
| View switch | Filter chips | Group | Sort | + New                               |
+--------------------------------------------------------------------------------+
| Status tabs: In Progress | Pending Confirm | Pending Review | At Risk | All     |
+---------------------------------------------+----------------------------------+
| Main View                                   | Task Detail Panel                |
|                                             |                                  |
| +-----------------------------------------+ | Header: close / pin / more       |
| | selected task row                        | | Title + status badges            |
| | status | project | due | progress | next | | Meta: owner / dates / project     |
| +-----------------------------------------+ | Tabs: Overview / Subtasks / ...   |
| | other task rows                          | | Scrollable tab content            |
| | virtualized list/table/board             | | Sticky state actions              |
| +-----------------------------------------+ |                                  |
+---------------------------------------------+----------------------------------+
```

要求：

- 面板打开后主视图宽度收缩，但筛选栏和 Tab 不消失。
- 主视图中的选中任务保持高亮，键盘 J/K 可切换任务并刷新右侧面板。
- 批量选择模式与详情面板互斥；进入批量选择后默认收起右侧面板。

## 3. 状态筛选 Tab

顶部提供快速状态筛选：

- 进行中（In Progress）
- 待确认（Pending Confirm）
- 待验收（Pending Review）
- 有风险（At Risk）
- 已归档（Archived）
- 全部（All）

## 4. 视图切换

参考 Notion 数据库视图：

| 视图 | 说明 |
|---|---|
| List | 默认高密度列表 |
| Table | 多字段表格，可配置列 |
| Board | 按状态分栏看板 |
| Timeline | 个人任务时间线 |
| Calendar | 按截止日期日历 |

视图可保存为个人偏好。

## 5. 列表字段

- 任务名称
- 状态
- 类型
- 优先级
- 所属项目
- 我的角色（负责人 / 成员 / 分工责任人）
- 起止时间
- 截止时间
- 进度
- 风险
- 下一步动作

## 6. 分组与排序

- 分组：按项目 / 状态 / 优先级 / 时间。
- 排序：截止时间、优先级、风险等级、创建时间。

## 7. 快捷操作

- hover 行显示：编辑、提交成果、更新进度、打开详情。
- 选中后顶部批量操作栏：批量导出、批量关注。

## 8. 空态

- 无任务时展示引导：创建第一个任务、查看项目。
- 当前筛选为空时提示清除筛选。

## 9. 与 Inbox 的关系

- Inbox 是需要响应的事项。
- My Work 是所有与我相关的工作，不区分是否待处理。

## 10. 使用场景细化

- 员工每天从“进行中”进入，优先更新进度、提交成果、查看即将到期任务。
- 负责人使用“待验收”和“有风险”快速处理成员提交、延期、冲突。
- 管理者更多使用按项目/人员分组，判断当前工作是否集中在少数人或少数项目。
- 从 Home、Inbox、Project、Team 进入 My Work 时，应自动带入来源筛选，并在顶部显示可清除的筛选标签。

## 11. 视图行为

- List 作为默认视图，保留最高信息密度；Table 用于批量维护字段；Board 用于状态推进；Timeline/Calendar 用于时间判断。
- 用户切换视图后，同一任务的选中状态、滚动位置和筛选条件应尽量保留。
- 分组折叠状态属于当前用户偏好，按视图分别保存。
- Board 拖拽变更状态时需要后端校验；如果状态流不允许，卡片回到原列并提示原因。
- Timeline 拖拽日期时要同步检查人员负载和冲突，风险只提示，不在前端直接强制阻止，最终以后端为准。

## 12. 空态与异常

- 当前筛选无结果时，优先展示“清除筛选”和“切换到全部”两个动作。
- 用户确实无任何工作时，展示“新建任务”“查看项目”“进入 Inbox”三个入口。
- 任务列表部分字段加载失败时，保留主列表并在字段位置显示短横线，不能让整页失败。
- 无权限查看某项目时，任务仍可显示名称和状态，但项目字段显示“无权限项目”，点击提示权限说明。
- 已归档任务默认不参与全局搜索结果排序，除非用户正在 Archived 或 All Tab。

## 建议组件

“我的工作”页面最复杂之处在于多视图切换（List / Table / Board / Timeline / Calendar），建议采用统一的数据层 + 可插拔视图层的架构。基础控件统一使用 **shadcn/ui**，并通过 CSS 变量覆盖保证五种视图在背景层级、文字层级、边框风格上完全一致：所有视图容器背景 `--bg-secondary`（#FFFFFF），列表/卡片分割线 `--border-subtle`（rgba(0,0,0,0.05)），hover 态统一使用 `--bg-tertiary`（#F5F4F2），禁止各视图自行定义背景色。文字层级统一：`--text-primary`（#1A1A1A）用于主内容，`--text-muted`（#7A7A7A）用于辅助信息。

顶部状态筛选 Tab 使用 shadcn/ui `Tabs`，包含“进行中 / 待确认 / 待验收 / 有风险 / 已归档 / 全部”，激活态文字 `--text-primary` 加底部 2px 指示条。视图切换、分组、排序、筛选均使用 shadcn/ui `DropdownMenu` 与 `Button`，主操作 + New 使用深色填充 `--text-primary`。行内操作按钮使用 `ghost` 变体，hover 背景 `--bg-tertiary`。视图偏好（当前视图、列配置、分组、排序）保存到用户设置或 localStorage。

**List 视图**建议自研高密度列表组件，行高 48–56px，字段包括任务名称、状态、类型、优先级、所属项目、我的角色、起止时间、截止时间、进度、风险、下一步动作。每行底部 1px `--border-subtle`。hover 行显示快捷操作（编辑、提交成果、更新进度、打开详情），按钮使用 shadcn/ui `Button` 的 `ghost` 变体。超过 100 条时使用 **TanStack Virtual** 虚拟滚动。状态 badge 使用 shadcn/ui `Badge`，优先级/风险仅使用语义色 `--error`/`--warning`/`--success`/`--info`，不新增颜色。

**Table 视图**使用 **TanStack Table** 实现，列宽可拖拽、列可隐藏/重排、排序可配置。表头背景 `--bg-tertiary`，行分割线 `--border-subtle`，选中行背景 `--bg-tertiary`。行选择使用 shadcn/ui `Checkbox`。列配置持久化保存到用户偏好，下次进入页面恢复。

**Board 视图**使用 **@dnd-kit** 实现拖拽排序与跨列改状态。每列是一个可垂直滚动的区域，列头背景 `--bg-tertiary`，列标题使用 `--text-primary`，任务数 `--text-muted`。卡片背景 `--bg-secondary`，阴影 `shadow-sm`，hover 抬升使用 `translate-y-[-2px] transition-transform duration-150`。拖拽占位使用 `--border-subtle` 虚线边框。状态列颜色仅通过列名左侧 4px 竖点区分，点为对应语义色，避免整列彩色背景。

**Timeline 视图**自研，复用 Schedule 页面的时间轴刻度逻辑（**date-fns** 生成），使用 SVG 绘制任务条与今天线。任务条填充使用 `--text-primary` 10% 透明度，有风险任务条边框 `--error`，今天线 `--info`。支持按天/周/月缩放。

**Calendar 视图**使用 **react-big-calendar** 但完全自定义样式：日历格子边框 `--border-subtle`，今天背景 `--bg-tertiary`，事件条背景 `--text-primary`，全天事件使用 `--bg-tertiary` 边框。所有日期计算由 **date-fns** 处理。点击日期可创建任务（预填日期）。

选中任务后右侧滑出上下文面板，使用 shadcn/ui `Sheet`（桌面 560px）或 **vaul**（移动端）。面板内展示任务详情，与 Task Detail 页面复用同一组件。空态使用自研 Empty State 组件，图标颜色 `--text-muted`，文案 `--text-muted`，引导按钮使用 shadcn/ui `Button` 的 `outline` 变体。批量选中后顶部出现悬浮批量操作栏，支持批量导出、批量关注。
