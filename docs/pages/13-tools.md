# 工具台（Tools）

> 参考 Notion Integrations 和 Wrike Apps，将常用工具和智能体工具作为应用市场式入口管理。

## 1. 页面目标

集中管理工作效率工具和智能体工具，支持按角色/组织配置可见性，并可将常用工具固定到首页。

## 2. 整体布局

```
┌────────────────────────────────────────────────────────────────┐
│  Tools                [搜索] [分类] [管理]                      │
├────────────────────────────────────────────────────────────────┤
│  我的常用工具                                                  │
├────────────────────────────────────────────────────────────────┤
│  分类列表                                                      │
├────────────────────────────────────────────────────────────────┤
│  工具卡片网格                                                  │
└────────────────────────────────────────────────────────────────┘
```

## 3. 我的常用工具

- 最近使用和已固定的工具。
- 支持取消固定。

## 4. 分类

- 常用工具（PDF 转换、翻译、表格处理等）
- 智能体工具（论文/专利辅助、知识库、问答助手等）
- 系统工具（邀请管理、审计日志等，管理员）

## 5. 工具卡片

- 图标、名称、描述。
- 打开方式：内嵌 / 新页签 / 弹窗。
- 固定到首页按钮。

## 6. 工具管理（管理员）

- 配置工具名称、图标、分类、入口地址、可见角色/组织、排序权重。

## 7. 工具使用细节

- 工具台默认先展示“我的常用工具”，再展示分类工具，避免用户每次从完整市场里寻找。
- 工具卡片的主操作为“打开”，固定到首页作为次操作，避免卡片操作区过重。
- 最近使用按用户维度记录，管理员调整工具可见性后，不可见工具自动从常用区隐藏。
- 内嵌工具打开后应保留工具台导航，用户可以返回工具列表；新页签工具需要明显标记。
- 工具搜索同时匹配名称、描述、分类和标签，结果为空时展示“申请新增工具”或联系管理员入口。

## 8. 管理与安全

- 工具入口地址必须校验协议和域名白名单，禁止配置 `javascript:` 等危险地址。
- 内嵌 iframe 工具需要配置 sandbox、allow 权限和可嵌入性检查；不可嵌入时自动退化为新页签打开。
- 管理员停用工具前需提示影响范围：固定人数、近 30 天使用次数、关联首页组件。
- 可见角色/组织由后端最终校验，前端只做筛选展示。
- 工具打开、配置变更、停用恢复需要写入审计日志。

## 9. 与其他页面联动

- 固定到首页后，Home 的工具区展示该工具；取消固定后立即同步。
- 系统工具如邀请管理、审计日志应优先跳转到 Admin/Settings 的原生页面，而不是重复做一个工具入口。
- 智能体工具如果产生任务、文件或报表，结果对象应能回写 NexusFlow，并在相关页面出现。
- 权限不足的工具不显示；从旧链接访问时展示无权限页，而不是跳回工具台造成困惑。

## 建议组件

工具台是“应用市场式”入口，视觉风格应偏向卡片化、轻量化，同时保持与全局一致的色调。建议基础控件统一使用 **shadcn/ui**，并通过 CSS 变量映射到 NexusFlow 色板：页面背景 `--bg-primary`（#FAF9F7），卡片 `--bg-secondary`（#FFFFFF），hover/选中态 `--bg-tertiary`（#F5F4F2），边框 `--border-subtle`（rgba(0,0,0,0.05)），文字 `--text-primary`（#1A1A1A）/`--text-muted`（#7A7A7A）。工具分类不引入彩色标签，统一使用中性 badge 或文字标签。

顶部搜索使用 shadcn/ui `Input`，占位符 `--text-muted`，聚焦边框 `--text-primary`，左侧搜索图标 `--text-muted`。分类切换使用 shadcn/ui `Tabs`：我的常用 / 常用工具 / 智能体工具 / 系统工具。管理按钮使用 shadcn/ui `Button` 的 `outline` 变体。

**我的常用工具**区域展示最近使用和已固定工具，使用自研水平滚动或 Grid 布局。每个工具项使用自研 **Tool Item 组件**：图标容器 48px×48px，背景 `--bg-tertiary`，圆角 12px，图标颜色 `--text-primary`；工具名称 `--text-primary font-medium`、描述 `--text-muted text-sm`；已固定状态使用 `--info` 小圆点提示。取消固定按钮 hover 显示，使用 shadcn/ui `Button` 的 `ghost` 变体。

**工具卡片网格**自研 **Tool Card 组件**，基于 shadcn/ui `Card`。每张卡片包含图标、名称、描述、打开方式标签（内嵌 / 新页签 / 弹窗）、固定到首页按钮。卡片背景 `--bg-secondary`，hover 背景 `--bg-tertiary`、阴影 `shadow-md`、抬升 `translate-y-[-2px]`，过渡 200ms。固定按钮使用 shadcn/ui `Switch` 或 `Button` 的 `ghost` 变体。卡片网格响应式，桌面 4 列、平板 3 列、移动 2 列。

**iframe 容器组件**用于内嵌外部工具：容器背景 `--bg-secondary`，顶部标题栏包含工具名称、关闭/刷新按钮（shadcn/ui `Button` 的 `ghost` 变体），加载态使用 shadcn/ui `Skeleton`。iframe 本身无边框，与容器融为一体。弹窗方式使用 shadcn/ui `Dialog`。打开方式可配置：内嵌（当前页 iframe）、新页签（`<a target="_blank">`）、弹窗（shadcn/ui `Dialog`）。

**工具管理（管理员）**使用 shadcn/ui `Dialog` 或 `Sheet` 打开配置表单，字段包括名称、图标、分类、入口地址、可见角色/组织、排序权重。表单校验使用 **react-hook-form + zod**。可见角色/组织选择使用 shadcn/ui `Select` 多选或自研 Tag Input，标签背景 `--bg-tertiary`。排序权重通过 **@dnd-kit** 拖拽调整工具列表顺序，拖拽占位背景 `--bg-tertiary`、边框 `--border-subtle` 虚线。

分类标签与管理端开关统一使用 shadcn/ui `Switch`，激活态背景 `--text-primary`，避免引入新的开关色。工具的可见性权限由后端校验，前端仅做展示过滤；常用工具固定状态保存到用户偏好。
