# 首页 / 今日指挥台（Home / Today Command Center）

> 首页从“今日待办列表”升级为“今日指挥台”：用户一进入系统，就能同时看见自己要处理的事、全局风险态势、未来 7 天节奏、项目健康和可立即执行的动作。视觉上要比当前版本更有能量，但仍保持 NexusFlow 的高密度、专业、可扫读。

## 0. 当前首页设计观察

当前 `nexusflow.pen` 与 `DashboardPage.tsx` 的首页已经具备清晰骨架：

- 顶部问候语、日期、`新建任务` 主操作。
- 四个统计数字：今日待处理、今日截止、本周冲突、待审批。
- 主内容两列：今日重点 + 我的日程。
- 设计稿中还包含我的任务、我的待办、风险提醒、快捷入口。

主要不足：

- 首屏信息层次偏平，指标、列表、日程之间缺少“态势感”。
- 内容元素不够丰富，项目、人员负载、资料缺口、工具入口、动态流没有形成完整闭环。
- 迷你甘特仍像占位区，缺少时间刻度、今天线、冲突标记、任务密度。
- 风险提醒和待办只是列表，没有升级路径、影响范围和推荐动作。
- 页面视觉偏安静，缺少代表 NexusFlow 调度能力的实时数据感、微动效和结构化图形。

## 1. 页面目标

首页只回答三个问题：

1. **我现在先处理什么？**
2. **今天和本周有什么风险正在逼近？**
3. **我能从这里一键进入哪些工作流？**

设计原则：

- 内容更多，但不堆叠。所有模块都要围绕“行动优先级”组织。
- 视觉更酷，但不营销化。不做大 Hero、不做装饰性渐变球，不牺牲业务密度。
- 首屏就是产品能力本身：调度、风险、项目、待办、工具、动态全部可达。
- 每个模块都必须有点击后续：打开任务详情、进入 Inbox、查看甘特、处理冲突、进入项目、打开工具。

## 2. 视觉方向

风格命名：**Calm Mission Control / 冷静指挥台**。

视觉语言：

- 大背景仍用 `--bg-primary #FAF9F7`，保持纸张感和全局一致性。
- 顶部新增一条 **Command Band**，使用 `--text-primary #1A1A1A` 做局部反色容器，形成指挥台第一视觉锚点。
- 内容区继续使用 `--bg-secondary #FFFFFF`、`--bg-tertiary #F5F4F2`、`--border-subtle`。
- 语义色只用于状态：`--info` 表示今日与进行中，`--warning` 表示逼近，`--error` 表示冲突和阻塞，`--success` 表示正常。
- 用细线、刻度、热力点、进度轨、扫描线、数字跳动来增强科技感，不用大面积彩色背景。
- 字体沿用当前系统字体，中文页面不引入新的展示字体，避免破坏企业后台稳定性。

动效方向：

- 页面首屏进入时，Command Band、指标、核心列表按 60ms 阶梯淡入。
- 今日线在迷你甘特中从上到下轻扫一次，之后静止。
- 统计数字支持 count-up，最大 400ms。
- 风险、冲突、逾期项使用极轻的左侧状态线脉冲，不做闪烁告警。
- Hover 时显示快捷操作按钮，列表行高度不变，避免布局跳动。

## 3. 桌面端整体布局

1440px 桌面首屏结构：

```text
+--------------------------------------------------------------------------------+
| Command Band                                                                  |
|  早上好，Anner / 今天 6月22日 周一 / 角色视角         + New Task  Cmd+K  Bell |
|  今日行动 24   今日截止 7   本周冲突 3   待审批 12   系统状态 正常            |
+--------------------------------------------------------------------------------+
| Signal Strip                                                                  |
|  [待办趋势 sparkline] [任务压力] [风险热度] [资料缺口] [工具推荐]              |
+-------------------------------+-----------------------------+------------------+
| Focus Command Stream          | 7-Day Mission Timeline      | Risk Radar       |
|  1. 逾期审批                  |  Mon Tue Wed Thu Fri ...    | 高风险 3         |
|  2. 今日成果确认              |  task bars + today line     | 阻塞成员 2       |
|  3. 冲突处理                  |  conflict markers           | 资料缺口 5       |
|  4. 进度填报                  |                             | recommended fix |
+-------------------------------+-----------------------------+------------------+
| Project Pulse                 | Inbox Triage                | Quick Dock       |
|  项目健康卡片 / 里程碑 / 风险 | 最近待办 + 批量入口         | 新建/排程/报表/AI |
+-------------------------------+-----------------------------+------------------+
| Activity Pulse / Team Load Mini Heatmap / Recently Opened                     |
+--------------------------------------------------------------------------------+
```

栅格建议：

- 页面内容最大宽度不强制居中，跟随全局 MainLayout 可用宽度。
- 主内容使用 12 栅格：左侧 Focus 5 栅格，中间 Timeline 4 栅格，右侧 Risk 3 栅格。
- 第二层 Project Pulse 5 栅格，Inbox Triage 4 栅格，Quick Dock 3 栅格。
- 卡片间距 12px 或 16px，避免首页变成营销式大卡片。
- 首屏高度不足时，Activity Pulse 放到第二屏，但顶部、Focus、Timeline、Risk 必须完整可见。

## 4. Command Band

Command Band 是首页最重要的视觉升级，放在页面顶部，替代普通欢迎区。

内容：

- 左侧：问候语、日期、当前角色视角。
- 中间：今日一句摘要，例如 `今天有 24 项待处理，3 个冲突需要在 18:00 前协调`。
- 右侧：`+ New Task`、命令面板入口、通知入口、角色视角切换。
- 底部嵌入一排 Live Chips：
  - 今日行动。
  - 今日截止。
  - 本周冲突。
  - 待审批。
  - 资料缺口。
  - 系统/同步状态。

交互：

- 每个 Live Chip 可点击，并跳转到对应页面且带筛选条件。
- 冲突、逾期、待审批数字如果大于 0，显示语义色小点和细线，不改变整个 Chip 背景。
- 角色视角可切换：个人 / 我负责的 / 部门 / 中心。切换后刷新首页所有组件。
- Command Band 右侧保留主操作，但不要把 `+ New Task` 做得过大，避免抢走风险处理优先级。

视觉规格：

- 高度 132px 左右。
- 背景使用 `--text-primary`，文字使用 `#FAFAFA`、次级文字使用 `rgba(250,250,250,0.62)`。
- 内部用 1px `rgba(255,255,255,0.10)` 分割线。
- Live Chip 高 34px，圆角 6px，背景 `rgba(255,255,255,0.06)`。
- 不使用彩色渐变；科技感来自反色、细线、数字和微动效。

## 5. Signal Strip

Command Band 下方新增一排更紧凑的信号卡，用于补足“多一点内容和元素”的首屏丰富度。

推荐 5 个信号：

| 信号 | 说明 | 点击去向 |
|---|---|---|
| 待办趋势 | 今日未处理、已处理、逾期趋势 sparkline | Inbox |
| 任务压力 | 今日截止、本周截止、延期任务 | My Work |
| 风险热度 | 冲突数、高风险数、升级数 | Conflicts |
| 资料缺口 | 需要补充的验收资料、缺失附件 | Files / Task Detail |
| 工具推荐 | 今日最可能需要的工具或智能体 | Tools |

组件要求：

- 每张信号卡高度 76px。
- 左侧是线图或小型状态轨，右侧是数字和标签。
- Sparkline 使用单色线，异常点用语义色圆点。
- Hover 时卡片轻微上浮 `translateY(-1px)`，右上角出现 `ChevronRight`。

## 6. Focus Command Stream

首页核心模块，替代“今日重点”普通列表。

排序规则：

1. 已逾期且需要当前用户处理。
2. 今天截止或今天必须确认。
3. 高风险冲突、待审批、待验收。
4. 本周即将到期的任务和里程碑。
5. 需要知晓但无需立刻处理的团队动态。

每条 Focus Item 展示：

- 排名编号：`01`、`02`、`03`，强化任务队列感。
- 状态线：左侧 3px，按 `info/warning/error/success`。
- 标题：任务或待办名称。
- 来源：项目 / 人员 / 审批流 / 资料。
- 需要你做什么：审批、填报、确认成果、处理冲突、补资料。
- SLA：`2h left`、`今天 18:00`、`已逾期 1 天`。
- 影响范围：影响项目、成员、里程碑或交付物。
- 快捷操作：处理、打开、稍后、转交。

交互：

- 点击行主体打开右侧任务/待办详情面板。
- 点击快捷操作在当前页直接完成轻操作；需要原因的操作弹出确认。
- `J/K` 移动焦点，`Enter` 打开，`E` 处理，`S` 稍后。
- 一条任务如果同时命中多个规则，只展示一次，并合并“需要你做什么”。
- 完成后该行折叠消失，顶部数字和 Signal Strip 同步刷新。

空态：

- 不展示大插画。
- 显示紧凑提示：`今天暂无必须处理事项`。
- 保留三个入口：查看我的工作、打开排程、新建任务。

## 7. 7-Day Mission Timeline

把“我的日程 / 甘特摘要”升级成真正可读的 7 天任务态势图。

内容：

- 顶部显示未来 7 天日期刻度。
- 中间显示任务条、里程碑点、今天线。
- 冲突日期用浅红底纹或红色刻度点。
- 截止日用蓝色刻度点。
- 管理者视角增加人员负载条：每人一行，显示未来 7 天占用。
- 员工视角显示个人任务条，按项目分组。

视觉规格：

- 使用 SVG 或 Canvas 绘制，不用普通 div 占位。
- 任务条高度 8px，圆角 999px。
- 今天线 1px，使用 `--text-primary`，顶部加小标签 `Today`。
- 冲突标记不大面积染红，只在对应日期列显示细小风险点。
- 鼠标悬停任务条显示浮层：任务名、项目、负责人、截止、风险。

交互：

- 点击任务条打开任务详情面板。
- 点击冲突点进入 Conflicts 并带日期筛选。
- 点击日期列进入 Schedule 的当天视图。
- 支持横向滚轮或拖拽查看未来 14 天，但默认只显示 7 天。

## 8. Risk Radar

右侧新增 Risk Radar，替代单纯“风险提醒”列表。

模块构成：

- 顶部风险总览：高 / 中 / 低风险数量。
- 中部 2x2 风险矩阵：
  - 即将逾期。
  - 资源冲突。
  - 资料缺口。
  - 等待审批。
- 下方推荐动作：
  - `处理最高风险冲突`。
  - `查看阻塞成员`。
  - `催办资料补齐`。
  - `打开全局甘特风险视图`。

交互：

- 点击矩阵块进入对应筛选。
- 推荐动作使用图标按钮 + 文本，不做大号按钮堆叠。
- 风险数为 0 时显示正常状态，但仍保留“查看风险中心”入口。

## 9. Project Pulse

新增项目态势模块，让首页不只关注个人待办，也能看到项目推进。

展示 3-5 个项目：

- 项目名称。
- 进度轨。
- 最近里程碑。
- 当前风险数。
- 本周新增任务 / 完成任务。
- 负责人与关键成员头像。

排序：

1. 有高风险或延期的项目。
2. 今天或本周有里程碑的项目。
3. 当前用户负责的项目。
4. 最近访问项目。

交互：

- 点击卡片打开项目详情面板或进入项目空间。
- 风险数字点击进入该项目的冲突筛选。
- 里程碑点击进入 Schedule 对应日期。

视觉：

- 项目卡不是大封面卡，保持 96-120px 高。
- 顶部可用一条细状态轨表现健康度。
- 允许项目色作为 4px 局部点缀，但不做彩色块背景。

## 10. Inbox Triage

保留待处理事项，但升级成可批量处理的 Triage 小队列。

内容：

- 最近 5-8 条待办。
- 每条显示类型图标、标题、来源对象、到期时间、处理按钮。
- 顶部显示筛选 Tab：全部 / 审批 / 填报 / 验收 / 冲突。
- 底部入口：查看全部、批量处理。

交互：

- 简单审批可在行内同意/拒绝。
- 复杂审批打开详情面板。
- 批量处理前需要确认影响范围。
- 已处理项从队列中移除，并刷新 Command Band 数字。

## 11. Quick Dock

新增首页快捷工作坞，承接工具、创建和高频动作。

推荐入口：

- 新建任务。
- 新建项目。
- 打开甘特。
- 打开报表。
- 上传资料。
- 处理冲突。
- 打开工具台。
- 智能体/效率工具推荐。

设计：

- 使用 2 列或 3 列紧凑图标按钮。
- 图标使用 Lucide，按钮要有 tooltip。
- 按钮固定尺寸，文字最多两行，避免中文溢出。
- 用户固定到首页的工具显示在 Quick Dock 底部，支持取消固定。

## 12. Activity Pulse

新增动态流，放在首屏下半区或第二屏顶部。

内容：

- 任务创建、状态变更、成果提交、审批、资料归档、冲突处理。
- 每条动态包含操作者、动作、对象、时间。
- 高风险动态左侧显示语义色点。

交互：

- 点击对象打开对应详情面板。
- 支持按项目、人员、风险筛选。
- 动态超过 50 条时使用虚拟滚动。

## 13. Team Load Mini Heatmap

管理者和负责人首页显示，员工首页可隐藏。

内容：

- 展示权限范围内 6-10 位成员未来 7 天负载。
- 正常、接近满载、超载使用 `success/warning/error` 语义色小块。
- 阻塞成员置顶。

交互：

- 点击成员进入 Team 的人员详情面板，并定位到负载或阻塞任务。
- 点击日期进入 Schedule 的人员视图。

## 14. 角色差异

| 角色 | 首页重点 | 额外模块 |
|---|---|---|
| 员工 | 今日分工、待提交成果、个人日程、资料缺口 | Quick Dock、个人工具推荐 |
| 任务负责人 | 负责项目进度、成员阻塞、待确认成果、任务风险 | Project Pulse、Team Load Mini Heatmap |
| 部门主任 | 部门负载、跨部门审批、部门风险、重点人员占用 | Risk Radar、部门负载排行 |
| 中心领导 | 中心重点项目、全局风险、跨部门协调、报表摘要 | Center Portfolio Pulse、升级协调入口 |
| SA | 默认进入 Admin Console，不进入普通 Home | 系统状态、待审核注册、邀请异常、权限审计风险 |
| 待审核用户 | 审核状态、资料补充、联系管理员 | 不展示业务数据 |

多角色用户：

- Command Band 中展示当前角色视角。
- 切换角色后只改变首页聚合范围，不改变用户账号身份。
- SA 如果同时有业务身份，可通过“返回业务前台”进入普通 Home。

## 15. 状态与边界

加载态：

- Command Band 先渲染骨架。
- Focus、Timeline、Risk 三个核心模块并行加载。
- Project Pulse、Activity Pulse 可延后加载，不阻塞首屏。

空态：

- 无 Focus 事项时，显示紧凑空态和三个下一步入口。
- 无项目时不展示空项目网格，改为显示“参与项目后会在这里出现”。
- 无风险时显示正常状态，同时保留风险中心入口。

错误态：

- 某个模块接口失败，只在该模块内显示 `暂不可用`，不能整页失败。
- 冲突、审批、资料缺口等风险数字接口失败时显示 `--`，不能显示 0。
- 快捷操作失败时回滚列表状态，并用 Sonner 提示失败原因。

权限态：

- 无权限模块直接隐藏，不显示“无权限”大面积提示。
- 用户有查看无操作权限时，保留列表和详情入口，隐藏处理按钮。
- 高风险操作必须二次确认，并记录审计日志。

## 16. 数据联动

首页动作完成后需要同步刷新：

- Command Band Live Chips。
- Signal Strip。
- Focus Command Stream。
- Inbox Triage。
- Risk Radar。
- 相关对象详情面板。

联动规则：

- 处理待办后，从 Inbox Triage 和 Focus 中移除。
- 处理冲突后，刷新 Risk Radar、Timeline 冲突点、Project Pulse 风险数。
- 提交成果或补资料后，刷新资料缺口信号和任务详情资料完整率。
- 新建任务后，刷新 Timeline、Project Pulse、My Work 和 Schedule。

## 17. 移动端退化

移动端不做横向压缩，改为纵向优先级：

1. Command Band 简化为问候语 + 今日行动数字 + 新建按钮。
2. Signal Strip 横向滚动。
3. Focus Command Stream。
4. Risk Radar。
5. 7-Day Timeline 横向滚动。
6. Inbox Triage。
7. Project Pulse。
8. Quick Dock。
9. Activity Pulse。

移动端详情统一打开全屏抽屉，保持 Header 和底部操作固定。

## 18. 建议组件与实现备注

- 基础 UI：继续使用当前自研 `Panel`、`StatCard`、`Button`，后续可逐步映射 shadcn/ui。
- 图标：使用 Lucide，不手写功能图标。
- 日期与排序：使用 `date-fns`。
- Timeline：自研 SVG，后续可复用 Schedule 的日期刻度逻辑。
- Sparkline：可使用轻量 SVG 自绘，避免为小图引入重图表依赖。
- Activity 和长列表：超过 50 条时使用 TanStack Virtual。
- Toast：使用 Sonner 或当前项目已有提示机制。
- 右侧详情：沿用全局 Context Panel / Sheet 规则。

## 19. 下一步落地顺序

1. 在 `nexusflow.pen` 中把首页画面从当前两列卡片升级为 Command Band + 三栏核心区 + 第二层模块。
2. 先实现静态布局和真实接口已有数据：dashboard、tasks、todos、conflicts、activities。
3. 再补 Project Pulse、资料缺口、Team Load 等需要新增聚合接口的数据。
4. 最后增加 Timeline SVG、Sparkline、键盘焦点和批量处理。
