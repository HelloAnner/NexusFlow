# 首页 / 今日工作台（Home / Today）

> 参考 Linear 的 Today 视图和 Monday.com 的 Dashboard，首页只回答一个问题："我现在应该关注什么？"

## 1. 页面目标

用户登录后看到的第一个页面，根据角色聚合今日最需要处理的工作，避免信息平铺堆砌。

## 2. 整体布局

```
┌────────────────────────────────────────────────────────────────┐
│  欢迎语 + 今日摘要                  [ + New Task ]              │
├────────────────────────────────────────────────────────────────┤
│  ┌─────────────────────┐  ┌────────────────────────────────┐  │
│  │                     │  │                                │  │
│  │  今日重点            │  │  我的日程 / 甘特摘要           │  │
│  │  (3-5 项最高优先级)  │  │                                │  │
│  │                     │  │                                │  │
│  └─────────────────────┘  └────────────────────────────────┘  │
├────────────────────────────────────────────────────────────────┤
│  待处理事项（Inbox 预览）          [ 查看全部 → ]              │
├────────────────────────────────────────────────────────────────┤
│  我的项目卡片 / 团队动态 / 风险提醒                              │
└────────────────────────────────────────────────────────────────┘
```

## 3. 顶部欢迎区

- **问候语**：`早上好，{姓名}`。
- **今日摘要数字**：
  - 今日待处理：N
  - 今日截止：N
  - 本周冲突：N
  - 待审批：N（管理者）
- **主操作**：+ New Task。

## 4. 今日重点（Focus）

> 参考 Linear 的 Focus 模式，只展示 3-5 件最重要的事。

- 根据截止时间、风险等级、审批状态智能排序。
- 每条展示：
  - 任务名称
  - 所属项目
  - 截止时间
  - 需要你做什么（审批/填报/验收/处理冲突）
  - 快捷操作按钮
- 支持点击整行打开任务详情面板。

## 5. 我的日程 / 甘特摘要

- 迷你时间线，展示未来 7 天。
- 员工：个人任务分布。
- 管理者：部门人员负载概览。
- 冲突日期红色标记，截止日蓝色标记。
- 点击任务条打开详情面板。

## 6. 待处理事项（Inbox Preview）

- 展示最近 5-8 条需要当前用户处理的事项。
- 类型：审批、填报、验收、冲突、待确认。
- 底部“查看全部”跳转到 Inbox。

## 7. 我的项目卡片

- 以卡片网格展示用户参与的 4-6 个活跃项目。
- 每张卡片：项目名称、进度条、最近更新、风险数。
- 点击卡片进入项目详情或项目空间。

## 8. 团队动态

- 权限范围内最近发生的动态：任务创建、变更、成果提交、审批、归档。
- 每条动态可点击跳转到对应对象。

## 9. 角色差异

| 角色 | 首页重点 |
|---|---|
| 员工 | 今日任务、待提交成果、个人日程 |
| 负责人 | 负责项目进度、成员阻塞、待验收成果 |
| 部门主任 | 部门负载、跨部门审批、部门风险 |
| 中心领导 | 中心重点项目、全局风险、跨部门协调 |
| SA | 系统状态、待审核注册、邀请异常 |

## 10. SA 首页

SA 超级管理员登录后默认进入 Admin Console，不展示普通 Home。Admin Console 首页：

- 系统健康状态卡片。
- 待审核注册。
- 邀请链接异常。
- 配置告警。
- 权限审计风险。

## 11. 信息优先级规则

首页不应展示所有信息，而应按“需要当前用户今天行动”的程度排序：

1. 已逾期且需要当前用户处理的事项。
2. 今天截止或今天必须确认的事项。
3. 高风险冲突、待审批、待验收。
4. 本周即将到期的任务和里程碑。
5. 只需知晓的团队动态。

同一个任务如果同时命中多个规则，只展示一次，并在“需要你做什么”字段中合并动作，例如“更新进度，并处理 1 个冲突”。

## 12. 交互细节

- 今日重点最多展示 5 条，超过后显示“还有 N 项需要处理”，点击进入 Inbox 或 My Work 的对应筛选。
- 摘要数字点击后进入对应页面并带上筛选条件，例如“本周冲突”进入 Conflicts 的本周高风险筛选。
- 迷你时间线默认展示今天到未来 7 天，周末和节假日降低视觉权重，但不隐藏。
- 团队动态默认只展示权限范围内最近 10 条，动态内容要突出对象与动作，不写长句。
- 首页刷新不应清空用户当前打开的右侧面板；如果任务处理完成，则面板显示已完成状态并允许返回列表。
- 管理者首页中的“成员阻塞”点击后进入 Team 的人员详情面板，直接定位到负载或阻塞任务。

## 13. 状态与边界

- 加载态按区块显示骨架，Focus 加载完成后可先渲染，不等待团队动态和项目卡片。
- 无今日重点时展示“今天暂无必须处理事项”，同时保留“查看我的工作”和“新建任务”入口。
- 当前用户没有项目时，不展示空项目网格，改为展示“参与项目后会在这里出现”的轻提示。
- 冲突数据接口失败时，摘要卡片显示“暂不可用”，不能显示 0。
- SA 角色不进入普通 Home；如果 SA 同时具有业务身份，可通过“返回业务前台”进入普通 Home。
- 首页所有快捷操作执行后，需要同步刷新对应摘要数字，避免处理完成后数字仍停留在旧值。

## 建议组件

首页 / 今日工作台需要在信息密度与视觉呼吸感之间取得平衡，建议以 **shadcn/ui** 的 `Card` 作为各区块容器，并通过 CSS 变量覆盖保持色调统一：所有卡片背景固定为 `--bg-secondary`（#FFFFFF），边框使用 `--border-subtle`（rgba(0,0,0,0.05)），阴影保持 shadcn 默认 `shadow-sm` 或完全去除，避免破坏 bg-primary #FAF9F7 营造的纸张感。标题使用 `text-primary`，辅助说明使用 `text-muted`，数字指标使用 `text-primary` 加粗，不额外引入强调色。

顶部欢迎区的问候语、今日摘要数字、+ New Task 主操作使用 shadcn/ui `Button`（主按钮填充 `--text-primary`）、`Badge`、`Skeleton`（加载态）。问候语使用 `text-xl font-semibold text-primary`，今日摘要数字卡片使用自研“指标微卡片”组件，内部 CSS Grid 等分四列，每张卡片背景 `--bg-secondary`，数字 `text-2xl font-bold text-primary`，标签 `text-sm text-muted`，hover 时背景从 `--bg-secondary` 过渡到 `--bg-tertiary`（#F5F4F2），过渡时长 150ms，不使用彩色 hover。四个数字分别对应今日待处理、今日截止、本周冲突、待审批，冲突数使用 `--error` 数字色，其余保持 `--text-primary`。

“今日重点（Focus）”是本页核心，建议自研 **Focus List 组件**。每行固定 56px 高度，背景 `--bg-secondary`，底部 1px `--border-subtle`；未读或高优先级项左侧使用 3px `--info`（#3B82F6）竖线，有风险项使用 `--error`（#EF4444）竖线。每行展示任务名称 `--text-primary`、所属项目 `--text-muted`、截止时间 `--text-muted`、需要你做什么（审批/填报/验收/处理冲突）`--text-muted`、快捷操作按钮。hover 行背景变为 `--bg-tertiary`，同时右侧渐显快捷操作按钮组（使用 shadcn/ui `Button` 的 `ghost` 变体）。点击整行打开任务详情面板。排序逻辑依赖 **date-fns** 计算截止时间、冲突等级、审批紧急度，最多展示 3–5 项。列表超过 10 条时使用 **TanStack Virtual** 虚拟滚动。

“我的日程 / 甘特摘要”建议自研 **Mini Timeline 组件**，宽度自适应，展示未来 7 天。使用 SVG 绘制时间轴、任务条、今天标记线与冲突/截止标记，任务条填充色统一使用 `--text-primary` 的 10% 透明度，冲突日期使用 `--error` 的 20% 透明度背景，截止日使用 `--info` 的 20% 透明度背景。员工视角展示个人任务分布，管理者视角展示部门人员负载概览。通过复用 Schedule 页面的日期刻度生成逻辑（**date-fns**）保证一致性。点击任务条打开任务详情面板。

“待处理事项（Inbox Preview）”展示最近 5–8 条需要当前用户处理的事项，类型包括审批、填报、验收、冲突、待确认。使用自研紧凑列表，行高 48px，左侧类型图标使用 Lucide 线图标 `--text-muted`，标题 `--text-primary`，来源对象/时间 `--text-muted`，hover 背景 `--bg-tertiary`。底部“查看全部 →”使用 shadcn/ui `Button` 的 `ghost` 变体。

“我的项目卡片”使用自研卡片网格或 shadcn/ui `Card` 组合：封面区域允许项目色但仅作为局部点缀，且项目色需从固定中性 palette 中选取，避免与系统语义色混淆；进度条使用 shadcn/ui `Progress` 并覆盖 `--progress-indicator` 为 `--text-primary`（#1A1A1A），保持中性；最近更新 `--text-muted`；风险数 badge 使用 `--error`，不引入新红。卡片网格响应式，桌面 3 列、平板 2 列、移动 1 列，hover 抬升 `translate-y-[-2px]` 阴影 `shadow-sm`。

“团队动态”是时间线型列表，建议自研 **Activity Feed 组件**，头像使用 shadcn/ui `Avatar`，动态文本使用 `text-sm text-muted`，对象链接使用 `text-primary hover:underline`。每条动态可点击跳转到对应对象。如果动态超过 50 条，使用 **TanStack Virtual**。

预置的“今日摘要趋势小图”可使用 **recharts** 或 **tremor** 绘制，但需关闭默认彩色主题，将折线/面积颜色固定为 `--text-primary`，辅助网格线使用 `--border-subtle`，工具提示背景使用 `--bg-secondary`。SA 首页的系统健康卡片、待审核注册、邀请异常、配置告警、权限审计风险使用同样的组件策略，异常项使用 `--error` 强调。所有区块加载态使用 shadcn/ui `Skeleton`，避免整页白屏；首页数据建议并行请求，首屏渲染目标 500ms 内。
