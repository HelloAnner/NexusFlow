# 资料库（Files）

> 参考 Notion Files 和 Wrike Files View，将资料作为独立对象管理，支持按项目/任务/类型组织、版本控制和快速预览。

## 1. 页面目标

统一管理系统中的所有文件与交付物，支持快速上传、检索、预览、版本管理和权限控制。

## 2. 整体布局

```
┌────────────────────────────────────────────────────────────────┐
│  Files                [视图切换] [筛选] [上传] [新建文件夹]     │
├───────────────────────┬────────────────────────────────────────┤
│                       │                                        │
│  左侧目录树           │  文件列表 / 网格                       │
│                       │                                        │
└───────────────────────┴────────────────────────────────────────┘
```

## 3. 目录树

- 全部资料
- 按项目
- 按任务
- 按类型
- 我的上传
- 回收站（管理员）

## 4. 视图切换

- List：文件列表，含版本/关联任务/上传人。
- Grid：缩略图网格。
- Timeline：按上传时间线。

## 5. 文件列表字段

- 文件名 / 缩略图
- 类型
- 关联任务 / 项目
- 上传人
- 版本号
- 上传时间
- 可见范围
- 成果标记
- 操作

## 6. 上传与版本

- 拖拽上传。
- 上传时选择类型、关联任务/分工、可见范围。
- 同名文件提示更新版本。
- 版本历史可回滚。

## 7. 文件详情

- 右侧滑出或弹窗。
- 文件预览。
- 版本历史。
- 关联任务/项目跳转。
- 评论与审批记录（如为成果）。

## 8. 文件对象规则

- 资料库中的文件不是简单附件，必须作为可被权限控制、版本追踪、关联任务/项目的独立对象。
- 文件可同时关联项目、任务、分工；列表中优先展示最具体关联，即分工 > 任务 > 项目。
- 成果类文件需要展示成果状态：待提交、待确认、已确认、被退回。
- 同名文件在同一关联对象下默认提示“上传新版本”，跨项目/跨任务同名则允许并存。
- 删除文件默认进入回收站，保留 30 天；被任务成果引用的文件不允许直接硬删除。

## 9. 上传与预览细节

- 拖拽多个文件时，先进入上传确认队列，允许逐个补充类型、关联对象和可见范围。
- 上传大文件时显示队列级进度和单文件进度；用户关闭页面前需要提示未完成上传。
- 预览不可用时展示文件元信息、下载按钮和失败原因，不让用户面对空白预览区。
- 上传新版本后，文件详情默认切到版本历史，并突出最新版本。
- 回滚版本需要二次确认，并记录回滚前版本号和操作原因。

## 10. 权限与安全

- 文件可见范围取决于项目/任务权限与文件自身可见范围的交集，前端不得只靠目录树过滤。
- 用户无权访问文件时，关联任务中仍可展示“受限文件”占位，但不能展示文件名和预览。
- 外部预览 iframe 必须 sandbox；不支持预览的类型只提供下载。
- 管理员查看回收站时可以恢复文件；普通用户只可恢复自己删除且仍有权限的文件。
- 文件下载、预览、版本回滚都应写入 Activity 或审计日志，便于追溯资料流转。

## 建议组件

资料库页面需要同时支持树形导航、文件列表/网格/时间线、上传、预览与版本管理，建议以 **shadcn/ui** 作为基础控件层，并通过 CSS 变量统一色调：页面背景 `--bg-primary`（#FAF9F7），左侧目录树与右侧面板 `--bg-secondary`（#FFFFFF），hover/选中态 `--bg-tertiary`（#F5F4F2），边框 `--border-subtle`（rgba(0,0,0,0.05)），文字 `--text-primary`（#1A1A1A）/`--text-muted`（#7A7A7A）。资料类型 badge 仅使用语义色：`--info`（文档）、`--success`（已确认成果）、`--warning`（待确认）、`--error`（被退回），不引入文件类型彩虹图标。

左侧目录树建议自研或使用 shadcn/ui `Tree`（如果项目已集成），包含全部资料、按项目、按任务、按类型、我的上传、回收站。树节点高度 36px，选中背景 `--bg-tertiary`，hover 背景 `--bg-tertiary`/50%，展开图标 `--text-muted`。回收站节点使用 `--error` 文字色作为唯一红色提示。目录树宽度 240px，可折叠，分割线 `--border-subtle`。

右侧视图切换（List / Grid / Timeline）使用 shadcn/ui `Tabs`。List 视图使用 **TanStack Table**，列包括文件名/缩略图、类型、关联任务/项目、上传人、版本号、上传时间、可见范围、成果标记、操作。表头背景 `--bg-tertiary`，行分割线 `--border-subtle`，hover 行背景 `--bg-tertiary`。文件名可点击，hover 下划线。Grid 视图自研卡片网格，缩略图区高度 120px，无缩略图时显示文件类型图标（`--text-muted`），文件名 `--text-primary`，元信息 `--text-muted`。卡片网格响应式，hover 阴影 `shadow-sm`。Timeline 视图自研时间线，按上传时间分组，节点 `--text-muted`，分组标题 `--text-primary`。

上传功能使用 **react-dropzone** 实现拖拽上传，拖拽区域使用虚线边框 `--border-subtle`，拖拽进入时背景 `--bg-tertiary`、边框 `--text-primary`。上传时选择类型、关联任务/分工、可见范围，使用 shadcn/ui `Select` 与 `Dialog`。上传进度使用 shadcn/ui `Progress`，指示器 `--text-primary`。同名文件提示更新版本。上传成功/失败/同名更新版本提示使用 **Sonner**，语义色反馈。

文件详情使用 shadcn/ui `Sheet`（桌面 480px）或 **vaul**（移动端），内部包含文件预览、版本历史、关联跳转、评论与审批记录。文件预览自研 **File Preview 组件**：PDF 使用浏览器原生 `<iframe>` 或 PDF.js；图片使用 `<img>`；Office 文件优先调用 OnlyOffice/微软在线预览，不可用时提供下载按钮。预览区背景 `--bg-secondary`，加载态使用 shadcn/ui `Skeleton`。

版本管理自研 **Version List 组件**，每行展示版本号、上传人、时间、变更说明，当前版本高亮背景 `--bg-tertiary`，回滚操作使用 shadcn/ui `Button` 的 `outline` 变体并二次确认。右键菜单使用 shadcn/ui `ContextMenu`，菜单背景 `--bg-secondary`，分割线 `--border-subtle`。大文件上传建议分片并显示进度；文件预览需做 XSS 防护，iframe 使用 sandbox 属性；回收站保留 30 天后自动清理。
