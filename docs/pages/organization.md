# 组织管理页（Organization）

## 1. 页面定位

组织管理页用于维护企业组织架构、查看部门人员，并支持在线调整人员归属与状态。页面以**部门树**为核心，右侧展示选中部门的人员列表；部门详情与人员详情均通过右侧抽屉展开，避免主界面信息堆叠。

## 2. 整体布局

页面采用“树 + 列表 + 抽屉”三段式布局：

```
┌─────────────────────────────────────────────────────────────┐
│  组织        组织树、部门治理、人员归属和审批关系              │
├─────────────────────────────────────────────────────────────┤
│  [组织总数] [启用组织] [组织层级] [当前成员]                   │
├──────────────────────────┬──────────────────────────────────┤
│                          │                                  │
│   部门树                  │   部门人员列表                    │
│   （重点展示、可多级展开） │   （点击行查看/编辑详情）          │
│                          │                                  │
├──────────────────────────┴──────────────────────────────────┤
│  ← 右侧滑出抽屉：部门详情 / 人员详情                            │
└─────────────────────────────────────────────────────────────┘
```

### 2.1 顶部指标

保留 4 个核心指标卡片：

| 指标 | 说明 |
| --- | --- |
| 组织总数 | 当前可见组织总数 |
| 启用组织 | 状态为启用的组织数量 |
| 组织层级 | 组织树最大深度 |
| 当前成员 | 选中部门下的人员数量 |

### 2.2 主区域

主区域为左右两栏：

- **左侧：部门树**（占比约 40%，最小宽度 360px）
  - 标题栏显示“部门树”+ 操作按钮。
  - 树节点支持无限层级展开/收起。
  - 默认展开到二级单位或中心层级（首次加载）。
  - 选中节点高亮。
- **右侧：人员列表**（占比剩余空间）
  - 标题栏显示“部门人员”+ 当前人数 + 新建人员按钮。
  - 表格展示人员关键信息。
  - 点击行打开人员详情抽屉。

### 2.3 右侧抽屉

从右侧滑出的抽屉分为两类：

1. **部门详情抽屉**
   - 默认不展开，点击部门节点的“详情”操作后展开。
   - 仅展示部门基本信息与关键管理字段。
   - 支持保存、删除、新建下级。
2. **人员详情抽屉**
   - 点击人员列表行后展开。
   - 展示并允许编辑人员部署信息：主组织、工作状态、标准工时、账号状态、角色等。

## 3. 部门树

### 3.1 树节点信息

每个树节点只展示最核心的信息，避免拥挤：

- 展开/收起箭头（仅在有子节点时显示）。
- 组织类型图标或类型标签（公司/中心/部门/工作室）。
- 组织名称。
- 状态指示（启用/停用，通过名称颜色或状态点表达）。

节点不再显示长串编码、负责人 ID 列表等字段；这些信息放入抽屉编辑。

### 3.2 节点交互

| 操作 | 行为 |
| --- | --- |
| 点击节点主体 | 选中该部门，右侧人员列表刷新 |
| 点击展开箭头 | 展开/收起子部门 |
| 悬停节点 | 显示操作按钮组（详情、新建下级、删除） |
| 点击“详情” | 右侧滑出部门详情抽屉 |
| 点击“新建下级” | 右侧滑出新建部门抽屉，并预填上级组织 |
| 点击“删除” | 在节点下方展开删除确认面板，显示引用检查与原因输入，确认后删除 |

### 3.3 删除确认面板

删除确认面板在部门树节点下方内联展开，包含以下内容：

- **风险提示**：若该部门仍有子部门、人员、项目或任务关联，面板直接以红色提示具体数量，例如“该部门仍有 2 名人员、1 个项目、1 个任务关联，无法删除”。
- **删除原因**：输入框，选填，用于审计日志。
- **操作按钮**：取消、确认删除。
- **提交失败**：若后端校验不通过，在面板内以红色错误提示展示具体原因，不依赖顶部全局消息条。

确认删除仅在无活跃引用时允许执行。

### 3.4 新建/编辑部门入口

将原来散落在树节点上的三点菜单改为更清晰的入口：

- 树标题栏保留“新建根组织”按钮。
- 悬停树节点时显示“详情”“+”“删除”图标按钮。
- 点击“删除”后在节点下方展开确认框，可填写删除原因后确认。
- 部门详情抽屉内仍保留“新建下级”和“删除”按钮。

## 4. 部门人员列表

### 4.1 列表字段

| 字段 | 说明 |
| --- | --- |
| 人员 | 头像 + 姓名 + 人员 ID |
| 工号 | employee_no |
| 组织关系 | 主组织 / 兼属 |
| 标准工时 | daily_standard_hours |
| 工作状态 | 在岗/出差/休假/离岗 |
| 账号状态 | 启用/待审核/禁用/锁定 |
| 操作 | 详情按钮 |

### 4.2 列表交互

- 点击整行打开人员详情抽屉。
- 标题栏“新建人员”按钮可在当前选中部门下创建人员。
- 空状态提示“该部门下暂无人员”。

## 5. 右侧抽屉

抽屉从右侧滑出，宽度为 560px（在窄屏下占 96% 视口），避免内容拥挤。

## 6. 部门详情抽屉

### 6.1 展示字段

只保留必要信息，分为两组，采用两列栅格布局：

**基本信息**

- 组织名称
- 组织编码
- 组织类型
- 启用状态

**管理信息**

- 负责人（多选 ID 文本）
- 默认审批人（多选 ID 文本）
- 业务负责人
- 办公地点
- 成本中心
- 组织说明（多行文本）

### 6.2 操作

- 保存字段：保存编辑后的部门信息。
- 新建下级：在该部门下创建子部门。
- 删除：删除当前部门（需校验无引用）。
- 移动到上级组织：下拉选择新上级并确认移动。

## 7. 人员详情抽屉

### 7.1 展示与编辑字段

| 字段 | 可编辑 |
| --- | --- |
| 姓名 | 是 |
| 主组织 | 是（下拉选择） |
| 工号 | 是 |
| 工作状态 | 是（在岗/出差/休假/离岗） |
| 每日标准工时 | 是 |
| 账号状态 | 是（启用/禁用/锁定） |
| 系统角色 | 是（标签勾选） |
| 审计备注 | 是（变更组织/角色/状态时必填） |

### 7.2 系统角色选择器

系统角色不使用原生多选下拉，而是展示为可勾选的标签列表：

- 每个角色显示为一个标签按钮。
- 已选角色高亮并带勾选图标。
- 点击即可切换选中/取消选中。
- 支持多选，布局自动换行。

### 7.3 操作

- 保存：调用 PATCH /api/users/{id} 更新人员。
- 取消/关闭：关闭抽屉，不保存。

## 8. 新建部门抽屉

从右侧滑出，字段精简为：

- 组织名称（必填）
- 组织编码（必填）
- 上级组织（下拉，创建时锁定为当前选中或根组织）
- 组织类型（下拉）
- 负责人 ID
- 默认审批人 ID
- 组织说明

## 9. 新建人员抽屉

从右侧滑出，字段精简为：

- 姓名（必填）
- 登录账号
- 工号
- 主组织（下拉，默认当前选中部门）
- 默认角色（标签勾选）

## 10. 异常与空状态

| 场景 | 处理 |
| --- | --- |
| 无组织 | 左侧树显示空状态，提示新建根组织 |
| 未选中部门 | 右侧人员列表显示空状态 |
| 部门下无人员 | 人员列表显示空状态 |
| 保存失败 | 顶部或抽屉内显示错误提示 |
| 删除失败（存在引用） | 删除确认面板内以红色提示具体引用数量，顶部消息条同时以错误样式展示 |

## 11. 权限与审计

- 拥有 `person.manage` 权限的用户才能编辑人员和部门字段。
- 修改组织、角色、账号状态等敏感字段时，必须填写审计备注并写入审计日志。
- 普通用户只能查看，不能编辑。

## 12. 路由

页面路由保持 `/orgs`。
