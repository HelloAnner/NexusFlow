# 项目中心（Projects）

> 参考 Linear Projects 和 Wrike Spaces，把项目作为独立的工作空间来管理，而不是任务的一个字段。

## 1. 页面目标

提供项目级别的统一视图，让用户可以按项目查看任务、成员、资料、进度、风险，并在项目内部完成协作。

## 2. 整体布局

```
┌────────────────────────────────────────────────────────────────┐
│  Projects             [视图切换] [筛选] [排序] [ + New Project ]│
├────────────────────────────────────────────────────────────────┤
│                                                                │
│  Main View（List / Table / Board / Gallery / Timeline）         │
│                                                                │
├────────────────────────────────────────────────────────────────┤
│  Context Panel（点击项目后滑出）                                │
└────────────────────────────────────────────────────────────────┘
```

### 2.1 项目预览面板嵌入结构

项目中心点击项目时，右侧预览面板复用 `Project Detail`，但主列表仍保留当前视图状态：

```text
+--------------------------------------------------------------------------------+
| Projects                                                                        |
+--------------------------------------------------------------------------------+
| View switch | Filter | Sort | + New Project                                     |
+---------------------------------------------+----------------------------------+
| Project List / Table / Board / Gallery      | Project Detail Panel             |
|                                             |                                  |
| +-----------------------------------------+ | Header: close / enter / more     |
| | selected project card or row             | | Project title + status badges    |
| | owner | org | progress | risks           | | Meta: owner / org / dates        |
| +-----------------------------------------+ | Progress + key metrics            |
| | other projects                           | | Tabs: Overview / Members / ...   |
| | current view remains scrollable          | | Scrollable tab content            |
| +-----------------------------------------+ | Sticky actions                    |
+---------------------------------------------+----------------------------------+
```

## 3. 项目列表视图

### 3.1 视图切换

- **List**：项目列表，高密度行。
- **Table**：多字段对比表格。
- **Board**：按状态分栏看板。
- **Gallery**：卡片网格，展示项目封面/进度/负责人。
- **Timeline**：项目周期时间线。

### 3.2 列表字段

- 项目名称 / 编号
- 状态
- 类型
- 负责人
- 归属组织
- 起止时间
- 进度
- 任务数
- 风险数
- 成员数

### 3.3 筛选与排序

- 筛选：状态、类型、级别、归属组织、负责人、风险等级。
- 排序：进度、截止时间、创建时间、风险等级。

## 4. 项目内部空间

点击项目后，默认进入项目内部空间：

```
┌────────────────────────────────────────────────────────────────┐
│  项目名称 │ 状态 │ 负责人 │ [ 返回项目列表 ]                  │
├────────────────────────────────────────────────────────────────┤
│  [ Tasks ] [ Members ] [ Files ] [ Timeline ] [ Risks ] [ Settings ]│
├────────────────────────────────────────────────────────────────┤
│  当前 Tab 内容                                                 │
└────────────────────────────────────────────────────────────────┘
```

### 4.1 Tasks Tab

- 与 My Work / Tasks 页面复用相同视图组件。
- 数据范围自动过滤为当前项目。
- 新建任务自动关联当前项目。

### 4.2 Members Tab

- 成员列表：姓名、项目身份、负责工作、加入/退出时间。
- 操作：添加成员、修改身份、移除成员。

### 4.3 Files Tab

- 项目级资料目录。
- 可勾选“包含任务资料”。

### 4.4 Timeline Tab

- 项目内任务与里程碑的甘特时间线。

### 4.5 Risks Tab

- 项目内所有风险与冲突汇总。

### 4.6 Settings Tab

- 项目基本信息、可见性、隐藏项目授权、归档/取消。

## 5. 项目列表细节

- 项目中心默认展示当前用户可见的活跃项目，已归档项目通过筛选进入，不混在默认列表中。
- 项目名称点击打开项目空间，行内“快速预览”或右侧箭头打开 Project Detail 面板，避免误触导致路由切换。
- 项目进度应区分“任务完成率”和“时间进度”，默认展示任务完成率，hover 或详情面板中补充时间进度。
- 风险数点击后进入项目空间 Risks Tab，并带上未解决筛选。
- 负责人头像点击打开人员详情面板，项目行本身不跳转。
- Gallery 视图只用于概览，不承载复杂批量操作；需要批量维护时引导切换 Table。

## 6. 项目空间细节

- 项目空间顶部固定展示项目状态、负责人、可见性、风险数，用户切换 Tab 时不消失。
- Tasks Tab 的筛选、分组、列配置与 My Work 独立保存，避免个人工作视图被项目视图污染。
- Members Tab 中移除成员时，如果该成员仍负责未完成任务，必须先提示影响范围并要求重新分配或确认保留历史负责人。
- Files Tab 的“包含任务资料”默认关闭，避免项目资料库被任务过程文件淹没。
- Timeline Tab 中里程碑应比普通任务更突出，点击里程碑打开对应任务或里程碑详情。
- Settings Tab 仅项目负责人、管理员或具备配置权限的人可见；普通成员只看到项目基本信息。

## 7. 权限与生命周期

- 新建项目后默认进入项目空间，并打开 Settings 或 Tasks Tab，引导补齐成员与任务。
- 归档项目后从默认项目列表移除，但任务、文件、审计记录仍可检索。
- 隐藏项目在列表中只对授权人员可见，跨项目引用时显示脱敏名称或“隐藏项目”。
- 删除项目属于极高风险操作，默认不建议提供硬删除；如提供，必须要求二次确认、原因和影响清单。
- 项目状态变更需要写入 Activity，并同步影响 Reports、Home 项目卡片和相关用户 Inbox。

## 建议组件

项目中心既要展示项目列表，又要承载项目内部空间，建议采用“列表视图复用 My Work 组件 + 项目空间自研 Tab 容器”的架构。基础控件统一使用 **shadcn/ui**，并通过 CSS 变量将 Card、Tabs、Table、Badge、Avatar、Button、DropdownMenu、Progress 等组件的主题锁定到 NexusFlow 色板：卡片/面板背景 `--bg-secondary`（#FFFFFF），Tab 激活态下划线 `--text-primary`（#1A1A1A），进度条填充 `--text-primary`，边框统一 `--border-subtle`（rgba(0,0,0,0.05)），文字层级 `--text-primary`/`--text-muted`。项目封面图可存在，但封面色需从固定中性 palette 选取，避免与系统语义色冲突。

项目列表的 **List / Table / Board / Gallery / Timeline** 五种视图尽量复用 My Work 与 Schedule 的现有实现，仅在数据范围上过滤为当前项目集合。**List 视图**使用自研高密度行组件，行高 56px，字段包括项目名称/编号、状态、类型、负责人、归属组织、起止时间、进度、任务数、风险数、成员数，hover 背景 `--bg-tertiary`；**Table 视图**使用 **TanStack Table**，支持列宽拖拽、列隐藏、排序；**Board 视图**使用 **@dnd-kit** 拖拽，列头与卡片背景层级与 My Work 保持一致；**Timeline 视图**复用自研 Timeline 组件，展示项目周期时间轴。

**Gallery 视图**建议自研 **Project Gallery Card 组件**。卡片使用 shadcn/ui `Card` 包裹，封面区高度 120px，允许项目自定义封面色（从有限 palette 选择，如灰、米、浅蓝等中性色）；封面下方展示项目名称（`text-primary font-medium`）、状态 badge（语义色）、进度条（shadcn/ui `Progress`，指示器 `--text-primary`）、负责人头像组（shadcn/ui `Avatar` + 自研 AvatarGroup，头像边框 `--bg-secondary`）、最近更新时间（`text-muted`）。卡片网格使用 CSS Grid 响应式（桌面 3 列、平板 2 列、移动 1 列），hover 抬升 `translate-y-[-2px]`、阴影 `shadow-md`，过渡 200ms。

项目内部空间的 Tab 导航使用 shadcn/ui `Tabs`，包含 Tasks、Members、Files、Timeline、Risks、Settings。每个 Tab 内容区背景 `--bg-secondary`，与全局页面背景 `--bg-primary` 形成层级。Tasks Tab 直接复用 My Work 的任务视图组件并注入项目过滤，新建任务自动关联当前项目；Members Tab 使用自研成员列表或 shadcn/ui `Table`，展示姓名、项目身份、负责工作、加入/退出时间，支持添加成员、修改身份、移除成员；Files Tab 复用 Files 页面目录树组件，可勾选“包含任务资料”；Timeline Tab 复用 Schedule 甘特组件并限定项目内任务与里程碑；Risks Tab 使用 shadcn/ui `Badge` + 自研风险列表；Settings Tab 使用 shadcn/ui `Form` + `Switch` + `Select` 编辑项目基本信息、可见性、隐藏项目授权、归档/取消。

筛选与排序位于列表顶部，使用 shadcn/ui `DropdownMenu` 与 `Select`：筛选包括状态、类型、级别、归属组织、负责人、风险等级；排序包括进度、截止时间、创建时间、风险等级。长列表统一使用 **TanStack Virtual**。项目空间内的右键菜单/更多操作使用 shadcn/ui `DropdownMenu`。删除/归档等危险操作使用 shadcn/ui `AlertDialog` 二次确认。所有状态 badge 严格使用语义色：`--success`（正常）、`--warning`（警告）、`--error`（风险/延期）、`--info`（进行中），不新增状态色。项目空间的路由建议使用子路由（如 `/projects/:id/tasks`），切换 Tab 时不丢失列表状态。
