# 任务详情（Task Detail）

> 参考 Linear Issue Detail 和 Figma Right Panel，任务详情是系统最高频的右侧面板，要求信息层级清晰、编辑顺滑、操作触手可及。

## 1. 页面目标

在任意列表/看板/时间线中点击任务后，从右侧滑出详情面板，集中展示任务全生命周期信息并支持处理。

## 3. 顶部信息区

- 任务名称：点击行内编辑。
- 状态 / 类型 / 优先级 / 风险：badge 展示。
- 负责人 / 起止时间 / 进度 / 关联项目：可点击编辑。

## 4. Overview Tab

- 任务概述（块编辑器）。
- 成果要求（清单）。
- 里程碑（小时间线）。
- 成员概览。
- 风险概览。
- 评论线程（支持 @ 提及）。

## 5. Subtasks Tab（分工）

- 分工列表。
- 每个分工：名称、责任人、协作人、时间、投入、进度、成果状态。
- 支持新增、编辑、提交成果、确认、退回。

## 6. Timeline Tab

- 当前任务及其分工、里程碑的迷你甘特图。

## 7. Files Tab

- 必需资料清单。
- 已上传资料列表。
- 支持上传新版本。

## 8. Approvals Tab

- 跨部门协调、强制安排、任务变更、隐藏项目授权记录。

## 9. Activity Tab

- 状态流转、字段变更、评论、资料上传、验收记录。
- 字段变更支持版本对比。

## 10. 底部主操作区

根据状态动态显示：

- 草稿：编辑 / 提交 / 删除
- 待协调：查看协调 / 撤回
- 待确认：确认发布 / 退回
- 进行中：更新进度 / 提交验收 / 暂停 / 变更
- 待验收：通过 / 退回 / 补充资料
- 已完成：归档

## 11. 生命周期与状态流

- 草稿任务只对创建者和有权限管理者可见，不进入成员负载和团队日程。
- 待确认任务需要负责人或成员确认后进入进行中；被退回时回到创建者待办，并保留退回原因。
- 进行中任务允许更新进度、提交成果、发起变更；进度更新应写入 Activity，但不必每次都产生通知。
- 待验收任务的主操作为通过/退回，退回必须填写原因并指向需要补充的成果或分工。
- 已完成任务默认只读，可归档；归档后从默认列表隐藏，但仍可在 All/Archived 中查询。
- 暂停、变更、强制安排属于敏感动作，必须进入审批或要求填写原因。

## 12. 编辑与协作细节

- 标题、负责人、日期、进度支持内联编辑；状态流转走底部主操作，不通过普通下拉随意改状态。
- 块编辑器内容自动保存草稿，失焦保存；保存失败时显示未同步标记并允许重试。
- 评论线程按时间正序展示，输入框固定在评论区底部；@ 提及人员后，对方收到 Inbox 通知。
- Subtasks 中单个分工的成果提交后，只影响该分工状态；所有必要分工完成后，任务才可提交验收。
- Files Tab 中“必需资料”未满足时，底部主操作“提交验收”置灰并说明缺少哪些资料。
- Approvals Tab 中的审批记录不可删除，只能追加补充说明。

## 13. 面板与上下文

- 从列表、看板、甘特、日历打开任务详情时，面板关闭后应返回原视图和原滚动位置。
- 固定面板后，用户点击其他任务时更新面板内容，不关闭面板；未保存编辑需要提示。
- 面板内打开关联项目或人员时，优先使用二级面板或替换策略，但必须保留返回当前任务的路径。
- 无权限字段不应隐藏整块信息，能脱敏就脱敏；例如隐藏项目显示“隐藏项目”，但保留任务自身可见信息。
- 任务被其他人更新时，如果当前用户正在编辑同一字段，需要提示冲突并允许查看最新值。

## 建议组件

任务详情是系统最高频的右侧面板，信息密度最高，建议面板容器使用 shadcn/ui `Sheet`（桌面端默认 560px，复杂审批可扩展至 720px；移动端使用 **vaul**）。所有 shadcn 组件通过 CSS 变量严格映射到 NexusFlow 色板：`Sheet` 背景 `--bg-secondary`（#FFFFFF），遮罩 `bg-black/20`，文字层级 `--text-primary`（#1A1A1A）/`--text-muted`（#7A7A7A），分隔线 `--border-subtle`（rgba(0,0,0,0.05)）。底部主操作区背景与面板背景一致，仅在顶部加 1px `--border-subtle` 分隔，避免视觉割裂。

顶部信息区的任务名称行内编辑建议自研 **Editable Field 组件**：普通态显示为 `text-xl font-semibold text-primary`，点击后切换为 shadcn/ui `Input`，背景 `--bg-tertiary`，边框聚焦 `--text-primary`，保存后切换回普通态，支持 `Enter` 保存、`Esc` 取消。状态、类型、优先级、风险使用 shadcn/ui `Badge`，颜色严格对应语义色：`--success`（已完成）、`--warning`（待确认/待验收）、`--error`（有风险/拒绝）、`--info`（进行中）。负责人/起止时间/进度/项目使用自研可编辑组件或 shadcn/ui `DropdownMenu`/`Popover`（日期选择器），日期选择器背景 `--bg-secondary`、选中 `--text-primary`、今天 `--bg-tertiary`。

Tab 导航使用 shadcn/ui `Tabs`：Overview、Subtasks、Timeline、Files、Approvals、Activity。内容区背景 `--bg-secondary`，与面板背景融合。Overview Tab 的任务概述使用自研块编辑器（基于 **BlockNote** 或 **Editor.js**），支持段落、标题、列表、待办、附件、@ 提及；成果要求使用清单组件，里程碑使用小时间线组件，成员概览使用 AvatarGroup，风险概览使用语义色 badge。评论线程也使用该编辑器，@ 提及人员搜索通过 **cmdk** 实现小范围命令面板，弹出层背景 `--bg-secondary`，边框 `--border-subtle`。

Subtasks Tab 的分工列表建议自研 **Subtask List 组件**，行高 48px，每行展示名称、责任人、协作人、时间、投入、进度、成果状态。hover 行背景 `--bg-tertiary`，右侧渐显操作按钮（提交成果、确认、退回），使用 shadcn/ui `Button` 的 `ghost` 变体。进度使用 shadcn/ui `Progress` 或 `Slider`（可编辑时），指示器 `--text-primary`。

Timeline Tab 使用自研迷你甘特组件（SVG），复用 Schedule 页面时间轴逻辑，任务条填充 `--text-primary` 10% 透明度，里程碑菱形 `--text-primary`，今天线 `--info`，风险标记 `--error`。Files Tab 复用 Files 页面轻量列表与上传组件。Approvals Tab 展示跨部门协调、强制安排、任务变更、隐藏项目授权记录，使用自研时间线组件，审批状态 badge 使用语义色。Activity Tab 字段变更支持版本对比，建议自研 **Diff View 组件**：删除行背景 `--error` 10% 透明度，新增行背景 `--success` 10% 透明度，仅使用语义色表示差异。

底部主操作区根据状态动态生成按钮组，主要操作使用 shadcn/ui `Button` 填充 `--text-primary`，危险操作（退回/暂停/删除）使用 `outline` 变体并设置文字 `--error`。操作反馈使用 **Sonner**，成功 `--success`，失败 `--error`，保持语义色一致。固定功能使用 shadcn/ui `Button` 的 `ghost` 变体，固定后图标颜色 `--text-primary`。任务详情的所有字段变更建议通过 optimistic update 即时反馈，失败时回滚并提示。
