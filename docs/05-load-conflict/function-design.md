# 负载与冲突功能设计

## 1. 功能目标

负载与冲突模块用于判断人员是否还能承接任务，并在任务派发、任务变更、人员状态变化时提前发现风险。

目标：

- 按人员和日期计算工作占用。
- 实时识别超载、全天占用、休假冲突、跨部门未审批等问题。
- 将冲突反馈给新建任务、任务详情、甘特图、人员详情和首页。
- 支持管理者处理冲突并保留记录。

## 2. 负载计算对象

参与负载计算的对象：

- 已生效任务。
- 已生效分工。
- 休假任务。
- 出差任务。
- 培训任务。
- 全天重点任务。
- 已审批通过的跨部门任务。

不参与正式负载计算但参与预估：

- 草稿任务。
- 待协调任务。
- 待确认任务。

## 3. 每日负载

### 3.1 基础公式

```text
某人某日负载 = 当日所有生效任务预计投入小时总和 / 每日标准工时
```

默认每日标准工时为 8 小时，可在人员档案或配置中心调整。

截止风险预测公式：

```text
剩余工时 = 预计总工时 - 已确认投入工时
可用工时 = 截止日前每个工作日可用容量之和
预测完成日 = 从当前日期起按每日可用容量抵扣剩余工时后的日期
延期天数 = max(0, 预测完成日 - 截止日期)
```

当剩余工时大于截止日前可用工时时，系统生成截止风险。若任务或分工未填写预计总工时，则只根据截止日期、进度、阶段成果和人工标记识别延期风险。

### 3.2 全天任务

任务投入可设置为全天。

全天任务规则：

- 全天任务记为占满当日标准工时。
- 全天任务期间人员在选择器中置灰。
- 全天任务与任何其他任务重叠都提示冲突。
- 管理者可强制安排，但必须填写原因。

## 4. 冲突类型

| 类型 | 触发条件 | 风险等级 |
| --- | --- | --- |
| 工时超载 | 当日投入超过标准工时 | 高 |
| 全天重叠 | 全天任务和其他任务重叠 | 高 |
| 休假冲突 | 休假期间安排工作 | 高 |
| 出差冲突 | 出差期间安排不兼容任务 | 中 |
| 培训冲突 | 培训期间安排任务 | 中 |
| 跨部门未审批 | 外部门人员未审批通过 | 高 |
| 隐藏项目未授权 | 成员未获得隐藏项目授权 | 高 |
| 截止风险 | 按当前投入预计无法按期完成 | 中 |

## 5. 风险等级

风险分为：

- 低：提示信息，不阻断发布。
- 中：需要负责人确认，可发布。
- 高：需要审批或处理后才可发布。
- 严重：默认阻断，只有中心主任/授权管理者可强制安排。

## 6. 冲突处理动作

处理动作：

- 调整人员。
- 调整任务时间。
- 调整每日投入。
- 拆分分工。
- 延长截止时间。
- 调整预计总工时。
- 发起跨部门协调。
- 强制安排。
- 取消任务。

强制安排要求：

- 只有具备权限的管理者可操作。
- 必须填写原因。
- 必须通知受影响任务负责人。
- 必须写入任务日志和冲突记录。

## 7. 冲突状态

冲突状态：

- 未处理。
- 处理中。
- 已调整。
- 已审批。
- 已强制安排。
- 已忽略。
- 已关闭。

“已忽略”只适用于低风险提示，高风险不得忽略。

## 8. 触发时机

系统应在以下时机重新计算：

- 新建任务选择人员。
- 修改任务起止时间。
- 修改成员每日投入。
- 新增或删除分工。
- 人员提交休假。
- 任务状态从待确认变为进行中。
- 跨部门审批通过。
- 管理者强制安排。
