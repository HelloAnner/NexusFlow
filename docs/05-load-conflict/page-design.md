# 负载与冲突页面设计

## 1. 页面清单

- 人员负载日历。
- 冲突中心。
- 冲突详情抽屉。
- 任务派发冲突检查步骤。
- 甘特图风险标记。

## 2. 人员负载日历

### 2.1 页面目标

让管理者和负责人按人员查看每日占用，判断是否适合安排新任务。

### 2.2 视图模式

- 周视图。
- 月视图。
- 自定义时间范围。

### 2.3 展示内容

每个日期格展示：

- 总负载百分比。
- 任务数量。
- 全天占用标识。
- 风险标识。

点击日期展示：

- 任务名称。
- 分工名称。
- 投入小时。
- 任务负责人。
- 任务状态。
- 是否重点任务。

### 2.4 颜色规则

- 绿色：低于 60%。
- 黄色：60% 到 90%。
- 橙色：90% 到 100%。
- 红色：超过 100%。
- 灰色：全天占用或不可派发。

## 3. 冲突中心

### 3.1 页面目标

集中处理所有未解决冲突，避免风险只散落在任务详情中。

### 3.2 顶部统计

- 未处理冲突数。
- 高风险冲突数。
- 今日新增冲突数。
- 超时未处理数。

### 3.3 筛选项

- 冲突类型。
- 风险等级。
- 处理状态。
- 涉及人员。
- 涉及部门。
- 涉及任务。
- 时间范围。
- 责任处理人。

### 3.4 列表字段

| 字段 | 说明 |
| --- | --- |
| 冲突类型 | 工时超载、休假冲突等 |
| 风险等级 | 低、中、高、严重 |
| 涉及人员 | 冲突人员 |
| 涉及任务 | 一个或多个任务 |
| 冲突时间 | 日期或日期范围 |
| 超载小时 | 超过标准工时的小时数 |
| 推荐动作 | 系统建议 |
| 处理人 | 当前责任人 |
| 状态 | 未处理、处理中等 |

## 4. 冲突详情抽屉

### 4.1 基础信息

- 冲突编号。
- 冲突类型。
- 风险等级。
- 触发来源。
- 生成时间。

### 4.2 时间线视图

按日期展示人员占用：

- 已有任务。
- 新增任务。
- 每日投入。
- 超载情况。

### 4.3 处理建议

系统根据冲突类型给出建议：

- 工时超载：降低投入或调整日期。
- 全天重叠：替换人员或申请强制安排。
- 休假冲突：调整任务周期或取消派发。
- 跨部门未审批：发起协调。

### 4.4 处理操作

- 修改任务计划。
- 修改分工投入。
- 发起协调。
- 标记已解决。
- 强制安排。

## 5. 派发页冲突检查

在新建任务最后一步展示：

- 全部冲突列表。
- 按人员分组的冲突。
- 是否阻断发布。
- 每个冲突的处理动作。

阻断规则：

- 高风险和严重冲突默认阻断。
- 中风险需要二次确认。
- 低风险仅提示。

## 6. 甘特图风险标记

甘特图中：

- 超载日期在人员行显示红色底纹。
- 冲突任务条显示红色描边。
- 延期任务条右侧显示延期标记。
- 鼠标悬浮展示冲突摘要。
