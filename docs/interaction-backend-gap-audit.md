# 交互逻辑与后端接口逐功能差距核查

核查范围：`docs` 设计文档、`frontend/src/App.tsx` 路由、`frontend/src/pages` 页面交互、`backend/src/app/routes.rs` 路由和 `backend/src/domains` 主要实现。验收口径仍按项目约定：服务器 `ssh nexusflow`，默认端口 `8089`。

## 总体判断

- 后端路由清单覆盖了文档里的大多数一级接口，但很多接口仍是通用 JSON CRUD、基础状态流转或最小统计，距离文档里的业务校验、审批策略、配置编辑器、审计详情还有差距。
- 前端已从 mock 切到真实 API，但当前页面数少于文档页面清单：只有首页、任务列表、新建任务、任务详情、项目列表、人员列表、甘特、冲突、配置、工具、资料、SA 后台等核心入口。
- 交互缺口集中在“详情页、编辑页、配置页、审批/协调页、报表/通知中心、全局搜索页、权限矩阵页”。
- SA 后台已经有二级导航，但多数二级功能仍是轻量表格和基础动作，未达到文档要求的完整管理台。
