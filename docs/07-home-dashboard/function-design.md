# 首页与看板功能设计

## 1. 功能目标

首页是用户进入系统后的工作入口。不同角色关注点不同，但首页必须使用统一结构，让用户能快速进入任务、待办、风险、甘特图和工具。

目标：

- 员工知道今天做什么、要交什么。
- 负责人知道成员进度、风险和待确认成果。
- 部门主任知道部门负载、跨部门协调和风险任务。
- 中心领导知道中心重点任务和全局风险。
- 管理员知道配置异常和系统待维护事项。
- SA 超级管理员知道系统运行状态、待审核注册、邀请链接和权限风险。

## 2. 首页组件

通用组件：

- 我的待办。
- 我的任务。
- 风险提醒。
- 甘特图摘要。
- 任务日历。
- 快捷入口。
- 最近动态。

可选组件：

- 系统运行状态。
- 待审核注册。
- 邀请链接使用情况。
- 权限审计风险。
- 部门负载。
- 中心任务总览。
- 重点任务。
- 跨部门协调。
- 资料缺失提醒。
- 工具推荐。

## 3. 角色首页

### 3.1 SA 超级管理员首页

重点展示：

- 系统运行状态。
- 配置异常。
- 待审核注册。
- 邀请链接使用情况。
- 最近权限变更。
- 敏感访问审计。

主操作：

- 新建组织。
- 新建人员。
- 生成邀请链接。
- 处理待审核注册。
- 查看系统运行状态。

### 3.2 员工首页

重点展示：

- 今日分工。
- 本周任务。
- 待提交成果。
- 即将截止任务。
- 个人负载。

主操作：

- 填报进度。
- 提交成果。
- 查看任务。
- 申请休假或记录个人工作。

### 3.3 任务负责人首页

重点展示：

- 我负责的任务。
- 成员进度异常。
- 待确认成果。
- 待提交验收。
- 任务风险。

主操作：

- 创建分工。
- 确认成果。
- 处理阻塞。
- 提交验收。

### 3.4 部门主任首页

重点展示：

- 部门任务甘特图。
- 部门人员负载。
- 跨部门协调审批。
- 部门风险任务。
- 重点人员占用。

主操作：

- 审批协调。
- 调整人员。
- 查看部门甘特图。
- 发起任务。

### 3.5 中心领导首页

重点展示：

- 中心任务总览。
- 重点任务进度。
- 高风险任务。
- 跨部门资源冲突。
- 部门负载排行。

主操作：

- 查看全局甘特图。
- 处理升级协调。
- 查看报表。
- 发起中心级任务。

### 3.6 待审核用户首页

重点展示：

- 注册信息确认。
- 审核状态。
- 联系管理员。

主操作：

- 补充资料。
- 查看审核状态。

## 4. 待办聚合

首页待办来自：

- 任务确认。
- 协调审批。
- 进度填报。
- 成果确认。
- 任务验收。
- 冲突处理。
- 资料补充。

待办要求：

- 每条待办必须有明确处理按钮。
- 点击待办进入具体页面和锚点。
- 已处理待办自动消失。
- 超时待办突出显示。

## 5. 首页配置

配置中心可配置：

- 角色默认首页组件。
- 组件排序。
- 默认时间范围。
- 默认数据范围。
- 是否展示快捷入口。
- 是否展示风险模块。
- 是否展示 SA 后台入口。
- 用户多角色时的默认角色视角。
