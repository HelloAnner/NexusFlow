# 全局领域模型

## 1. 核心对象关系

```text
组织 -> 人员 -> 项目归属
邀请 -> 注册 -> 账号 -> 人员
项目 -> 项目成员 -> 项目任务
人员 -> 角色 -> 数据权限
项目 -> 隐藏授权 -> 数据权限
任务 -> 分工 -> 成员进度
任务 -> 资料 -> 阶段成果/最终成果
任务 -> 审批 -> 协调记录
任务 -> 风险 -> 冲突中心
配置 -> 任务模板/审批规则/告警规则/页面视图
```

## 2. 账号与邀请

账号是登录身份，人员是业务身份。一个账号绑定一个人员档案，人员档案承接组织、角色、项目和任务关系。

邀请用于降低注册门槛：

- 邀请链接绑定默认组织、角色、项目和项目身份。
- 用户通过邀请链接注册后自动创建或绑定人员档案。
- 需要审核时，账号状态为待审核，不能访问业务数据。
- SA 超级管理员可生成、停用、审计邀请链接。

SA 后台用于：

- 系统初始化。
- 组织和人员全量维护。
- 邀请模板和邀请链接管理。
- 角色入口配置。
- 账号安全。
- 审计日志。

## 3. 组织

组织是人员权限和统计口径的基础。主组织路径为：

```text
公司 / 二级单位 / 三级单位或中心 / 部门
```

创新工作室是横向组织，一个人员可以加入多个工作室，但不替代主组织。

组织应保存：

- 组织名称。
- 上级组织。
- 组织类型。
- 组织负责人。
- 组织副负责人。
- 技术监督。
- 默认审批人。
- 是否启用。

## 4. 人员

人员是任务派发、负载计算和权限判断的核心实体。

人员与组织关系：

- 一个主组织。
- 多个创新工作室。
- 多个项目归属。
- 多个技能标签。

人员与任务关系：

- 可作为任务发起人。
- 可作为任务负责人。
- 可作为任务成员。
- 可作为分工责任人。
- 可作为审批人或验收人。

## 5. 项目

项目是任务归属、人员项目身份、项目甘特图和项目报表的统一口径。

项目包含：

- 基本信息。
- 归属组织。
- 项目负责人。
- 项目成员。
- 项目身份和负责工作。
- 项目可见性。
- 项目任务。
- 项目资料。
- 项目授权。
- 项目状态。

项目和任务关系：

- 一个任务最多关联一个主项目。
- 任务也可以不关联项目，作为独立工作、个人事项、出差、学习、休假或安全任务。
- 项目成员不等于任务成员，只有加入任务或分工后才产生工时占用。
- 隐藏项目未授权时，项目、任务、资料、搜索和报表明细都不可见。

## 6. 任务

任务是一切工作事项的统一载体。任务可以关联项目，也可以是独立工作。

任务包含：

- 基本信息。
- 参与人员。
- 分工。
- 计划时间。
- 每日投入。
- 任务概述。
- 成果要求。
- 资料。
- 审批。
- 风险。
- 状态流转记录。

任务和分工的关系：

- 一个任务可以有多个分工。
- 一个分工至少有一个责任人。
- 分工可以有协作人。
- 分工进度汇总后影响任务整体进度。

## 7. 资料

资料可以关联任务或分工。资料既是过程记录，也是验收依据。

资料状态：

- 草稿。
- 已提交。
- 已确认。
- 已退回。
- 已归档。

资料类型由配置中心维护，不同任务类型可以要求不同必填资料。

## 8. 审批与协调

审批不是独立业务目标，而是任务派发和任务变更中的控制点。

审批触发场景：

- 跨部门派发。
- 隐藏项目授权。
- 强制安排冲突人员。
- 任务重大变更。
- 任务取消或暂停。

审批记录必须关联具体任务和触发原因。

## 9. 风险与冲突

风险面向任务管理，冲突面向资源协调。

风险类型：

- 时间延期。
- 阶段成果未提交。
- 验收退回。
- 资料不完整。
- 审批超时。

冲突类型：

- 人员工时超载。
- 全天任务重叠。
- 休假、出差、培训状态冲突。
- 跨部门未审批。
- 隐藏项目未授权。

## 10. 甘特图与搜索

甘特图是任务排期和资源协调视图，不单独保存业务事实。

甘特图数据来自：

- 项目。
- 任务。
- 分工。
- 里程碑。
- 人员状态。
- 冲突。
- 风险。

搜索覆盖：

- 任务。
- 项目。
- 人员。
- 资料。

甘特图和搜索必须共用权限裁剪规则，隐藏项目和个人事项不得泄露。

## 11. 配置

配置中心影响系统全局行为。核心配置包括：

- 组织层级。
- 角色权限。
- 任务类型。
- 任务模板。
- 分工模板。
- 审批规则。
- 告警规则。
- 资料类型。
- 首页视图。
- 甘特图规则。
- 工具入口。
- 邀请注册策略。
- 角色入口配置。

## 12. 技术运行模型

系统运行形态：

- Rust 后端。
- TypeScript + Bun 前端。
- 单二进制部署。
- 默认端口 `8089`。
- `.env` 配置 PostgreSQL、Redis、S3 兼容对象存储等基础组件连接地址。
- 不要求 Docker。
