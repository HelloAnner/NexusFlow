# 产品范围与全局体验

## 1. 产品定位

工作管理系统是中心内部的统一工作协同平台。它不只管理科研或市场项目，也管理任何会占用人员时间、产生分工、需要汇报或需要留痕的工作事项。

系统需要解决四个核心问题：

- 谁能看什么、派什么、审批什么。
- 谁正在做什么、什么时候完成、还能不能接新工作。
- 一项工作从发起到归档，中间怎么拆分、怎么协同、怎么验收。
- 管理者如何在首页和甘特图中及时发现风险并完成协调。

## 2. 一期必须覆盖

### 2.1 组织与人员

- 支持公司、二级单位、三级单位/中心、部门、创新工作室。
- 支持人员主组织、工作室多归属、项目多归属。
- 支持人员等级、系统角色、项目身份、技能标签。
- 支持人员当前状态和每日标准工时，用于任务派发和负载计算。
- 支持邀请注册自动创建人员档案、绑定组织和角色。
- 支持 SA 超级管理员配置人员全部信息。

### 2.1.1 账号与邀请注册

- 支持按组织、角色、项目生成邀请链接。
- 不同角色使用不同邀请链接。
- 注册后自动进入对应组织、角色和项目。
- 支持待审核注册、已有账号绑定邀请、邀请失效和重复注册处理。
- 支持角色默认首页、导航、快捷入口和待办类型配置。

### 2.1.2 项目管理

- 支持项目基础信息、项目负责人、项目成员、项目身份和负责工作。
- 支持普通项目、隐藏项目和指定范围项目。
- 支持项目下任务、资料、甘特图和报表统计。
- 支持项目成员历史和项目归档快照。

### 2.2 任务全生命周期

- 任务创建、草稿保存、提交协调、确认发布。
- 负责人分工、成员进度填报、阶段成果提交。
- 负责人审核、发起人或验收人验收、归档。
- 任务概述、时间、成员、投入量、交付物变更留痕。

### 2.3 权限与协同

- 按角色、组织、项目、隐藏项目授权控制数据范围。
- 跨部门派发自动触发协调审批。
- 项目负责人只能派发项目相关工作，不能绕过部门资源协调。
- 中心主任/书记具备全局查看、派发和最终协调权限。

### 2.4 负载与风险

- 按人员、日期、任务投入计算负载。
- 超过每日标准工时预警。
- 全天任务期间人员置灰。
- 休假、出差、培训和任务重叠时提示状态冲突。
- 冲突中心集中处理待协调事项。

### 2.5 页面与配置

- 登录、邀请注册、SA 后台、首页工作台、任务列表、任务详情、新建任务、甘特图、全局搜索、人员管理、项目管理、冲突中心、资料库、工具台、配置中心。
- 任务类型、字段模板、审批规则、告警规则、资料类型、首页组件可配置。

### 2.6 技术交付

- 后端使用 Rust。
- 前端使用 TypeScript + Bun。
- 最终交付单二进制，默认监听端口 `8089`。
- 基础组件通过 `.env` 配置连接地址和凭据，不使用 Docker 作为部署前提。
- 默认基础组件为 PostgreSQL、Redis、S3 兼容对象存储，搜索一期使用 PostgreSQL 全文检索。

## 3. 一期不做或弱化

- 不做复杂人员能力画像算法，只保留能力标签和后续评分入口。
- 不做代码仓库解析，只保留代码资料类型。
- 智能体工具一期只做入口管理和可见性配置，不做复杂业务编排。
- 不做外部系统强集成，但页面和数据模型预留外部通知、日历、IM 集成能力。

## 4. 全局导航

建议一级导航：

- 首页。
- 任务。
- 甘特图。
- 项目。
- 人员。
- 冲突中心。
- 资料库。
- 工具台。
- 配置中心。

导航规则：

- 当前用户无权限访问的菜单隐藏。
- “新建任务”作为全局主操作，出现在顶部导航或首页快捷区。
- SA 超级管理员默认进入 SA 后台首页。
- 待审核用户只进入资料完善和审核状态页。
- 列表页、详情页、甘特图、人员详情之间必须互相跳转。
- 所有任务、人员、资料链接都应能落到一个明确详情页，而不是只在列表中展示。

## 5. 全局搜索

一期建议支持对象级搜索：

- 搜任务：按任务名称、概述、负责人、成员、任务编号搜索。
- 搜项目：按项目名称、编号、负责人、成员搜索。
- 搜人员：按姓名、组织、技能、项目搜索。
- 搜资料：按资料名称、任务、上传人、资料类型搜索。

搜索结果展示：

- 区分任务、项目、人员、资料四类结果。
- 每条结果展示对象状态和权限提示。
- 未授权对象不展示标题和摘要，只提示无访问权限或直接过滤。

## 6. 全局工作流入口

所有角色的高频动作应在首页可达：

- 员工：填报进度、提交成果、查看今日任务。
- 负责人：分配分工、确认成果、提交验收、处理风险。
- 部门主任：审批跨部门协调、查看部门负载、调整人员。
- 中心领导：查看中心风险、重点任务和全局甘特图。
- 管理员：维护配置、组织、角色、任务模板。
