# 权限功能设计

## 1. 权限目标

权限模块负责回答三个问题：

- 当前用户能看到哪些数据。
- 当前用户能执行哪些操作。
- 当前用户跨组织、跨项目操作时是否需要审批。

权限必须在所有页面保持一致，包括首页、任务列表、任务详情、人员、资料库、甘特图、报表和全局搜索。

## 2. 权限组成

权限由四类规则共同决定：

- SA 超级管理员：决定系统初始化和全局兜底管理能力。
- 系统角色：决定操作能力。
- 组织归属：决定组织数据范围。
- 项目归属：决定项目数据范围。
- 任务可见性：决定特殊任务或隐藏项目的访问范围。

最终权限取交集。任何一类规则不允许访问时，页面不得展示敏感数据。

## 3. 系统角色

| 角色 | 查看范围 | 操作能力 |
| --- | --- | --- |
| SA 超级管理员 | 系统全量，含后台配置和审计 | 组织、人员、角色、邀请、配置、账号安全和审计兜底 |
| 中心主任/书记 | 中心全量 | 全员派发、全局协调、查看报表 |
| 中心副主任/技术总监 | 分管部门和项目 | 分管范围派发、审批、协调 |
| 部门主任/技术监督 | 所属部门 | 部门人员派发、跨部门审批、资源协调 |
| 部门副主任 | 所属部门授权范围 | 协助派发和审批 |
| 项目负责人 | 所属项目 | 项目任务派发、分工、成果确认 |
| 任务负责人 | 所负责任务 | 分工、进度跟踪、验收提交 |
| 员工 | 本人任务和所属项目 | 填报进度、提交资料 |
| 系统管理员 | 配置和基础数据 | 维护组织、角色、配置 |
| 待审核用户 | 无业务数据 | 完善资料、查看审核状态 |

## 4. 数据范围

### 4.1 组织范围

组织范围分为：

- 本人。
- 本部门。
- 分管部门。
- 所属中心。
- 全中心。
- 自定义组织集合。

组织范围影响：

- 人员列表。
- 任务列表。
- 甘特图。
- 负载日历。
- 报表。

### 4.2 项目范围

项目范围分为：

- 本人参与项目。
- 本人负责项目。
- 本部门项目。
- 分管项目。
- 全中心项目。
- 指定授权项目。

项目范围影响：

- 项目列表。
- 项目详情。
- 项目任务。
- 项目资料。
- 项目成员。
- 项目甘特图。
- 项目报表。

项目授权类型：

- 查看项目。
- 查看项目任务。
- 参与项目任务。
- 上传项目资料。
- 管理项目成员。
- 查看项目报表。

### 4.3 任务可见性

任务可见性：

- 普通：按组织和项目权限可见。
- 隐藏项目：仅显式授权用户可见。
- 指定范围：仅指定组织、人员或角色可见。
- 个人事项：本人和直属管理者可见。

隐藏项目规则：

- 未授权用户在搜索、列表、甘特图、报表中都不可见。
- 如果用户因任务成员身份被授权，只能查看该任务和必要上下文。
- 隐藏项目资料也必须继承隐藏规则。
- 隐藏项目的通知不得暴露项目名称、任务名称和资料名称。

## 5. 操作权限

操作权限分为：

- 查看。
- 新建。
- 编辑。
- 派发。
- 审批。
- 变更。
- 验收。
- 归档。
- 配置。
- 导出。

操作校验示例：

- 员工只能编辑自己的进度和成果，不能修改任务整体信息。
- 任务负责人可以编辑分工，但不能绕过跨部门审批新增外部门人员。
- 部门主任可以审批本部门人员被外部门调用。
- 中心主任可以强制安排，但必须留痕。
- 系统管理员可以维护配置，但默认不参与业务审批。
- SA 超级管理员可以维护全部人员信息和邀请注册策略，但业务数据敏感访问必须审计。
- 待审核用户不可访问任务、项目、人员、资料和报表。

## 6. 跨部门操作控制

跨部门判断：

- 发起人权限范围不包含目标成员主组织。
- 任务所属组织与成员主组织不同，且发起人不是中心主任。
- 项目负责人选择非项目成员或非本部门成员。

处理规则：

- 自动生成协调审批。
- 审批通过前，成员处于待加入状态。
- 待加入成员不计入正式进度，但可用于草案负载预估。
- 审批拒绝后，系统提示发起人调整人员或任务安排。

## 7. 权限审计

所有敏感操作应记录：

- 操作人。
- 操作时间。
- 操作对象。
- 操作类型。
- 变更前值。
- 变更后值。
- 操作来源页面。

敏感操作包括：

- 生成、复制、停用邀请链接。
- 审核注册用户。
- 修改角色。
- 修改组织。
- 添加隐藏项目授权。
- 强制安排冲突任务。
- 修改审批规则。
- 导出人员或任务数据。
- 导出项目或隐藏项目数据。
- SA 查看隐藏项目或个人事项。
