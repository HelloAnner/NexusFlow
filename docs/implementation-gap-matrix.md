# 交互与后端接口实现差距矩阵

本矩阵基于 `docs` 设计文档、`frontend/src/pages` 当前页面和 `backend/src/main.rs` API 路由静态核查生成。验收默认在 `ssh nexusflow` 服务器、`8089` 端口完成。

## 总体结论

- 后端 API 路由基本覆盖各模块技术设计清单，但大量接口仍是最小 CRUD 或最小状态流转。
- 前端已经接入真实 API，但多数页面仍偏列表/看板展示，筛选、批量操作、详情页、配置编辑器和多步骤向导不完整。
- SA 后台原先只有单页聚合看板；现已改为按二级功能逐项渲染，并接入邀请模板、邀请链接、注册审核、人员、账号安全、审计和运行状态。

## 01 组织与人员

- 已有接口：组织树、组织 CRUD、人员 CRUD、人员负载摘要、技能标签 CRUD。
- 已有前端：人员列表、SA 后台内的新建组织/人员和人员基础编辑。
- 差距：缺组织管理专页、人员详情页、技能标签管理页、人员负载日历和完整人员全量编辑字段。
- 下一步：补 `/people/:id`、组织树管理、技能标签管理，人员编辑扩展到账号安全、组织归属、角色权限、项目归属、工时和审计备注。

## 02 权限

- 已有接口：角色、角色动作、权限检查、隐藏项目授权、权限审计。
- 已有前端：SA 后台角色入口预览、审计日志读取。
- 差距：缺角色管理页、权限矩阵、数据范围配置页、隐藏项目授权页。
- 下一步：配置中心或独立权限页接入角色动作保存、数据范围规则和隐藏授权管理。

## 03 任务管理

- 已有接口：任务列表/详情/创建/更新、状态流转、分工和进度。
- 已有前端：任务列表、新建任务基础表单、任务详情页签和状态按钮。
- 差距：新建任务还不是五步向导；缺人员选择、分工草案、冲突检查、确认发布、验收页和编辑页。
- 下一步：把 `/tasks/new` 改为派发向导，接入 `/dispatch/preview`、`/dispatch/submit` 和 `/workload/preview`。

## 04 派发协同

- 已有接口：派发预览/提交、协调审批单、审批动作、会议记录。
- 已有前端：任务详情显示部分可用动作。
- 差距：缺协调单列表、协调单详情、审批调整、升级、后补填报确认。
- 下一步：新增协调中心或嵌入任务详情审批页签，逐项接入审批接口。

## 05 负载与冲突

- 已有接口：负载预览、人员负载、负载日历、冲突列表/详情/处理/强制/重算。
- 已有前端：冲突中心列表、处理、强制排期。
- 差距：缺冲突详情抽屉、处理建议、原因输入、二次确认、负载日历。
- 下一步：冲突中心补详情抽屉和强制原因表单，人员页补负载日历入口。

## 06 资料库

- 已有接口：资料列表/详情、上传地址、上传完成、版本、下载、关联、归档、需求检查。
- 已有前端：资料列表、上传资料、下载资料。
- 差距：对象存储未配置时只能登记元数据；缺资料详情页、版本列表、归档、任务资料页签和必需资料清单。
- 下一步：补资料详情和版本管理，任务详情资料页签接入资源关联与需求检查。

## 07 首页工作台

- 已有接口：首页、组件、角色入口、角色视角、最近动态。
- 已有前端：首页工作台。
- 差距：角色默认入口未真正用于登录跳转，菜单未按权限隐藏，SA 默认入口仍依赖用户手动进入系统管理。
- 下一步：登录后按 `/dashboard/role-entry` 跳转；侧边栏按 `/permissions/me` 裁剪。

## 08 配置中心

- 已有接口：配置模块、当前配置、草稿、发布、停用、版本、运行状态。
- 已有前端：配置中心读取模块、版本和运行状态；SA 后台角色入口可写入 `role_entry` 草稿并发布。
- 差距：缺任务类型、任务模板、审批规则、告警规则、首页视图、邀请注册配置的编辑器。
- 下一步：按 namespace 增加 JSON/表单编辑器，支持草稿、发布、停用和版本回看。

## 09 通知与报表

- 已有接口：待办、待办完成、通知、通知已读、报表、报表导出。
- 已有前端：首页/配置中心读取待办；已新增 `/todos` 待办中心、`/notifications` 通知中心和 `/reports` 报表中心。
- 差距：报表仍是快照目录和详情，缺趋势图、专项图表口径、导出文件下载；待办/通知暂未做高级筛选和批量操作。
- 下一步：补报表图表化展示、导出文件流、通知批量已读和待办批量处理。

## 10 工具台

- 已有接口：工具列表/详情/创建/更新、上下文、使用记录。
- 已有前端：工具台列表、打开工具并记录使用；已新增 `/tools/:id` 详情、上下文探测和使用记录。
- 差距：收藏仍无后端模型；工具创建/编辑、权限配置仍未独立接入。
- 下一步：补工具创建/编辑表单、工具权限配置和收藏模型。

## 11 项目管理

- 已有接口：项目 CRUD、成员、隐藏授权、归档、统计。
- 已有前端：项目列表。
- 差距：缺项目详情、成员页签、任务/甘特/资料/授权/日志页签，以及项目创建和归档交互。
- 下一步：新增 `/projects/:id`，从列表进入详情并接入成员和授权接口。

## 12 甘特与搜索

- 已有接口：甘特、摘要、搜索、建议、保存筛选。
- 已有前端：甘特图；顶部搜索跳任务列表。
- 差距：甘特维度切换未传给接口；缺全局搜索弹窗、搜索结果页、保存筛选、权限提示。
- 下一步：新增 `/search` 页面和全局搜索弹窗；甘特请求加入 dimension、granularity、riskOnly 等参数。

## 13 SA 后台、账号与邀请注册

- 已有接口：登录、登出、当前用户、邀请模板、邀请链接、邀请注册、注册审核、SA 首页、账号列表、账号禁用/解锁。
- 已有前端：登录、邀请注册入口和注册页；SA 后台二级功能视图；注册审核、人员/组织创建、邀请模板、邀请链接、角色入口配置、账号禁用/解锁、审计和运行状态。
- 差距：忘记密码、重置密码、强制下线、解绑登录方式无接口；审核时修改组织/角色、合并人员、标记异常无接口；邀请使用记录无接口。
- 下一步：补后端接口后接入 SA 对应二级功能。
