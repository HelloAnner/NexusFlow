# 设计实现差距矩阵

本矩阵基于 `docs` 设计文档、当前前端路由和后端 API 静态核查更新。验收默认在 `ssh nexusflow` 服务器、端口 `8089` 完成。

## 总体结论

- 核心一级入口已基本存在：任务、项目、组织、人员、甘特、冲突、审批、待办、通知、报表、搜索、资料、工具、配置和 SA 后台。
- 任务新建已接入五步派发向导，并调用派发预览、负载预览和派发提交。
- 登录后已读取角色入口，侧边栏已消费 `/permissions/me` 做菜单裁剪。
- `role_entry` 发布已同步 `role_entry_configs`，旧版“角色入口读写不一致”结论已失效。
- 项目、人员、资料详情工作台已落地：`GET /api/projects/{id}`、`GET /api/users/{id}`、`GET /api/resources/{id}` 返回聚合数据，前端 `/projects/:id`、`/people/:id`、`/resources/:id` 已接入列表跳转和对象操作。
- 冲突中心已补详情工作台基线：`GET /api/conflicts/{id}` 返回任务、人员、相关负载、事件和审计聚合，前端可查看处理建议、时间线、原因输入，并可跳转人员负载与甘特风险视图。
- 任务详情工作台已补执行闭环基线：`GET /api/tasks/{id}` 聚合成员、分工、进度汇报、资料、必需资料、资料完整率、审批、验收、变更日志和事件；前端任务详情已支持任务时间调整、状态操作理由、资料上传/关联、验收资料规则配置、分工新增/编辑、成果提交、成果确认/退回、进度填报、审批/验收/日志回看和任务删除。
- 当前最大差距从“有没有入口”转为“详情工作台、结构化配置、深交互和验收闭环是否完整”。
- 权限数据范围已接入任务、项目、人员、资料、甘特、全局搜索、报表快照生成和隐藏目标通知脱敏；更细粒度动作范围、导出文件下载内容和通知偏好仍需继续补强。

## P0 闭环差距

1. 项目、人员、资料详情工作台已补基础闭环，但仍缺更深编辑：项目风险/授权细粒度编辑、资料成果验收和任务资料闭环还需补强；人员技能绑定、人员项目归属编辑、人员负载日历和历史事件详情已完成服务器验收。
2. 配置中心仍以通用版本为主：任务模板、审批规则、告警规则、资料要求、首页视图缺结构化编辑器和发布校验。
3. 甘特图仍偏任务时间线：风险参数跳转和风险过滤基线已补，维度、粒度、项目/人员/组织筛选和条目浮层仍未完整实现。
4. 任务详情已有执行闭环基线，分工新增/编辑、成果确认/退回、任务级验收资料规则配置和资料完整率已完成服务器验收；仍缺风险聚合、归档档案视图和更细的异常状态提示。
5. 权限数据范围仍需扩展到导出文件下载内容和更细粒度授权动作；报表服务器聚合口径和隐藏目标通知脱敏已完成服务器验收。
