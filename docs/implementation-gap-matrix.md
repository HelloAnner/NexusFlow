# 设计实现差距矩阵

本矩阵基于 `docs` 设计文档、当前前端路由和后端 API 静态核查更新。验收默认在 `ssh nexusflow` 服务器、端口 `8089` 完成。

## 总体结论

- 核心一级入口已基本存在：任务、项目、组织、人员、甘特、冲突、审批、待办、通知、报表、搜索、资料、工具、配置和 SA 后台。
- 任务新建已接入五步派发向导，并调用派发预览、负载预览和派发提交。
- 登录后已读取角色入口，侧边栏已消费 `/permissions/me` 做菜单裁剪。
- `role_entry` 发布已同步 `role_entry_configs`，旧版“角色入口读写不一致”结论已失效。
- 项目、人员、资料详情工作台已落地：`GET /api/projects/{id}`、`GET /api/users/{id}`、`GET /api/resources/{id}` 返回聚合数据，前端 `/projects/:id`、`/people/:id`、`/resources/:id` 已接入列表跳转和对象操作。
- 任务详情工作台已补执行闭环基线：`GET /api/tasks/{id}` 聚合成员、分工、进度汇报、资料、必需资料、资料完整率、审批、验收、变更日志和事件；前端任务详情已支持任务时间调整、状态操作理由、资料上传/关联、验收资料规则配置、分工新增/编辑、成果提交、成果确认/退回、进度填报、审批/验收/日志回看。
- 当前最大差距从“有没有入口”转为“详情工作台、结构化配置、深交互和验收闭环是否完整”。
- 权限数据范围已接入任务、项目、人员、资料、甘特和全局搜索的列表与详情直连校验；报表、通知脱敏和更细粒度动作范围仍需继续补强。

## P0 闭环差距

1. 项目、人员、资料详情工作台已补基础闭环，但仍缺更深编辑：项目风险/授权细粒度编辑、人员技能绑定/负载日历、资料成果验收和任务资料闭环还需补强。
2. 配置中心仍以通用版本为主：任务模板、审批规则、告警规则、资料要求、首页视图缺结构化编辑器和发布校验。
3. 甘特图仍偏任务时间线：维度、粒度、项目/人员/组织筛选、风险过滤和条目浮层未完整实现。
4. 任务详情已有执行闭环基线，分工新增/编辑、成果确认/退回、任务级验收资料规则配置和资料完整率已完成服务器验收；仍缺风险聚合、归档档案视图和更细的异常状态提示。
5. 权限数据范围仍需扩展到报表统计口径、通知脱敏、导出文件内容和更细粒度授权动作。

## 模块差距

### 01 组织与人员

- 已有：组织树、组织 CRUD、组织移动、人员列表、人员编辑、多组织归属字段、技能标签基础 CRUD、人员详情工作台。
- 差距：技能绑定、人员负载日历、人员项目归属编辑和历史记录仍缺完整闭环。
- 下一步：补人员技能绑定接口和人员负载日历交互。

### 02 权限

- 已有：角色 CRUD、角色动作、权限检查、隐藏项目授权、权限审计、菜单权限裁剪。
- 新增基线：`/data-scope-rules` 管理接口、`/permissions` 前端管理页，以及任务、项目、人员、资料、甘特、全局搜索的数据范围过滤和详情直连 403 校验。
- 差距：数据范围规则尚未覆盖报表统计、通知脱敏、导出内容和授权动作粒度。
- 下一步：将 `data_scope_rules` 接入报表/导出 SQL，并补隐藏项目通知脱敏与授权动作校验。

### 03/04 任务与派发协同

- 已有：五步任务向导、派发预览、负载预览、派发提交、协调单列表/详情、审批动作、会议记录，以及任务详情的时间调整、资料上传/关联、验收资料规则配置、资料完整率、分工新增/编辑、成果提交、成果确认/退回、进度填报、验收和日志聚合。
- 已验证：2026-06-19 在 `ssh nexusflow` 服务器 8089 以超级管理员完成分工新增、编辑、成果提交、退回、再次提交、确认；`GET /api/tasks/{id}` 可回读分工状态、成果资料、进度汇报、`assignment.created`、`assignment.updated`、`assignment.progress_reported`、`assignment.returned`、`assignment.confirmed` 变更日志和事件。
- 已验证：2026-06-19 在 `ssh nexusflow` 服务器 8089 以超级管理员配置任务级必需资料规则，缺资料提交验收返回 409；上传最终报告和阶段数据后资料完整率达到 100%，随后提交验收成功；`GET /api/tasks/{id}` 可回读缺失原因、完整率摘要、资料成果标记、`resource_requirements.updated` 变更日志和事件。
- 差距：模板加载、项目成员/技能筛选、冲突建议、推荐他人、后补填报确认、审批调整表单、批量分工和异常状态解释仍不足。
- 下一步：增强向导的候选人和冲突展示，再补批量分工、风险聚合、异常状态和归档档案视图。
