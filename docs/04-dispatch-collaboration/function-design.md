# 派发协同功能设计

## 1. 功能目标

派发协同模块负责控制任务如何从发起人传递到负责人和成员，尤其处理跨部门、跨项目、负载冲突和后补填报场景。

目标：

- 让有权限的派发可以快速完成。
- 让跨部门、跨层级派发自动进入协调。
- 让负责人能够在任务内部继续拆分工作。
- 让所有人员安排在正式生效前完成冲突检查。
- 保留每次派发、协调和确认记录。

## 2. 派发类型

### 2.1 直接派发

适用条件：

- 发起人有目标人员派发权限。
- 任务不涉及跨部门。
- 人员负载未超限。
- 不涉及隐藏项目授权。

流程：

```text
填写任务 -> 选择人员 -> 冲突检查通过 -> 负责人确认 -> 任务进行中
```

### 2.2 跨部门派发

触发条件：

- 目标成员主组织不在发起人派发范围内。
- 目标成员不属于任务关联项目。
- 项目负责人选择外部门人员参与项目相关任务。

流程：

```text
填写任务 -> 选择跨部门人员 -> 生成协调单 -> 部门审批 -> 项目负责人确认 -> 任务生效
```

### 2.3 后补填报

适用场景：

- 任务已口头启动。
- 发起人暂时无法填写完整信息。
- 负责人先补录任务内容。

流程：

```text
负责人补填 -> 发起人确认 -> 冲突检查 -> 必要审批 -> 任务生效
```

### 2.4 个人工作记录

适用场景：

- 个人学习。
- 自我安排工作。
- 休假。
- 简单出差记录。

流程：

```text
个人填写 -> 系统校验 -> 直属管理者可见 -> 参与负载或状态计算
```

## 3. 协调单

协调单是跨部门派发和冲突处理的载体。

### 3.1 协调单字段

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| 协调单编号 | 自动生成 | 全局唯一 |
| 关联任务 | 任务 | 必填 |
| 协调类型 | 枚举 | 跨部门、负载冲突、隐藏项目、强制安排 |
| 发起人 | 人员 | 提交协调的人 |
| 目标人员 | 人员多选 | 被协调人员 |
| 目标部门 | 组织 | 人员所属部门 |
| 任务周期 | 日期范围 | 计划占用时间 |
| 每日投入 | 数字/全天 | 计划占用 |
| 冲突分析 | 系统生成 | 展示已有占用和风险 |
| 审批人 | 人员 | 部门主任、副主任、技术监督等 |
| 审批状态 | 枚举 | 待审批、通过、拒绝、调整后通过 |
| 审批意见 | 文本 | 必填于拒绝或调整 |
| 研讨记录 | 文本/附件 | 可记录会议研讨结论 |

### 3.2 审批动作

审批人可选择：

- 同意。
- 拒绝。
- 调整投入后同意。
- 调整周期后同意。
- 推荐其他人员。
- 升级到上级协调。

拒绝或调整必须填写意见。

### 3.3 会议研讨记录

当技术总监、项目负责人或跨部门管理者需要线下研讨时，系统用协调单承接会议结论。

记录字段：

- 会议时间。
- 参会人。
- 研讨事项。
- 结论。
- 后续动作。
- 附件或纪要资料。

会议研讨记录不替代审批。最终是否生效仍以协调单审批结果为准。

## 4. 项目负责人确认

当任务关联项目时，项目负责人需要确认：

- 任务是否属于该项目范围。
- 成员分工是否合理。
- 时间节点是否匹配项目计划。
- 成果要求是否明确。

项目负责人不是资源归属审批人。涉及外部门人员时，仍需目标部门审批。

## 5. 派发生效规则

任务正式生效必须满足：

- 必填信息完整。
- 负责人确认。
- 所有跨部门审批通过。
- 负载冲突已处理或已强制安排。
- 隐藏项目授权完成。

生效后：

- 成员任务清单更新。
- 甘特图更新。
- 负载日历更新。
- 首页待办更新。
- 通知发送给相关人员。

## 6. 撤回与变更

### 6.1 撤回

允许撤回状态：

- 草稿。
- 待协调。
- 待确认。

撤回后：

- 协调单关闭。
- 待审批事项关闭。
- 任务回到草稿或取消。

### 6.2 生效后变更

任务生效后变更成员、时间、投入量时：

- 重新进行冲突检查。
- 如涉及跨部门，重新协调。
- 如影响任务截止时间，需负责人和发起人确认。
- 变更记录写入任务日志。
