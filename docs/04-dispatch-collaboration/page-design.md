# 派发协同页面设计

## 1. 页面清单

- 新建任务派发向导。
- 协调单列表页。
- 协调单详情页。
- 后补填报确认页。
- 派发记录页签。

## 2. 新建任务派发向导

派发向导和任务新建页共用，但强调协同检查。

### 2.1 步骤一：任务基本信息

展示：

- 任务类型。
- 任务名称。
- 任务周期。
- 任务概述。
- 成果要求。
- 关联项目。

系统行为：

- 根据任务类型加载模板。
- 根据发起人角色预设所属组织。
- 根据关联项目判断候选成员范围。

### 2.2 步骤二：选择负责人

负责人选择器展示：

- 姓名。
- 部门。
- 项目身份。
- 当前负载。
- 是否有权限担任负责人。

校验：

- 负责人必须可见该任务。
- 负责人不能处于离职状态。
- 负责人休假期间被选择时提示风险。

### 2.3 步骤三：选择成员与分工

成员表格字段：

- 成员。
- 部门。
- 成员角色。
- 分工内容。
- 起止时间。
- 每日投入。
- 是否跨部门。
- 负载状态。
- 审批状态。

交互：

- 支持批量导入成员。
- 支持按技能推荐人员。
- 支持从项目成员一键选择。
- 支持将某成员设置为负责人。

### 2.4 步骤四：冲突与协调

页面分区：

- 可直接生效人员。
- 需要跨部门审批人员。
- 存在负载冲突人员。
- 存在状态冲突人员。

每个问题项展示：

- 问题说明。
- 影响日期。
- 已有任务。
- 建议动作。

### 2.5 步骤五：确认发布

展示最终摘要：

- 任务信息。
- 负责人。
- 成员和分工。
- 协调事项。
- 风险项。
- 发布后影响。

按钮：

- 保存草稿。
- 提交协调。
- 直接发布。

## 3. 协调单列表页

### 3.1 默认视图

- 我发起的协调。
- 待我审批。
- 我部门相关。
- 已完成。

### 3.2 筛选项

- 协调类型。
- 审批状态。
- 目标部门。
- 目标人员。
- 任务类型。
- 时间范围。
- 是否超时。

### 3.3 列表字段

- 协调单编号。
- 任务名称。
- 协调类型。
- 目标人员。
- 目标部门。
- 占用时间。
- 每日投入。
- 审批人。
- 当前状态。
- 提交时间。

## 4. 协调单详情页

### 4.1 顶部摘要

- 任务名称。
- 发起人。
- 目标人员。
- 协调类型。
- 当前状态。

### 4.2 内容区

#### 任务信息

- 任务概述。
- 任务周期。
- 任务级别。
- 成果要求。

#### 人员占用分析

- 当前任务占用。
- 新任务占用。
- 超载日期。
- 已有重点任务。
- 休假、出差、培训信息。

#### 审批意见

- 审批动作。
- 调整建议。
- 意见文本。

### 4.3 审批按钮

- 同意。
- 拒绝。
- 调整后同意。
- 推荐他人。
- 升级协调。

## 5. 后补填报确认页

用于发起人确认负责人补填的任务。

确认项：

- 任务开始时间。
- 任务截止时间。
- 任务概述。
- 成果要求。
- 负责人。
- 成员。
- 分工。

发起人可：

- 确认。
- 修改后确认。
- 退回负责人补充。
- 取消任务。
