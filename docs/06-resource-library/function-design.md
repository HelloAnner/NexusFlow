# 资料库功能设计

## 1. 功能目标

资料库用于沉淀任务过程资料、阶段成果和最终交付物。资料不是单独的文件仓库，而是任务验收和归档闭环的一部分。

目标：

- 让每个任务有统一资料目录。
- 支持资料类型、版本、可见范围和成果标记。
- 支持任务验收前检查必需资料。
- 支持按任务、人员、组织、资料类型检索。
- 支持归档后只读，保证历史资料稳定。

## 2. 资料类型

默认资料类型：

- 个人报告。
- 整体报告。
- 汇报 PPT。
- 申报报告。
- 数据。
- Word 文档。
- PPT。
- PDF。
- Excel。
- 代码资料，预留。
- 其他。

资料类型由配置中心维护，可配置：

- 是否允许上传。
- 是否可作为阶段成果。
- 是否可作为最终成果。
- 哪些任务类型中必填。
- 默认可见范围。

## 3. 资料字段

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| 资料编号 | 自动生成 | 是 | 全局唯一 |
| 资料名称 | 文本 | 是 | 文件或资料标题 |
| 资料类型 | 枚举 | 是 | 来自配置中心 |
| 关联任务 | 任务 | 是 | 归属任务 |
| 关联分工 | 分工 | 否 | 归属具体分工 |
| 上传人 | 人员 | 是 | 当前版本上传人 |
| 版本号 | 文本 | 是 | v1、v2 等 |
| 文件格式 | 枚举 | 是 | docx、pptx、pdf 等 |
| 描述 | 文本 | 否 | 资料说明 |
| 可见范围 | 枚举/对象 | 是 | 继承任务或自定义 |
| 阶段成果 | 布尔 | 是 | 是否用于阶段确认 |
| 最终成果 | 布尔 | 是 | 是否用于最终验收 |
| 状态 | 枚举 | 是 | 草稿、已提交、已确认、已退回、已归档 |

## 4. 资料版本

版本规则：

- 同一资料可有多个版本。
- 上传新版本时保留旧版本。
- 默认展示最新版本。
- 验收时锁定验收版本。
- 归档后不可覆盖，只能补充新资料或申请重新打开。

版本记录包含：

- 版本号。
- 上传人。
- 上传时间。
- 变更说明。
- 文件大小。
- 审核状态。

## 5. 可见范围

资料默认继承任务可见范围。

可见范围可配置：

- 任务成员可见。
- 负责人和管理者可见。
- 指定人员可见。
- 指定组织可见。
- 隐藏项目授权人员可见。

限制：

- 资料可见范围不得大于任务可见范围。
- 隐藏项目资料不得出现在未授权用户搜索结果中。
- 归档资料下载需校验权限。

## 6. 与任务验收联动

任务模板可配置必需资料：

- 科研类：整体报告、数据、汇报 PPT。
- 文职类：文档、PPT、申报材料。
- 出差类：出差报告、会议纪要。
- 学习类：学习总结。
- 休假类：可不要求资料。

验收前校验：

- 必需资料是否已上传。
- 必需资料是否为最新版本。
- 资料是否已确认。
- 最终成果是否已标记。

缺失时，验收页展示缺失清单，并允许验收人退回补充。

## 7. 归档规则

任务归档时：

- 所有关联资料进入归档状态。
- 归档资料默认只读。
- 归档记录保存资料版本号。
- 后续补充资料需要负责人或管理员授权。
