# 甘特图与全局搜索功能设计

## 1. 功能目标

甘特图是任务排期和资源协调的核心视图。全局搜索用于快速定位任务、项目、人员和资料。两者都必须严格遵守同一套权限规则。

目标：

- 支持按任务、人员、部门、项目查看排期。
- 支持展示任务、分工、里程碑、人员占用、冲突和风险。
- 支持从甘特图直接跳转任务、项目、人员、冲突详情。
- 支持对象级全局搜索。
- 搜索结果按权限裁剪，隐藏项目和个人事项不得泄露。

## 2. 甘特图视图

### 2.1 视图维度

- 按任务：以任务为行，展示任务和分工时间线。
- 按人员：以人员为行，展示个人任务占用。
- 按部门：以部门为分组，展示部门任务和人员负载。
- 按项目：以项目为分组，展示项目任务和里程碑。

### 2.2 时间粒度

- 日。
- 周。
- 月。
- 季度。

默认：

- 员工：周视图，显示本人任务。
- 任务负责人：周或月视图，显示负责任务。
- 部门主任：月视图，显示本部门任务和人员。
- 中心领导：月或季度视图，显示中心任务和重点项目。

## 3. 甘特图数据对象

甘特条目类型：

- 项目条。
- 任务条。
- 分工条。
- 里程碑点。
- 人员占用条。
- 休假/出差/培训状态条。
- 冲突标记。
- 审批状态标记。

字段：

| 字段 | 说明 |
| --- | --- |
| id | 条目 ID |
| type | 项目、任务、分工、里程碑、人员状态等 |
| title | 展示名称，按权限脱敏 |
| start | 开始时间 |
| end | 截止时间 |
| progress | 进度 |
| status | 当前状态 |
| risk_level | 风险等级 |
| assignee_ids | 关联人员 |
| project_id | 关联项目 |
| task_id | 关联任务 |
| conflict_ids | 关联冲突 |
| readonly | 当前用户是否只读 |

## 4. 甘特图颜色和标记

颜色规则：

- 普通任务：默认色。
- 重点任务：高亮色。
- 紧急任务：强调色。
- 延期任务：红色。
- 冲突任务：红色描边或角标。
- 已完成任务：弱化展示。
- 已归档任务：灰色。
- 全天占用：灰色块。
- 休假：灰色斜纹。
- 出差/培训：蓝色或绿色状态条。

标记：

- 审批中。
- 跨部门。
- 隐藏项目。
- 资料缺失。
- 截止风险。
- 验收退回。

## 5. 甘特图交互

一期必做：

- 筛选时间范围。
- 切换视图维度。
- 点击条目进入详情。
- 鼠标悬停查看摘要。
- 展开/收起分组。
- 隐藏已完成。
- 只看风险。

二期可做：

- 拖拽调整任务时间。
- 拖拽调整分工时间。
- 批量调整时间。
- 导出图片或 PDF。

拖拽规则：

- 前端只提交变更意图。
- 后端重新校验权限、负载、审批和状态。
- 若产生冲突，进入协调流程，不直接生效。

## 6. 甘特图权限

权限规则：

- 列表查询阶段即裁剪不可见数据。
- 隐藏项目未授权时，不返回标题、时间和摘要。
- 个人休假只对本人和直属管理者可见。
- 部门主任只能看到本部门和授权范围数据。
- 项目负责人只能看到所属项目和关联任务。

脱敏规则：

- 可见对象正常展示。
- 有占用但无详情权限时，可展示“已占用”但不展示任务名称。
- 隐藏项目未授权时直接过滤，不展示占用来源。

## 7. 甘特图后端实现

查询流程：

1. 根据用户生成数据范围。
2. 根据视图维度和时间范围查询任务、分工、人员状态、里程碑。
3. 合并冲突和风险记录。
4. 做权限脱敏。
5. 生成甘特条目。
6. 写入短期 Redis 缓存。

缓存键应包含：

- 用户 ID。
- 权限版本。
- 视图维度。
- 时间范围。
- 筛选条件。

## 8. 全局搜索范围

搜索对象：

- 任务。
- 项目。
- 人员。
- 资料。

字段：

- 任务：任务编号、名称、概述、负责人、成员、任务类型。
- 项目：项目编号、名称、负责人、成员、项目类型。
- 人员：姓名、账号、组织、项目、技能标签。
- 资料：资料名称、资料类型、任务、项目、上传人。

一期搜索后端使用 PostgreSQL 全文检索。后续如数据量或相关性要求提高，可通过 `SEARCH_BACKEND` 切换到 Meilisearch 或 OpenSearch。

## 9. 搜索结果

结果字段：

- 对象类型。
- 标题。
- 摘要。
- 状态。
- 归属组织。
- 关联项目。
- 更新时间。
- 跳转地址。
- 权限提示。

排序：

- 精确匹配优先。
- 标题匹配优先。
- 最近更新优先。
- 风险和待办对象可按角色加权。

## 10. 搜索权限

规则：

- 搜索必须先确定用户数据范围。
- 未授权隐藏项目对象不进入结果。
- 个人事项未授权不进入结果。
- 资料结果必须同时满足资料可见性和关联任务/项目可见性。
- 无权限对象不展示标题和摘要。

## 11. 搜索索引更新

触发时机：

- 创建任务。
- 更新任务名称、概述、负责人、成员。
- 创建或更新项目。
- 更新人员信息、组织、技能标签。
- 上传或更新资料。
- 修改隐藏项目授权。
- 权限规则发布。

实现方式：

- 一期直接更新 PostgreSQL 搜索向量字段。
- 大批量重建索引通过后端管理命令触发。
- 权限规则变更不必重建全文索引，但查询时必须重新裁剪。

## 12. 与其他模块联动

- 首页：甘特摘要和全局搜索入口。
- 任务：任务详情页甘特页签复用甘特查询能力。
- 项目：项目详情页甘特页签按项目过滤。
- 人员：人员详情页展示个人负载甘特。
- 冲突中心：冲突记录可在甘特图中定位。
- 报表：甘特维度和报表指标使用一致的任务、分工和负载口径。
