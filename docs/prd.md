# 工作管理系统 PRD

## 1. 背景与目标

### 1.1 背景

中心内部工作包含科研、市场、出差、文职、党建、学习、休假等多种类型。当前工作协同的核心痛点包括：

- 人员组织归属、项目归属、角色权限和技能能力分散，任务发起时难以快速找到合适人员。
- 任务既包含正式项目，也包含临时工作、个人学习、出差和休假，缺少统一的排期、占用和状态视图。
- 项目负责人、项目成员、项目身份和负责工作需要形成独立项目口径，不能只散落在人员和任务字段中。
- 新用户需要通过邀请链接快速注册，不同角色进入系统后应直接看到对应工作台。
- 除中心主任外，跨部门、跨层级派发需要授权和协调，缺少清晰流程留痕。
- 任务拆分、阶段成果、资料沉淀和验收闭环不完整，任务执行过程难追踪。
- 人员工作冲突依赖人工判断，不能根据时间、投入量和优先级提前预警。
- 不同角色关注的信息不同，需要首页、任务页、人员页、配置页联动，形成顺畅工作流。

### 1.2 产品目标

建设一套面向中心内部的工作管理系统，实现：

- 统一人员、组织、项目、任务、资料、工具入口。
- 支持从任务发起、人员协调、任务拆分、执行汇报、风险预警、成果验收到归档的完整闭环。
- 用角色、组织归属、项目归属和隐藏项目规则控制可见范围与操作权限。
- 用甘特图、任务清单、人员负载视图提升排期和协调效率。
- 用配置中心统一维护组织层级、角色权限、任务模板、字段、流程和告警规则，保证后续可扩展。
- 采用 Rust 后端、TypeScript + Bun 前端，并以单二进制方式部署，降低内部环境交付复杂度。
- 通过 SA 超级管理员后台完成系统初始化、邀请注册、人员全量配置、角色入口配置和账号安全管理。

### 1.3 用户角色

- SA 超级管理员：维护全量组织、人员、账号、邀请、权限、配置和审计，不默认参与业务审批。
- 中心主任/书记：查看中心全量信息，具备全员工作派发与最终协调权限。
- 中心副主任/技术总监：查看和管理分管部门、分管项目范围内信息；可在权限范围内派发任务。
- 部门主任/技术监督：查看所属部门信息；负责本部门人员协调、资源协调和项目监督。
- 部门副主任：协助部门主任管理所属部门信息和人员工作安排。
- 项目负责人/任务负责人：管理所属项目或任务，负责人员分工、进度把控、阶段成果和验收提交。
- 项目成员/员工：查看本人参与项目和个人任务，填报进度、阶段成果和资料。
- 系统管理员：维护组织、角色、权限、字段、流程、标签、基础字典和系统参数。
- 待审核用户：通过邀请注册但尚未审核，只能完善资料和查看审核状态。

## 2. 产品范围

### 2.1 一期范围

- 人员与组织管理。
- SA 后台、账号登录、邀请注册和角色默认入口。
- 角色权限与数据范围控制。
- 任务创建、派发、审批、拆分、执行、汇报、验收、归档。
- 甘特图、任务清单、人员负载和冲突预警。
- 项目管理、项目成员、项目授权和项目归档。
- 全局搜索。
- 资料库与任务资料管理。
- 首页工作台。
- 基础配置中心。
- 智能工具台入口和工具分类管理。

### 2.2 暂缓范围

- 人员能力画像评分体系的复杂模型。
- 代码仓库管理与代码资料解析。
- 论文、专利自动撰写等智能体的深度业务编排。
- 与外部 OA、IM、日历系统的深度集成。
- 容器化部署编排。系统不以容器作为运行前提，只要求 `.env` 中的基础组件连接可用。

## 3. 核心业务对象

### 3.1 账号与邀请对象

账号是登录身份，人员是业务身份。一个账号绑定一个人员档案，人员档案承接组织、角色、项目和任务关系。

账号字段：

| 字段 | 说明 | 必填 |
| --- | --- | --- |
| 登录账号 | 用户登录标识 | 是 |
| 绑定人员 | 对应人员档案 | 是 |
| 账号状态 | 待审核、启用、禁用、锁定 | 是 |
| 最近登录时间 | 安全审计 | 否 |
| 失败登录次数 | 安全策略 | 是 |

邀请模板字段：

| 字段 | 说明 | 必填 |
| --- | --- | --- |
| 邀请名称 | 后台识别名称 | 是 |
| 默认组织 | 注册后自动加入的主组织 | 是 |
| 默认角色 | 注册后自动获得的系统角色 | 是 |
| 默认项目 | 注册后自动加入的项目 | 否 |
| 默认项目身份 | 负责人、核心成员等 | 否 |
| 是否需要审核 | 注册后是否待审核 | 是 |
| 有效期 | 链接失效时间 | 是 |
| 最大使用次数 | 单人或批量邀请 | 是 |

邀请注册规则：

- 不同角色可生成不同邀请链接。
- 注册后自动绑定邀请模板中的组织、角色、项目、项目身份和负责工作。
- 需要审核时，用户进入待审核状态，不可访问业务数据。
- 邀请链接 token 只存哈希，完整链接只在生成时展示一次。
- 邀请生成、使用、停用、审核都必须审计。

### 3.2 组织对象

组织结构为树形层级：

- 公司
- 二级单位
- 三级单位/中心
- 部门
- 创新工作室

组织配置要求：

- 公司、二级单位、三级单位、部门为主组织路径。
- 创新工作室为可选多归属，一个人员可属于多个工作室。
- 每个组织节点可配置负责人、副负责人、技术监督、可见范围和默认审批人。
- 组织调整需要保留历史记录，历史任务仍按任务创建时的组织快照展示。

### 3.3 人员对象

人员基础字段：

| 字段 | 说明 | 必填 |
| --- | --- | --- |
| 姓名 | 人员姓名 | 是 |
| 工号/账号 | 登录和唯一识别 | 是 |
| 主组织 | 公司-二级单位-三级单位-部门 | 是 |
| 创新工作室 | 可多选 | 否 |
| 人员等级 | 管理等级或专业等级 | 是 |
| 系统角色 | 决定操作权限 | 是 |
| 项目归属 | 当前参与项目，可多选 | 否 |
| 技能标签 | 研发、文职、需求对接等标签 | 否 |
| 工作状态 | 在岗、出差、培训、休假、离职等 | 是 |
| 每日标准工时 | 默认用于负载计算 | 是 |
| 账号状态 | 待审核、启用、禁用、锁定 | 是 |
| 邀请来源 | 注册来源 | 否 |
| 默认首页角色 | 多角色时的默认入口 | 否 |

技能标签分组：

- 研发类：前端、后端、Web 端、手机端、PC 端、运维、Python、Java、Go、SQL 等。
- 文职类：PPT 制作、报告撰写、论文、专利、申报材料、党建材料等。
- 需求对接类：技术支持、系统培训、需求收集、运维客服等。

SA 超级管理员可配置人员全部信息，包括基础信息、账号安全、组织归属、角色权限、项目归属、技能标签、工作状态、工时、默认首页、扩展授权和审计备注。

### 3.4 项目对象

项目用于承载人员项目归属、项目负责人、项目身份、项目任务、项目资料和项目维度统计。

项目字段：

| 字段 | 说明 | 必填 |
| --- | --- | --- |
| 项目编号 | 全局唯一编号 | 是 |
| 项目名称 | 项目展示名称 | 是 |
| 项目类型 | 科研、市场、内部专项、创新工作室项目等 | 是 |
| 项目级别 | 公司级、处级、中心级等 | 否 |
| 归属组织 | 项目统计和管理口径 | 是 |
| 项目负责人 | 项目总体负责人 | 是 |
| 分管领导 | 项目管理和审批口径 | 否 |
| 项目成员 | 当前有效成员 | 否 |
| 项目状态 | 筹备、进行中、暂停、完成、归档、取消 | 是 |
| 可见性 | 普通、隐藏项目、指定范围 | 是 |
| 开始日期 | 项目计划开始 | 否 |
| 结束日期 | 项目计划结束 | 否 |
| 项目概述 | 背景、目标、范围 | 否 |

项目成员字段：

- 人员。
- 项目身份：负责人、核心成员、参与成员、支持人员。
- 负责工作。
- 加入日期。
- 退出日期。
- 当前有效。

项目规则：

- 项目是任务归属和统计口径，不替代任务。
- 一个任务最多关联一个主项目，也可以不关联项目。
- 项目成员不等于任务成员，只有加入任务或分工后才占用工时。
- 项目负责人可以发起项目相关任务，但不能绕过外部门资源审批。
- 隐藏项目必须显式授权，未授权用户在列表、搜索、甘特图、报表、资料中都不可见。

任务类型：

- 文职。
- 党建。
- 安全。
- 学习。
- 休假。
- 其他自定义类型。

任务主字段：

| 字段 | 说明 | 必填 |
| --- | --- | --- |

| 任务类型 | 来自任务类型字典 | 是 |
| 任务级别 | 普通、重点、紧急等 | 是 |
| 发起人 | 任务提出或派发人员 | 是 |
| 负责人 | 任务总体负责人 | 是 |
| 参与成员 | 可多选，需配置分工 | 是 |
| 所属组织 | 任务归属部门或中心 | 是 |
| 关联项目 | 可关联已有项目或创建为独立任务 | 否 |
| 开始时间 | 计划开始时间 | 是 |
| 截止时间 | 计划截止时间 | 是 |
| 每日投入 | 按人员配置预计每日投入小时或全天 | 是 |

| 可见性 | 普通、隐藏项目、指定范围 | 是 |

### 3.6 分工对象

一个任务下可拆分多个分工，分工面向具体人员或小组。

分工字段：

- 分工名称。
- 责任人。
- 协作人。
- 分工类型：研发、文职、客服、培训、汇报、材料、需求对接等。

- 阶段成果要求。

- 汇报频率。
- 关联资料。
- 验收人。

资料类型：

- 汇报 PPT。
- 申报报告。

资料字段：

- 资料类型。

## 4. 权限与数据范围

### 4.1 权限原则

- 权限由“系统角色 + 组织归属 + 项目归属 + 任务可见性”共同决定。
- 隐藏项目只对显式授权人员可见。
- 除中心主任/书记外，任何跨部门或跨层级派发都必须进入协调审批流程。
- 任务执行数据默认对任务成员、负责人、相关管理者可见。
- 个人休假、学习等任务仅对本人和具备管理权限的直属管理者可见。

### 4.2 角色权限矩阵

| 角色 | 查看范围 | 派发范围 | 审批/协调 | 配置权限 |
| --- | --- | --- | --- | --- |
| SA 超级管理员 | 系统全量后台、账号、配置、审计；敏感业务访问需审计 | 不参与业务派发 | 不参与业务审批 | 全局最高配置、人员全量维护、邀请注册、账号安全 |
| 中心主任/书记 | 中心全量信息 | 全员 | 可最终协调全部冲突 | 可配置中心级规则 |
| 中心副主任/技术总监 | 分管部门/项目，不含未授权隐藏项目 | 分管部门/项目人员 | 可审批分管范围协调 | 可维护分管项目配置 |
| 部门主任/技术监督 | 所属部门，不含未授权隐藏项目 | 所属部门人员 | 可审批本部门人员占用 | 可维护部门标签和人员信息 |
| 部门副主任 | 所属部门，不含未授权隐藏项目 | 所属部门人员 | 可按授权审批 | 可维护授权范围内信息 |
| 项目负责人 | 所属项目，不含未授权隐藏项目 | 所属项目成员，仅限项目相关工作 | 可发起人员协调 | 可维护项目任务和资料 |
| 员工 | 本人任务和所属项目 | 个人工作记录 | 不可审批 | 仅维护个人进度和资料 |
| 系统管理员 | 系统全量配置 | 不参与业务派发 | 不参与业务审批 | 全局配置 |
| 待审核用户 | 无业务数据 | 不可派发 | 不可审批 | 仅完善注册资料 |

### 4.3 跨部门派发规则

跨部门派发必须满足：

1. 任务发起人填写任务基本信息和用人需求。
2. 系统识别候选成员所属部门与任务发起人权限范围。
3. 若存在跨部门人员，自动生成协调审批。
4. 该人员所属部门的部门主任、副主任或技术监督至少一人审批。
5. 如任务属于项目工作，还需项目负责人确认任务内容和分工边界。
6. 审批通过后，成员才正式加入任务并占用工时。
7. 审批拒绝时，发起人需调整人员、时间或任务方案。

## 5. 任务生命周期

### 5.1 总体状态

任务状态流：

```text
草稿 -> 待协调 -> 待确认 -> 进行中 -> 待验收 -> 已完成 -> 已归档
                  |        |        |
                  |        |        -> 验收退回 -> 进行中
                  |        -> 已暂停/有风险
                  -> 已取消
```

状态说明：

- 草稿：任务信息未提交，可由创建人编辑。
- 待协调：涉及跨部门、跨项目或负载冲突，需要管理者确认。
- 待确认：人员、时间、内容已确定，等待负责人或发起人最终确认。
- 进行中：任务正式生效，成员开始填报进度和成果。
- 已暂停：因资源、时间或外部条件暂停，需填写暂停原因。
- 有风险：系统或人工标记存在延期、负载、成果风险。
- 待验收：负责人提交阶段或最终成果，等待验收人确认。
- 验收退回：验收人要求补充或修改。
- 已完成：验收通过。
- 已归档：任务资料、报告和过程记录归档。
- 已取消：任务不再执行，保留取消原因和审批记录。

### 5.2 任务概述变更留痕

任务概述、时间、负责人、成员、投入量、交付物发生变化时必须记录版本：

- 变更前内容。
- 变更后内容。
- 变更原因。
- 变更发起人。
- 变更确认人。
- 变更时间。

任务详情页展示当前版本，并提供历史版本时间线，例如“第 1 次任务概述”“第 2 次任务概述”。

### 5.3 分工状态

分工状态流：

```text
未开始 -> 进行中 -> 待确认 -> 已完成
          |        |
          |        -> 退回修改 -> 进行中
          -> 阻塞/延期
```

员工只维护自己的分工进度、阶段成果、投入情况和阻塞说明；任务负责人负责确认分工成果并汇总任务状态。

## 6. 核心工作流

### 6.1 人员建档与维护流程

1. 系统管理员维护组织树和基础角色。
2. 部门主任或管理员创建人员档案。
3. 配置主组织、工作室、人员等级、系统角色和技能标签。
4. 项目负责人或部门主任维护人员项目归属。
5. 系统根据组织和角色自动生成数据可见范围。
6. 人员调整部门或角色时，系统保留历史归属，不影响历史任务展示。

### 6.2 标准任务派发流程

1. 任务发起人在“新建任务”页填写任务名称、类型、时间、概述、负责人和用人需求。
2. 系统根据发起人权限判断是否可直接派发。
3. 系统根据成员时间、每日投入和已有任务计算负载。
4. 若无跨部门和负载冲突，任务进入待确认。
5. 负责人确认任务内容和人员安排。
6. 任务进入进行中，首页、甘特图、人员负载和成员任务清单同步更新。
7. 成员按分工填报进度和成果。
8. 负责人汇总并提交验收。
9. 发起人或指定验收人确认完成。
10. 资料和过程记录归档。

### 6.3 后补任务填报流程

适用于发起人暂时无时间录入完整信息的场景：

1. 发起人创建简要任务或口头确认任务启动。
2. 项目/任务负责人补填任务信息。
3. 系统标记为“待发起人确认”。
4. 发起人重点确认任务时间、任务概述、交付要求和参与成员。
5. 确认后任务正式生效。
6. 若发起人修改内容，系统记录变更版本。

### 6.4 跨部门人员协调流程

1. 发起人选择跨部门人员或填写跨部门技能需求。
2. 系统创建协调单，包含任务信息、人员需求、时间占用、冲突分析。
3. 候选人员所属部门主任、副主任或技术监督审批。
4. 如审批人认为负载过高，可选择：
   - 同意加入。
   - 调整投入时间。
   - 调整任务周期。
   - 推荐其他人员。
   - 拒绝并填写原因。
5. 审批通过后，项目负责人确认分工。
6. 任务正式生效并占用人员工时。

### 6.5 任务内部派发流程

1. 任务负责人进入任务详情页，打开“分工”页签。
2. 为成员创建分工，配置分工类型、时间节点、每日投入和成果要求。
3. 系统再次校验每个成员的负载。
4. 成员收到分工通知，在个人任务清单中查看。
5. 成员按计划填报进度、阶段成果和资料。
6. 负责人查看任务看板和甘特图，处理阻塞和延期。

### 6.6 任务冲突协调流程

触发条件：

- 同一人员同一日期的预计投入超过每日标准工时。
- 重点任务设置“全天”投入，人员在该周期内仍被安排其他任务。
- 截止时间按当前投入量预计无法完成。
- 人员状态为休假、出差或培训，但仍被分配不兼容任务。

处理流程：

1. 系统在任务创建、人员变更、分工调整时实时计算冲突。
2. 冲突以红色风险标记展示在人员选择器、甘特图、任务详情和冲突中心。
3. 发起人或负责人提交协调请求。
4. 管理者选择调整人员、调整时间、降低投入、提升优先级或强制安排。
5. 强制安排必须填写原因，并通知受影响任务负责人。
6. 处理结果回写任务时间线和人员负载日历。

### 6.7 阶段成果与验收流程

1. 成员在分工中提交阶段成果，可关联资料。
2. 负责人确认成员成果，必要时退回修改。
3. 负责人汇总整体报告、PPT、数据等最终成果。
4. 发起人或验收人进行验收。
5. 验收通过后任务完成；未通过则退回进行中。
6. 完成后系统提示归档资料，形成任务档案。

## 7. 功能模块

### 7.1 首页工作台

首页根据角色展示不同内容，但交互结构保持一致，避免用户切换角色后体验断裂。

通用组件：

- 我的待办：待确认、待审批、待填报、待验收事项。
- 我的任务：本人参与或负责的任务清单。
- 甘特图入口：按权限展示对应范围任务。
- 风险提醒：延期、冲突、待审批超时、成果未提交。
- 快捷入口：新建任务、人员查询、资料库、工具台。

角色差异：

- 员工：首页重点展示个人分工、今日任务、阶段成果待提交、个人甘特图。
- 项目/任务负责人：首页重点展示负责任务、成员进度、风险分工、待确认成果。
- 部门主任：首页重点展示部门人员负载、部门任务甘特图、跨部门协调审批。
- 中心领导：首页重点展示中心任务总览、重点任务、风险排行、跨部门资源协调。

### 7.2 人员管理

功能：

- 人员列表：支持按组织、角色、技能、项目、状态筛选。
- 人员详情：展示组织归属、项目归属、任务负载、技能标签、历史任务。
- 人员编辑：维护基础信息、标签、角色、项目归属。
- 负载日历：展示人员按天的任务占用和可用容量。
- 技能检索：按技能组合查找人员，用于任务派发时选择候选人。

关键联动：

- 新建任务选择人员时复用人员管理中的技能和负载数据。
- 人员详情可跳转到该人员参与任务、分工和资料。
- 组织和角色变更后，权限立即重算，但历史记录保留原快照。

### 7.3 项目管理

功能：

- 项目列表：按类型、级别、状态、负责人、归属组织、风险筛选。
- 项目详情：展示项目概览、成员、任务、甘特图、资料、风险、授权和日志。
- 项目成员：维护项目身份、负责工作、加入退出时间。
- 隐藏项目授权：维护人员、组织、角色的查看和协作权限。
- 项目归档：归档项目资料、任务清单、成员历史和甘特快照。

关键联动：

- 人员详情展示项目归属和负责工作。
- 任务创建时可关联项目，并按项目可见性、项目成员和项目负责人规则校验。
- 项目甘特图按项目任务、分工、里程碑和成员负载生成。
- 项目报表统计必须遵守项目授权。

### 7.4 任务管理

功能：

- 任务列表：支持按类型、状态、负责人、成员、组织、项目、时间、风险筛选。
- 任务详情：集中展示任务概述、状态、成员、分工、甘特图、资料、日志、审批。
- 新建任务：支持标准创建、后补填报、从模板创建。
- 甘特图：按任务、分工、人员、组织维度展示排期。
- 阶段汇报：成员填报进度和成果，负责人审核。
- 任务变更：支持变更时间、人员、概述、交付物，并记录版本。
- 任务归档：完成后归档资料和全过程记录。

任务详情页签建议：

- 概览：任务目标、负责人、状态、时间、风险。
- 分工：成员分工、进度、成果和验收。
- 甘特图：任务与分工时间轴。
- 资料：资料库、版本、交付物。
- 审批：协调单、确认记录。
- 日志：变更历史、评论、状态流转。

### 7.5 任务派发

任务派发可作为独立向导，也可从任务详情中发起。

派发步骤：

1. 基本信息：任务类型、名称、概述、时间、优先级。
2. 负责人：选择任务负责人，校验其权限和负载。
3. 成员需求：按人员、组织或技能选择成员。
4. 分工草案：配置每个成员的工作内容、时间节点和投入。
5. 冲突检查：展示负载、跨部门、隐藏项目和休假冲突。
6. 协调审批：自动生成需要审批的事项。
7. 确认发布：负责人和发起人确认后生效。

### 7.6 冲突与负载管理

功能：

- 人员负载日历：按天展示工作占用。
- 冲突中心：集中展示未处理冲突。
- 负载计算：以每日标准工时、任务投入、全天占用和人员状态计算。
- 重点任务锁定：选择全天后，人员在周期内置灰，不推荐接受其他任务。
- 协调记录：保留每次调整和强制安排原因。

负载计算规则：

- 默认每日标准工时为 8 小时，可在配置中心调整。
- 任务投入支持小时数或全天。
- 同一天投入总和超过标准工时即预警。
- 全天任务与任意其他任务重叠即冲突。
- 管理者可强制安排，但必须记录原因。
- 截止风险按剩余工时、截止日前可用工时和预测完成日计算。

### 7.7 资料库

功能：

- 按任务自动生成资料目录。
- 支持上传、下载、版本管理、资料类型标注。
- 支持将资料设置为阶段成果或最终成果。
- 支持按任务、人员、资料类型、时间检索。
- 支持资料可见范围配置。

资料与任务联动：

- 分工提交成果时可直接选择或上传资料。
- 任务验收时必须检查必需资料是否齐全。
- 任务归档后资料进入只读状态，管理员或负责人可重新打开补充。

### 7.8 甘特图与全局搜索

甘特图功能：

- 按任务、人员、部门、项目四种维度展示。
- 支持日、周、月、季度粒度。
- 展示项目、任务、分工、里程碑、人员状态、风险和冲突。
- 支持点击跳转项目、任务、人员、冲突详情。
- 一期不要求拖拽变更，后续如支持拖拽，必须由后端重新校验权限、负载和审批。

全局搜索功能：

- 搜任务：任务编号、名称、概述、负责人、成员。
- 搜项目：项目编号、名称、负责人、成员。
- 搜人员：姓名、组织、技能、项目。
- 搜资料：资料名称、类型、任务、项目、上传人。

搜索规则：

- 一期使用 PostgreSQL 全文检索。
- 搜索结果必须按权限裁剪。
- 未授权隐藏项目不展示任何标题、摘要和时间。

### 7.9 智能工具台

工具台定位为工作效率入口，分为常用工具和智能体工具。

常用工具：

- PDF 格式转换。
- 文档格式转换。
- 翻译。
- 表格处理。
- 图片压缩或识别。

智能体工具：

- 论文辅助撰写。
- 专利辅助撰写。
- 个人知识库。
- 平台问答助手。
- 汇报材料辅助。

工具配置：

- 工具名称。
- 工具分类。
- 入口地址或内嵌方式。
- 可见角色。
- 可见组织。
- 是否推荐到首页快捷入口。

### 7.10 配置中心

配置中心用于保证全局体验一致，避免每个页面单独写死规则。

配置项：

- 组织层级：公司、二级单位、三级单位、部门、工作室。
- 角色权限：角色、数据范围、操作权限、审批权限。
- 项目配置：项目类型、级别、状态、项目成员身份、隐藏项目授权规则。
- 任务类型：类型、子类型、默认字段、默认流程、资料要求。
- 任务模板：不同任务类型的表单字段、里程碑模板、分工模板。
- 技能标签：标签分组、标签名称、是否启用。
- 人员等级：管理等级、专业等级。
- 审批规则：跨部门、隐藏项目、强制安排、任务变更。
- 告警规则：负载超限、延期、审批超时、成果未提交。
- 资料类型：资料分类、必填规则、可见范围。
- 首页配置：不同角色首页组件和默认筛选范围。
- 甘特图配置：颜色规则、时间粒度、默认视图。

### 7.11 技术架构

技术栈：

- 后端：Rust、Axum、Tokio、SQLx。
- 前端：TypeScript、Bun、React。
- 数据库：PostgreSQL。
- 缓存和轻量队列：Redis。
- 对象存储：S3 兼容对象存储，默认可接 MinIO。
- 搜索：一期使用 PostgreSQL 全文检索。

部署形态：

- 最终交付单二进制。
- 默认端口 `8089`。
- 后端二进制同时提供 API 和前端静态资源。
- 不使用 Docker 作为部署前提。
- `.env` 只配置基础组件地址、凭据和运行参数。

`.env` 关键配置：

- `APP_HOST`。
- `APP_PORT`，默认 `8089`。
- `DATABASE_URL`。
- `REDIS_URL`。
- `S3_ENDPOINT`。
- `S3_BUCKET`。
- `S3_ACCESS_KEY`。
- `S3_SECRET_KEY`。
- `SESSION_SECRET`。
- `JWT_SECRET`。

### 7.12 SA 后台与邀请注册

功能：

- 登录和账号会话。
- SA 后台首页。
- 邀请模板管理。
- 邀请链接生成、复制、停用和使用记录。
- 邀请注册。
- 待审核注册处理。
- 人员全量编辑。
- 账号禁用、解锁、重置密码和强制下线。
- 角色默认入口配置。
- 审计日志。

邀请注册流程：

```text
生成邀请链接 -> 用户打开链接 -> 填写注册信息 -> 自动绑定组织/角色/项目 -> 直接启用或进入待审核 -> 进入对应角色首页
```

角色入口规则：

- SA 超级管理员默认进入 SA 后台首页。
- 待审核用户只能进入资料完善和审核状态页。
- 中心主任、部门主任、项目负责人、任务负责人、员工分别进入对应工作台。
- 多角色用户可配置默认角色视角。

人员全量配置：

- SA 可维护人员全部字段。
- 普通管理员只能维护授权范围内字段。
- 修改组织、角色、隐藏项目授权、账号状态必须填写原因并写入审计。

- 快捷入口按钮。

首页体验要求：

- 涉及跨部门时即时提示需要审批。

人员详情页：

- 基础信息。
- 技能标签。
- 当前任务。
- 负载日历。
- 历史任务。
- 所属项目。
- 资料贡献。

### 8.8 甘特图页配置

视图模式：

- 按任务。
- 按人员。
- 按部门。
- 按项目。

时间粒度：

- 日。
- 周。
- 月。

颜色规则：

- 普通任务：默认色。
- 重点任务：高亮色。
- 延期任务：红色。
- 冲突任务：红色描边或冲突标记。
- 已完成任务：弱化展示。

交互：

- 点击任务条进入任务详情。
- 点击人员行进入人员负载详情。
- 支持按角色默认加载不同范围。
- 支持隐藏已完成任务。

### 8.9 全局搜索配置

搜索入口：

- 顶部导航。
- 首页。
- 任务、项目、人员、资料选择器。

搜索结果类型：

- 任务。
- 项目。
- 人员。
- 资料。

搜索结果必须展示对象类型、标题、状态、摘要、归属组织、关联项目和跳转入口。未授权对象必须过滤。

### 8.10 冲突中心配置

冲突中心字段：

- 冲突类型。
- 涉及人员。
- 涉及任务。
- 冲突时间。
- 超载小时数。
- 责任处理人。
- 当前状态。
- 推荐处理动作。

处理动作：

- 调整人员。
- 调整时间。
- 调整投入。
- 升级审批。
- 强制安排。
- 标记已解决。

### 8.11 配置中心页面配置

配置中心应分模块管理：

- 组织与人员。
- 角色与权限。
- 任务类型与模板。
- 工作流与审批。
- 告警规则。
- 标签字典。
- 资料类型。
- 首页与视图。
- 工具台。

配置变更要求：

- 重要配置变更需记录操作日志。
- 权限规则变更需即时生效。
- 任务模板变更只影响新建任务，不影响历史任务。
- 字典项停用后历史数据仍可展示。

## 9. 通知与待办

### 9.1 通知场景

- 被分配为任务负责人。
- 被加入任务成员。
- 分工创建或变更。
- 跨部门协调待审批。
- 任务即将截止。
- 分工延期或阻塞。
- 阶段成果待确认。
- 验收通过或退回。
- 任务概述或时间发生变更。
- 强制安排影响已有任务。

### 9.2 待办规则

待办必须能跳转到具体处理位置：

- 审批待办跳转到审批页签。
- 进度填报待办跳转到分工填报区。
- 验收待办跳转到成果区。
- 冲突待办跳转到冲突中心。

## 10. 数据与报表

### 10.1 基础报表

- 中心任务总览。
- 部门任务总览。
- 人员负载排行。
- 延期任务清单。
- 重点任务进度。
- 跨部门协调统计。
- 资料归档完整率。

### 10.2 指标口径

- 任务进度：可由负责人手动维护，也可按分工进度加权计算。
- 人员负载：某日期所有任务预计投入总和 / 每日标准工时。
- 延期：当前日期超过截止时间且任务未完成。
- 冲突：投入超限、全天任务重叠、状态不兼容。
- 资料完整率：已上传必需资料数量 / 任务模板要求资料数量。

## 11. 非功能要求

### 11.1 易用性

- 同一业务对象在不同页面展示字段一致。
- 列表筛选、排序和列配置支持记忆。
- 关键操作提供清晰状态反馈。
- 风险信息应在创建、列表、详情、甘特图、首页同时可见。

### 11.2 可配置性

- 任务类型、字段、模板、审批规则、告警规则可配置。
- 角色首页和列表默认视图可配置。
- 标签和资料类型可扩展。

### 11.3 可追溯性

- 任务创建、变更、审批、汇报、验收、归档均需留痕。
- 任务概述变更必须保留历史版本。
- 强制安排和跨部门协调必须保留审批记录。

### 11.4 安全性

- 严格按角色和数据范围控制访问。
- 隐藏项目不得出现在未授权人员的搜索、列表、报表和通知中。
- 资料下载需受任务和资料可见范围控制。

## 12. MVP 交付建议

### 12.1 第一阶段：基础闭环

- 组织与人员管理。
- 角色权限。
- 任务创建、派发、分工、进度填报。
- 任务列表、详情、个人首页。
- 基础甘特图。

### 12.2 第二阶段：协同与风险

- 冲突中心。
- 阶段成果、验收和归档。

### 12.3 第三阶段：配置与扩展

- 任务模板和字段配置。
- 告警规则配置。
- 邀请注册和角色入口配置。
- 资料库增强。
- 项目归档增强。
- 工具台。
- 报表和人员能力画像基础能力。

## 13. 验收标准

- 员工可在首页看到本人任务、分工、截止时间和待提交成果。
- 新用户可通过邀请链接注册，并自动进入对应组织、角色和项目。
- 待审核用户只能看到资料完善和审核状态。
- SA 超级管理员可在后台配置人员全部信息、邀请链接、角色入口和账号安全。
- 项目/任务负责人可完成任务创建、成员分工、进度跟踪、成果确认和验收提交。
- 项目负责人可维护项目成员、项目身份、负责工作和项目资料。
- 部门主任可查看本部门人员负载，并处理跨部门协调。
- 中心领导可查看中心级甘特图、重点任务和风险任务。
- 跨部门派发必须触发审批，审批通过前不占用正式任务成员名额。
- 人员负载超限、全天任务重叠、休假冲突必须有预警。
- 任务概述、时间、成员、投入量变更必须留痕。
- 资料可按任务归档，最终验收前可校验必需资料。
- 配置中心可维护角色、任务类型、模板、标签、告警和首页视图。
- 隐藏项目对未授权用户不可见。
- 系统可通过单二进制启动，默认端口为 `8089`，基础组件通过 `.env` 连接。
