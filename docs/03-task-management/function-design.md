# 任务管理功能设计

## 1. 功能目标

任务管理是系统主干。系统中的任务覆盖科研、市场、出差、培训、会议交流、技术支持、文职、党建、安全、学习、休假等所有会占用人员时间或需要过程记录的事项。

目标：

- 用统一任务模型承载所有工作事项。
- 支持任务从草稿到归档的完整生命周期。
- 支持任务负责人拆分分工、成员填报进度和成果。
- 支持任务变更留痕，保证过程可追溯。
- 支持甘特图、负载、资料、通知、报表等模块联动。

## 2. 任务类型

任务类型由配置中心维护，默认包括：

- 科研：公司级、处级、中心级、市场类科研。
- 市场。
- 出差：培训、会议交流、技术支持。
- 文职。
- 党建。
- 安全。
- 学习。
- 休假。
- 其他。

不同任务类型可以配置：

- 表单字段。
- 默认审批流程。
- 是否需要负责人。
- 是否需要成员分工。
- 是否参与负载计算。
- 是否改变人员状态。
- 必需资料类型。
- 默认里程碑模板。

## 3. 任务字段

### 3.1 基础字段

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| 任务编号 | 自动生成 | 是 | 全局唯一 |
| 任务名称 | 文本 | 是 | 清晰描述工作事项 |
| 任务类型 | 枚举 | 是 | 由配置中心维护 |
| 子类型 | 枚举 | 否 | 例如外部培训、内部会议 |
| 任务级别 | 枚举 | 是 | 普通、重点、紧急 |
| 优先级 | 枚举 | 是 | 低、中、高、紧急 |
| 所属组织 | 组织 | 是 | 统计和权限口径 |
| 关联项目 | 项目 | 否 | 可为空 |
| 可见性 | 枚举 | 是 | 普通、隐藏、指定范围、个人事项 |
| 发起人 | 人员 | 是 | 提出或派发工作的人 |
| 负责人 | 人员 | 是 | 总体负责执行和汇总 |
| 验收人 | 人员 | 否 | 默认发起人，可调整 |
| 开始时间 | 日期时间 | 是 | 计划开始 |
| 截止时间 | 日期时间 | 是 | 计划完成 |
| 预计总工时 | 数字 | 否 | 用于预测是否能按期完成 |
| 任务概述 | 长文本 | 是 | 背景、目标、交付物 |
| 成果要求 | 长文本 | 否 | 验收标准和资料要求 |
| 备注 | 长文本 | 否 | 补充信息 |

### 3.2 成员字段

任务成员不是简单人员列表，每个成员都要有工作安排。

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| 成员 | 人员 | 是 | 参与人员 |
| 成员角色 | 枚举 | 是 | 负责人、核心成员、协作成员、支持人员 |
| 工作内容 | 文本 | 是 | 成员承担事项 |
| 预计总工时 | 数字 | 否 | 该成员总工作量 |
| 每日投入 | 数字/全天 | 是 | 用于负载计算 |
| 开始时间 | 日期 | 是 | 成员工作开始 |
| 截止时间 | 日期 | 是 | 成员工作截止 |
| 是否待审批 | 布尔 | 是 | 跨部门未通过时为是 |

### 3.3 里程碑字段

- 里程碑名称。
- 计划完成时间。
- 责任人。
- 阶段成果要求。
- 当前状态。
- 关联资料。

## 4. 任务状态

任务状态流：

```text
草稿 -> 待协调 -> 待确认 -> 进行中 -> 待验收 -> 已完成 -> 已归档
                  |        |        |
                  |        |        -> 验收退回 -> 进行中
                  |        -> 已暂停/有风险
                  -> 已取消
```

状态说明：

- 草稿：任务未发布，不占用正式工时。
- 待协调：存在跨部门、负载冲突、隐藏项目授权等问题。
- 待确认：协调已完成，等待负责人或发起人确认。
- 进行中：成员正式执行，开始进度填报。
- 已暂停：任务暂停，保留暂停原因和预计恢复时间。
- 有风险：任务仍在执行，但存在延期、冲突、成果风险。
- 待验收：负责人提交最终成果。
- 验收退回：验收人要求修改。
- 已完成：验收通过。
- 已归档：过程记录和资料归档。
- 已取消：任务终止。

## 5. 分工设计

分工是任务执行层的最小管理单元。

### 5.1 分工字段

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| 分工名称 | 文本 | 是 | 例如前端开发、报告撰写 |
| 分工类型 | 枚举 | 是 | 研发、文职、客服、培训等 |
| 责任人 | 人员 | 是 | 主要执行人 |
| 协作人 | 人员多选 | 否 | 协助执行 |
| 开始时间 | 日期 | 是 | 不得早于任务开始 |
| 截止时间 | 日期 | 是 | 不得晚于任务截止 |
| 每日投入 | 数字/全天 | 是 | 参与负载计算 |
| 预计总工时 | 数字 | 否 | 用于截止风险预测 |
| 成果要求 | 长文本 | 否 | 阶段或最终成果 |
| 当前进度 | 百分比 | 是 | 成员或负责人维护 |
| 当前状态 | 枚举 | 是 | 未开始、进行中、阻塞、待确认、已完成 |
| 验收人 | 人员 | 否 | 默认任务负责人 |

### 5.2 分工状态

```text
未开始 -> 进行中 -> 待确认 -> 已完成
          |        |
          |        -> 退回修改 -> 进行中
          -> 阻塞/延期
```

### 5.3 进度汇总

任务进度支持两种方式：

- 手动维护：负责人直接维护任务总进度。
- 分工加权：按分工权重和分工进度自动计算。

分工加权公式：

```text
任务进度 = sum(分工进度 * 分工权重)
```

权重默认按预计投入工时计算，也可由负责人手动调整。

截止预测：

```text
分工剩余工时 = 分工预计总工时 - 已确认投入工时
人员可用工时 = 分工截止日前每个工作日剩余可用容量之和
如果 分工剩余工时 > 人员可用工时，则产生截止风险
```

任务截止风险由所有未完成分工汇总生成。没有维护预计总工时的任务，只做日期延期和阶段成果未提交预警，不做精确预测完成日。

## 6. 任务变更

以下变更必须留痕：

- 任务概述。
- 时间计划。
- 负责人。
- 成员。
- 分工。
- 每日投入。
- 优先级。
- 任务级别。
- 可见性。
- 成果要求。

变更记录字段：

- 变更类型。
- 变更前内容。
- 变更后内容。
- 变更原因。
- 变更发起人。
- 变更确认人。
- 变更时间。

重大变更可配置是否需要重新审批。

## 7. 任务完成与归档

任务完成前校验：

- 必填分工已完成。
- 必需资料已提交。
- 阶段成果已确认。
- 未处理高风险冲突。
- 验收人已确认。

归档内容：

- 任务基础信息。
- 任务概述版本。
- 成员和分工。
- 进度汇报。
- 审批记录。
- 资料和版本。
- 验收记录。
- 风险处理记录。
