# 智能工具台功能设计

## 1. 功能目标

工具台是日常效率工具和智能体工具的统一入口。它不替代任务管理，而是服务任务执行过程中的文档处理、材料撰写、格式转换和知识问答。

目标：

- 汇聚常用软件和工具入口。
- 按角色、组织、任务类型控制可见范围。
- 支持将工具推荐到首页快捷入口。
- 支持从任务详情携带上下文打开工具。
- 为后续智能体能力扩展预留配置。

## 2. 工具分类

### 2.1 常用工具

- PDF 格式转换。
- 文档格式转换。
- 翻译。
- 图片压缩。
- OCR 识别。
- 表格处理。

### 2.2 智能体工具

- 论文辅助撰写。
- 专利辅助撰写。
- 汇报材料辅助。
- 个人知识库。
- 平台问答助手。
- 需求整理助手。

## 3. 工具字段

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| 工具名称 | 文本 | 是 | 页面展示 |
| 工具分类 | 枚举 | 是 | 常用工具、智能体工具 |
| 工具描述 | 文本 | 是 | 简要说明用途 |
| 入口类型 | 枚举 | 是 | 外链、内嵌、系统功能 |
| 入口地址 | URL/路由 | 否 | 外链或内嵌地址 |
| 图标 | 图标 | 否 | 页面展示 |
| 可见角色 | 角色多选 | 否 | 不填表示按默认 |
| 可见组织 | 组织多选 | 否 | 不填表示不限制 |
| 关联任务类型 | 类型多选 | 否 | 用于任务详情推荐 |
| 是否推荐首页 | 布尔 | 是 | 首页快捷入口 |
| 是否启用 | 布尔 | 是 | 停用后隐藏 |

## 4. 上下文传递

从任务详情打开工具时，可携带：

- 任务编号。
- 任务名称。
- 任务类型。
- 任务概述。
- 负责人。
- 成果要求。
- 已有资料。

示例：

- 在科研任务中打开论文辅助撰写，工具可读取任务概述和资料清单。
- 在汇报任务中打开 PPT 辅助，工具可读取成果要求和已有资料。

## 5. 权限规则

- 用户只能看到有权限的工具。
- 工具可见不等于任务数据可见，工具读取任务上下文仍需校验任务权限。
- 隐藏项目上下文不得传递给未授权工具或未授权用户。
- 外链工具需要明显标识会跳转到外部系统。

## 6. 使用记录

记录：

- 使用人。
- 工具名称。
- 来源页面。
- 关联任务。
- 使用时间。

用途：

- 统计工具使用频率。
- 优化首页推荐。
- 审计敏感工具访问。
