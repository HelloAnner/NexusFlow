# 角色账号矩阵

全量 e2e 不能只用 SA。以下角色必须具备独立账号，并且每个账号都需要通过浏览器登录截图和 `GET /api/auth/me` 验证。

| 角色 | 建议账号 | 必备数据范围 | 主要验证点 |
| --- | --- | --- | --- |
| SA 超级管理员 | `Anner` | 全系统 | 系统管理、邀请、权限、审计、配置、全量数据 |
| 系统管理员 | `nf_sysadmin` | 可限制在中心或组织 | 组织、人员、配置、技能、基础字典 |
| 中心主任/书记 | `nf_center_lead` | 全中心 | 中心任务、项目、人员、报表、最终协调 |
| 中心副主任/技术总监 | `nf_center_deputy` | 分管部门和分管项目 | 分管范围数据裁剪、派发范围 |
| 部门主任/技术监督 | `nf_dept_lead` | 本部门 | 跨部门审批、本部门人员负载 |
| 部门副主任 | `nf_dept_deputy` | 授权部门子集 | 部分管理权限、边界裁剪 |
| 项目负责人 | `nf_project_owner` | 所属项目 | 项目详情、成员、项目任务、项目资料 |
| 任务负责人 | `nf_task_owner` | 负责的任务 | 分工、进度、提交验收、成果 |
| 普通员工 | `nf_employee` | 本人参与任务 | 我的任务、进度填报、资料上传 |
| 待审核用户 | `nf_pending` | 无业务数据 | 审核状态页、业务 API 403 |
| 隐藏项目未授权用户 | `nf_hidden_denied` | 普通数据，不含隐藏项目 | 隐藏项目不可搜索、不可见、无泄露 |
| 隐藏项目授权用户 | `nf_hidden_allowed` | 指定隐藏项目 | 隐藏任务、资料、甘特、搜索可见 |

## 角色验收

每个账号必须记录：

- 登录页截图。
- 默认首页截图。
- `/api/auth/me` 响应。
- `/api/permissions/me` 响应。
- 可见菜单截图。
- 访问无权限页面或 API 的 403 证据。

## 最低账号创建路径

优先通过邀请注册创建业务角色账号，因为这同时覆盖邀请、注册、审核和角色入口：

1. SA 新建组织和角色。
2. SA 新建邀请模板。
3. SA 生成邀请链接。
4. 目标用户通过链接注册。
5. SA 审核通过或保留待审核。
6. 目标用户登录并确认默认入口。
