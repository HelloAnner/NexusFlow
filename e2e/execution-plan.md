# 分批执行计划

517 条用例不能无序执行。按以下批次推进，先保证基础环境、数据和核心链路，再扩展到权限矩阵和边界场景。

## P0 环境与数据准备

目标：确认服务器、登录、基础数据可用。

- `E2E-00-001` 到 `E2E-00-015`
- [roles.md](./roles.md) 所有账号登录验收。
- [test-data-plan.md](./test-data-plan.md) 数据准备验收。

通过条件：

- SA 可登录。
- 所有角色账号存在。
- `healthz/readyz` 有证据。
- 基础组织、项目、人员、任务、资料、配置数据齐全。

## P1 主干闭环

目标：验证从建档到任务完成归档的最短业务闭环。

- 组织和人员：`E2E-01-001` 到 `E2E-01-023`
- 任务主流程：`E2E-03-014` 到 `E2E-03-044`
- 资料验收：`E2E-06-010`、`E2E-06-025` 到 `E2E-06-028`
- 首页待办跳转：`E2E-07-007` 到 `E2E-07-011`

通过条件：

- 能新建人员、项目、任务。
- 能分工、填进度、提交验收、通过/退回、归档。
- 截图和 API 后验都通过。

## P2 权限与隐藏项目

目标：验证系统不会泄露不该看的数据。

- 权限全部：`E2E-02-001` 到 `E2E-02-036`
- 隐藏项目项目侧：`E2E-11-005`、`E2E-11-023`、`E2E-11-035` 到 `E2E-11-039`
- 甘特搜索脱敏：`E2E-12-030`、`E2E-12-031`、`E2E-12-041`
- 资料隐藏：`E2E-06-029`

通过条件：

- 未授权用户在页面和 API 都不可见。
- 允许“已占用”的场景不泄露标题、摘要、项目、时间。
- 授权取消即时生效。

## P3 派发协同与冲突

目标：验证跨部门、负载和冲突处理。

- 派发协同全部：`E2E-04-001` 到 `E2E-04-034`
- 负载冲突全部：`E2E-05-001` 到 `E2E-05-029`
- 任务冲突检查：`E2E-03-026` 到 `E2E-03-031`

通过条件：

- 高风险阻断，中风险二次确认，低风险提示。
- 跨部门审批单生成并可审批。
- 冲突处理后状态和负载数据一致。

## P4 配置、SA、邀请

目标：验证系统初始化和配置发布能力。

- 配置中心全部：`E2E-08-001` 到 `E2E-08-040`
- SA 与邀请全部：`E2E-13-001` 到 `E2E-13-065`

通过条件：

- 邀请链接一次展示完整 token，后续脱敏。
- 注册、审核、待审核隔离完整。
- 配置草稿、发布、停用、版本记录、审计完整。

## P5 信息入口、报表、工具、甘特搜索

目标：验证日常使用效率入口和统计口径。

- 首页全部：`E2E-07-001` 到 `E2E-07-036`
- 通知报表全部：`E2E-09-001` 到 `E2E-09-032`
- 工具台全部：`E2E-10-001` 到 `E2E-10-025`
- 项目管理全部：`E2E-11-001` 到 `E2E-11-040`
- 甘特搜索全部：`E2E-12-001` 到 `E2E-12-043`

通过条件：

- 待办、通知、搜索、甘特都能跳到真实对象。
- 报表指标可由底层 API 数据解释。
- 工具上下文和使用记录可追踪。
